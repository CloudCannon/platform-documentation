This guide covers how to add <comp.GlossaryTerm term="/user/glossary/i/internationalization.yml">internationalization (i18n)</comp.GlossaryTerm> and multilingual support to your static site on CloudCannon using <comp.GlossaryTerm term="/user/glossary/r/rosey.yml">Rosey</comp.GlossaryTerm>. You'll learn how Rosey works, how to set it up, and how to optionally add the <comp.GlossaryTerm term="/user/glossary/r/rosey-cloudcannon-connector.yml">Rosey CloudCannon Connector</comp.GlossaryTerm> for inline translation editing in the <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm>.

This guide is structured in two sections:

- **Rosey** — install and configure Rosey to build locale-specific versions of your *Site*
- **Rosey CloudCannon Connector** — optionally add inline translation editing to the *Visual Editor*

The Connector is not required to use Rosey. You can complete the first section and have a fully functional multilingual *Site* without it.

Before you begin, this guide assumes you already have a static *Site* connected to CloudCannon and NPM initialized on your project (`npm init`).

<comp.Notice info_type="info">
  This is a fresh setup guide. If you're migrating from CloudCannon's deprecated built-in i18n, see the [Rosey migration guide](/documentation/developer-guides/migrate-from-legacy-internationalization-to-rosey/) instead.
</comp.Notice>
