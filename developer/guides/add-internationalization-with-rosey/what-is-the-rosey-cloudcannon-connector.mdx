The <comp.GlossaryTerm term="/user/glossary/r/rosey.yml">Rosey</comp.GlossaryTerm> CloudCannon Connector is an optional add-on for Rosey sites that enables inline translation editing in the <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm>. It is not required to use Rosey — your multilingual site will build and run correctly without it. The Connector is for teams who want to edit translations visually within CloudCannon rather than editing locale JSON files directly.

The Connector is itself built using the [*Visual Editor API*](/documentation/developer-articles/what-is-the-visual-editor-api/), making it a real-world example of a custom *Visual Editor* integration.

## What the Connector adds

When you load a Rosey site with the Connector installed in the *Visual Editor*, you get:

- **A locale switcher** — a floating button that lets editors switch between available locales directly in the *Visual Editor* preview
- **Inline translation editing** — clicking on any `data-rosey`-tagged element opens an inline editor connected to the corresponding locale JSON entry
- **Stale translation detection** — elements whose source text has changed since the translation was written are highlighted with amber borders, with a count badge and resolve panel to help translators work through them
- **AI-powered translation assistance** — optional AI suggestions to speed up translation workflows

## How the Connector works

The Connector is a client-side JavaScript library that you load in your site's layout. When the site loads in the *Visual Editor*, the Connector:

1. Detects all `data-rosey`-tagged elements on the page
2. Injects the locale switcher interface
3. Connects each tagged element to its corresponding entry in the locale JSON via the *Visual Editor API*
4. Exposes inline editors that write directly to your CloudCannon Datasets when editors save

The Connector uses `window.inEditorMode` to detect when the site is running inside the *Visual Editor*, so the editing interface only appears in CloudCannon — not on your live site.

## Prerequisites

- Rosey v2 configured and working on your site (see [Set up Rosey on CloudCannon](/documentation/developer-guides/add-internationalization-with-rosey/set-up-rosey-on-cloudcannon/))
- `data-rosey` attributes on your translatable elements
- CloudCannon *Visual Editor* enabled for your site
- Locale files stored as CloudCannon Datasets

You don't need <comp.GlossaryTerm term="/user/glossary/e/editable-region.yml">Editable Regions</comp.GlossaryTerm> or <comp.GlossaryTerm term="/user/glossary/b/bookshop.yml">Bookshop</comp.GlossaryTerm> to use the Connector. If your site does use Bookshop, the Connector is automatically compatible — no additional configuration is required.

## What the Connector does not do

The Connector does not replace Rosey. It extends Rosey by connecting the *Visual Editor* to your locale JSON files. Rosey still handles the build-time translation process — generating localized output from your JSON files during the postbuild step.
