Welcome to this guide on building custom interfaces with CloudCannon's <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm>. The *Visual Editor API* lets you run your own JavaScript inside the *Visual Editor* to read and write content, upload files, and build editing experiences tailored to your <comp.GlossaryTerm term="/user/glossary/s/site.yml">Site</comp.GlossaryTerm>.

To learn these skills, we'll build a real example: the Doodle Gallery. The Doodle Gallery adds a drawing tool to the *Visual Editor* — an editor sketches a doodle, gives it a caption, and submits it. Behind the scenes, the integration uploads the drawing as an image and adds it to a gallery data file that renders on the live *Site*. You can find the complete example in the <a href="https://github.com/CloudCannon/doodle-example" target="_blank" rel="noopener">doodle-example repository</a>.

The Doodle Gallery is built with Astro and React, but the *Visual Editor API* skills you'll learn transfer to any framework or static site generator. For a full reference of every method the API provides, see the [Visual Editor API articles](/documentation/developer-articles/what-is-the-visual-editor-api/).

In this guide, we'll detect the *Visual Editor* and initialize the API, render a custom interface that only appears for editors, upload a file, and write new data back to the *Site* — without overwriting other people's changes.

Before we begin, this guide assumes you already have a *Site* whose pages render in CloudCannon's *Visual Editor*, and that you're comfortable writing JavaScript.
