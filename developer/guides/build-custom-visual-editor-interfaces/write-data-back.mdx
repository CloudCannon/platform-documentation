We're still inside the same `submit()` function. The data file is already referenced and locked, and we have the path to the uploaded image. All that's left is to add the new doodle to the gallery.

We read the current `doodles` array, then append a new item to it. Each item holds the uploaded image path and the caption the editor typed.

<comp.CodeBlock language="tsx" source="src/components/doodle.tsx">
```
  const doodles = await file.data.get({ slug: 'doodles' }); /*1*/
  if (!Array.isArray(doodles)) {
    throw new Error('The data file is incorrectly formatted.');
  }

  await file.data.addArrayItem({ /*2*/
    slug: 'doodles',
    index: doodles.length, /*3*/
    value: { src: uploadedPath, caption }, /*4*/
  });
}
```
<comp.Annotation number="1">
  Read the current `doodles` array from the data file with `file.data.get()`, passing the `slug` of the field we want.
</comp.Annotation>
<comp.Annotation number="2">
  Add a new item to that array with `file.data.addArrayItem()`.
</comp.Annotation>
<comp.Annotation number="3">
  Append to the end by using the current array length as the index.
</comp.Annotation>
<comp.Annotation number="4">
  The new item carries the uploaded image path and the caption, matching the shape of the other doodles in the file.
</comp.Annotation>
</comp.CodeBlock>

We used `file()` with a path here, rather than `currentFile()`, because we're writing to a specific data file instead of the page the editor has open. For the full set of ways to read and write files, see [Read and write data with the Visual Editor API](/documentation/developer-articles/read-and-write-data-with-the-visual-editor-api/).

That completes `submit()`. The new doodle now appears in any open <comp.GlossaryTerm term="/user/glossary/d/data-editor.yml">Data Editor</comp.GlossaryTerm> and is written to `doodles.json`, ready to commit when the editor saves the <comp.GlossaryTerm term="/user/glossary/s/site.yml">Site</comp.GlossaryTerm>. Because the gallery page renders straight from that data file, the new doodle shows up on the live *Site* after the next build.
