The Doodle Gallery's drawing tool should only appear while someone is editing in the <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm> — never on the live <comp.GlossaryTerm term="/user/glossary/s/site.yml">Site</comp.GlossaryTerm>. So before our integration does anything else, it checks whether it's running inside the *Visual Editor*, and only then initializes the *Visual Editor API*.

CloudCannon sets `window.inEditorMode` to `true` when your *Site* is loaded inside the *Visual Editor*, and leaves it `undefined` everywhere else. We use it to gate the whole integration. Once we know we're in the editor, we request the API with `window.CloudCannonAPI.useVersion('v1', true)`.

To get type checking and autocomplete as you write, type the global `window` with the types from the `@cloudcannon/visual-editor-api` package. For more on typing the API, see [Use TypeScript with the Visual Editor API](/documentation/developer-articles/use-typescript-with-the-visual-editor-api/).

<comp.CodeBlock language="tsx" source="src/components/doodle.tsx">
```
import { useState, useEffect } from 'react';
import type { CloudCannonVisualEditorAPIV1, CloudCannonVisualEditorWindow } from '@cloudcannon/visual-editor-api'; /*1*/

declare const window: CloudCannonVisualEditorWindow & { inEditorMode: true | undefined } /*2*/

export function Doodler() {
  let cloudcannonApi: CloudCannonVisualEditorAPIV1 | undefined;
  const [inCloudCannonEditor, setInCloudCannonEditor] = useState(false);

  useEffect(() => {
    if (window.inEditorMode) { /*3*/
      setInCloudCannonEditor(true);
      cloudcannonApi = window.CloudCannonAPI?.useVersion('v1', true); /*4*/
    }
  });

  if (!inCloudCannonEditor) {
    return undefined; /*5*/
  }

  // ...render the drawing tool
}
```
<comp.Annotation number="1">
  Import the API and window types from the `@cloudcannon/visual-editor-api` package.
</comp.Annotation>
<comp.Annotation number="2">
  Declare `window` with CloudCannon's types so `inEditorMode` and `CloudCannonAPI` are typed.
</comp.Annotation>
<comp.Annotation number="3">
  Only run inside the *Visual Editor* — `window.inEditorMode` is `true` there and `undefined` on the live *Site*.
</comp.Annotation>
<comp.Annotation number="4">
  Request v1 of the API with `useVersion('v1', true)`. The second argument also exposes the handler as `window.CloudCannon`.
</comp.Annotation>
<comp.Annotation number="5">
  Render nothing outside the *Visual Editor*, so the tool never appears for site visitors.
</comp.Annotation>
</comp.CodeBlock>

The Doodle Gallery does this inside a React `useEffect`, but the pattern is the same in any framework: check `window.inEditorMode`, then call `useVersion('v1')` once the API is available. If your integration code can run before the *Visual Editor* finishes loading, you can wait for the `cloudcannon:load` event instead — see [Initialize the Visual Editor API](/documentation/developer-articles/initialize-the-visual-editor-api/) for both patterns.
