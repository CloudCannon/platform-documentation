## Go further with the Visual Editor API

The Doodle Gallery only uses a handful of the methods the *Visual Editor API* provides. These articles cover the rest, so you can build integrations that react to changes, reuse CloudCannon's own editing surfaces, and work across many files.

- [Listen for changes](/documentation/developer-articles/listen-for-changes-with-the-visual-editor-api/) — Keep your interface in sync when content changes in the *Visual Editor*.
- [Create a custom Data Panel](/documentation/developer-articles/create-a-custom-data-panel-with-the-visual-editor-api/) — Open CloudCannon's own input panels from your integration.
- [Create custom inline editing](/documentation/developer-articles/create-custom-inline-editing-with-the-visual-editor-api/) — Make elements on the page directly editable.
- [Work with Collections and Datasets](/documentation/developer-articles/work-with-collections-and-datasets-using-the-visual-editor-api/) — Read and react to many files at once.
- [Read and write data](/documentation/developer-articles/read-and-write-data-with-the-visual-editor-api/) — The full set of methods for file data, body content, and raw source.

## Try another integration

The same skills you used for the Doodle Gallery apply to all kinds of custom tools. If you're looking for somewhere to start, try one of these.

- A live word or character counter that updates as an editor types.
- An SEO preview card that renders a Google or social result from the page's title and description.
- A validation overlay that flags broken links or missing image alt text directly on the preview.
- A content dashboard that lists files across your <comp.GlossaryTerm term="/user/glossary/c/collection.yml">Collections</comp.GlossaryTerm> and shows their status inside the *Visual Editor*.
