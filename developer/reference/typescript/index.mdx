CloudCannon provides TypeScript type definitions for JavaScript and TypeScript configuration files, enabling autocomplete and type checking in your IDE. These types are also useful for creating tooling and working with LLMs.

## Configuration Types

The [@cloudcannon/configuration-types](https://www.npmjs.com/package/@cloudcannon/configuration-types) package is built with [Zod](https://zod.dev/), a TypeScript-first schema validation library. This means the schemas can be used for more than just type definitions — you can use them for runtime validation, JSON Schema generation, and integration with the broader Zod ecosystem.

Install the package:

<comp.CodeBlock language="bash" source="Terminal">
```
npm install --save-dev @cloudcannon/configuration-types
```
</comp.CodeBlock>

Use JSDoc annotations in JavaScript files for autocomplete:

<comp.CodeBlock language="javascript" source="cloudcannon.config.cjs">
```
/** @type {import('@cloudcannon/configuration-types').Configuration} */
module.exports = {
  // Your configuration with full autocomplete
};
```
</comp.CodeBlock>

Or use TypeScript directly:

<comp.CodeBlock language="typescript" source="cloudcannon.config.ts">
```
import type { Configuration } from '@cloudcannon/configuration-types';

const config: Configuration = {
  // Your configuration with full autocomplete
};

export default config;
```
</comp.CodeBlock>

### Available Types

The package exports the following types:

- `Configuration` — [CloudCannon configuration file](/documentation/developer-reference/configuration-file/)
- `Routing` — [Routing file](/documentation/developer-reference/routing-file/) configuration
- `InitialSiteSettings` — [Initial site settings file](/documentation/developer-reference/initial-site-settings-file/) configuration

The source code is available on [GitHub](https://github.com/CloudCannon/configuration-types).

## JavaScript API Types

For the CloudCannon JavaScript API, the [@cloudcannon/javascript-api](https://github.com/CloudCannon/javascript-api) package provides TypeScript declarations.

<comp.CodeBlock language="bash" source="Terminal">
```
npm install --save-dev @cloudcannon/javascript-api
```
</comp.CodeBlock>
