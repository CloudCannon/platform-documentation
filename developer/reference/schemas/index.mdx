CloudCannon provides JSON Schemas for all configuration files, enabling autocomplete, validation, and inline documentation in your IDE.

All schemas are registered with <a href="https://github.com/SchemaStore/schemastore" target="_blank" rel="noopener noreferrer">SchemaStore</a>, so most IDEs will automatically detect and apply them based on filename — no configuration required. Schemas are also available from the <a href="https://github.com/CloudCannon/configuration-types" target="_blank" rel="noopener noreferrer">@cloudcannon/configuration-types</a> package on GitHub.

## Available Schemas

Most IDEs will automatically apply these schemas based on filename. If needed, you can explicitly specify the schema:

- **JSON files**: Add a `$schema` property to your configuration file
- **YAML files**: Add a `# yaml-language-server` comment at the top of the file (requires the <a href="https://marketplace.visualstudio.com/items?itemName=redhat.vscode-yaml" target="_blank" rel="noopener noreferrer">Red Hat YAML extension</a> in VS Code/Cursor)

**Configuration File** (`cloudcannon.config.json` or `cloudcannon.config.yaml`):

<comp.CodeTabs>
  <comp.CodeTab language="json" source="cloudcannon.config.json">
```
{
  "$schema": "https://raw.githubusercontent.com/CloudCannon/configuration-types/main/cloudcannon-config.schema.json"
}
```
  </comp.CodeTab>
  <comp.CodeTab language="yaml" source="cloudcannon.config.yaml">
```
# yaml-language-server: $schema=https://raw.githubusercontent.com/CloudCannon/configuration-types/main/cloudcannon-config.schema.json
```
  </comp.CodeTab>
</comp.CodeTabs>

**Routing File** (`.cloudcannon/routing.json`):

<comp.CodeBlock language="json" source=".cloudcannon/routing.json">
```
{
  "$schema": "https://raw.githubusercontent.com/CloudCannon/configuration-types/main/dist/cloudcannon-routing.schema.json",
  "routes": []
}
```
</comp.CodeBlock>

**Initial Site Settings File** (`.cloudcannon/initial-site-settings.json`):

<comp.CodeBlock language="json" source=".cloudcannon/initial-site-settings.json">
```
{
  "$schema": "https://raw.githubusercontent.com/CloudCannon/configuration-types/main/dist/cloudcannon-initial-site-settings.schema.json",
  "ssg": "eleventy"
}
```
</comp.CodeBlock>

## IDE Configuration

Most modern IDEs automatically detect CloudCannon configuration files and apply the correct schema via SchemaStore integration.

### VS Code and Cursor

**JSON files** work automatically with no setup required.

**YAML files** require the <a href="https://marketplace.visualstudio.com/items?itemName=redhat.vscode-yaml" target="_blank" rel="noopener noreferrer">Red Hat YAML extension</a>. After installing, add schema associations to your `settings.json`:

<comp.CodeBlock language="json" source=".vscode/settings.json">
```
{
  "yaml.schemas": {
    "https://raw.githubusercontent.com/CloudCannon/configuration-types/main/cloudcannon-config.schema.json": "cloudcannon.config.yaml"
  }
}
```
</comp.CodeBlock>

For more information, see the <a href="https://code.visualstudio.com/docs/languages/json#_json-schemas-and-settings" target="_blank" rel="noopener noreferrer">VS Code JSON Schema documentation</a>.

### JetBrains IDEs

WebStorm, IntelliJ IDEA, and other JetBrains IDEs have built-in JSON and YAML schema support that works automatically via SchemaStore integration.

For manual configuration, open **Settings** > **Languages & Frameworks** > **Schemas and DTDs** > **JSON Schema Mappings**. See the <a href="https://www.jetbrains.com/help/idea/json.html#ws_json_schema_add_custom" target="_blank" rel="noopener noreferrer">JetBrains JSON Schema documentation</a> for details.

### Sublime Text

Install the <a href="https://packagecontrol.io/packages/LSP" target="_blank" rel="noopener noreferrer">LSP</a> and <a href="https://packagecontrol.io/packages/LSP-json" target="_blank" rel="noopener noreferrer">LSP-json</a> packages for JSON schema support — schemas are automatically discovered via SchemaStore. For YAML files, also install <a href="https://packagecontrol.io/packages/LSP-yaml" target="_blank" rel="noopener noreferrer">LSP-yaml</a>.

### Neovim

Use <a href="https://github.com/neovim/nvim-lspconfig" target="_blank" rel="noopener noreferrer">nvim-lspconfig</a> with `jsonls` for JSON files and `yamlls` for YAML files. Both language servers automatically discover schemas through SchemaStore.

### Zed

Zed has built-in JSON schema support that works automatically via SchemaStore. See the <a href="https://zed.dev/docs/languages/json" target="_blank" rel="noopener noreferrer">Zed JSON documentation</a> for more details.

## Legacy SSG-Specific Schemas

For older Sites using the non-unified configuration format, SSG-specific schemas are available. Unlike the unified schema, these are not registered with SchemaStore and must be specified manually using the `$schema` property (JSON) or `# yaml-language-server` comment (YAML).

We recommend [migrating to Unified Configuration](/documentation/guides/unified-configuration-migration-guide/) to benefit from automatic schema detection and the latest features.

}
