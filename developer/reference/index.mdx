Welcome to CloudCannon's Developer Reference. It's organized into two parts: the files you add to your repository to configure CloudCannon, and the tools and controls for working with the CloudCannon platform.

## Site configuration

The files you add to your repository to customize how CloudCannon builds and edits your site — the configuration file, initial site settings, routing, editable regions, and the schemas and types that validate them.

[Browse Site configuration](/documentation/developer-reference/site-configuration/)

- [Configuration File](/documentation/developer-reference/configuration-file/) — collections, inputs, editing interfaces, and more
- [Initial Site Settings File](/documentation/developer-reference/initial-site-settings-file/) — pre-configure SSG, hosting mode, and build settings
- [Routing File](/documentation/developer-reference/routing-file/) — custom redirects and HTTP headers for hosted Sites
- [Editable Regions](/documentation/developer-reference/editable-regions/) — mark parts of your built HTML as editable
- [Visual Editor API](/documentation/developer-reference/visual-editor-api/) — connect your front end to the visual editor
- [JSON Schemas](/documentation/developer-reference/schemas/) and [TypeScript Types](/documentation/developer-reference/typescript/) — IDE autocomplete and validation

## Platform

The tools for working with the CloudCannon backend programmatically — the HTTP API, the SDK, and the CLI — plus the permissions that control what your team can do.

[Browse Platform](/documentation/developer-reference/platform/)

- [CLI](/documentation/developer-reference/cli/) — manage CloudCannon from the terminal
- [TypeScript SDK](/documentation/developer-reference/sdk/) — a typed client for the API
- [API](/documentation/developer-reference/api/) — manage organizations, projects, sites, and builds over HTTP
- [Permissions](/documentation/developer-reference/permissions/) — control what your team members can do
