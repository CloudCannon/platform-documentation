The <comp.GlossaryTerm term="/user/glossary/v/visual-editor-api.yml">Visual Editor API</comp.GlossaryTerm> is a JavaScript API for building custom integrations that run inside CloudCannon's <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm>. Obtain the v1 API object with `window.CloudCannonAPI.useVersion('v1', true)`, then call its methods to access files, <comp.GlossaryTerm term="/user/glossary/c/collection.yml">Collections</comp.GlossaryTerm>, and <comp.GlossaryTerm term="/user/glossary/d/dataset.yml">Datasets</comp.GlossaryTerm>, read and write content, upload files, and build custom editing interfaces. This reference covers the v1 API.

For an introduction and walkthroughs, please read our documentation on the [Visual Editor API](/documentation/developer-articles/what-is-the-visual-editor-api/) and the [Building custom visual editor interfaces](/documentation/developer-guides/build-custom-visual-editor-interfaces/) guide. For IDE autocomplete and validation, install the `@cloudcannon/visual-editor-api` package — see [Use TypeScript with the Visual Editor API](/documentation/developer-articles/use-typescript-with-the-visual-editor-api/).

## Objects

Several methods return objects, and a `File` exposes `data` and `content` objects. Each is documented on its own page:

- [`File`](/documentation/developer-reference/visual-editor-api/File/)
- [`FileData`](/documentation/developer-reference/visual-editor-api/FileData/)
- [`FileContent`](/documentation/developer-reference/visual-editor-api/FileContent/)
- [`Collection`](/documentation/developer-reference/visual-editor-api/Collection/)
- [`Dataset`](/documentation/developer-reference/visual-editor-api/Dataset/)
- [`TextEditableRegion`](/documentation/developer-reference/visual-editor-api/TextEditableRegion/)
