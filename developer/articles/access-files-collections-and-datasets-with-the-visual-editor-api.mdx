<comp.Notice info_type="important">
  The *Visual Editor API* is designed for advanced CloudCannon users. If you need help with custom integrations or your CloudCannon setup in general, please contact our friendly [support team](/support/).
</comp.Notice>

CloudCannon's *Visual Editor API* exposes files, <comp.GlossaryTerm term="/user/glossary/c/collection.yml">Collections</comp.GlossaryTerm>, and <comp.GlossaryTerm term="/user/glossary/d/dataset.yml">Datasets</comp.GlossaryTerm> from your <comp.GlossaryTerm term="/user/glossary/s/site.yml">Site</comp.GlossaryTerm> code through the *Visual Editor API*. After you access one of these, you can read and write stored values and text, listen for `change` and `delete` events, list or filter the members of a *Collection* or *Dataset*, and claim a file lock in the *Visual Editor*.

This article assumes you know how to [initialize the Visual Editor API](/documentation/developer-articles/initialize-the-visual-editor-api/).

## Access a file

You can access your website files via the *Visual Editor API* in two ways: `currentFile()`, which returns the file currently open in the *Visual Editor*, or `file(path)`, which returns a specific file by its source path regardless of what is currently open in the *Visual Editor*.

Both functions return a file object with the same interface.

To access a file:

1. Open your website files in your local development environment.
2. Open the JavaScript file where your *Visual Editor API* integration runs.
3. Obtain the v1 API object in your integration script.
4. Call `api.currentFile()` for the file open in the preview, or call `api.file('path/from/site/root.md')` with a path relative to your *Site* root.

<comp.CodeBlock language="javascript" source="script.js">
```
const current = api.currentFile(); /*1*/
const specific = api.file('/content/pages/about.md'); /*2*/
```
<comp.Annotation number="1">
  This function accesses the file currently open in the *Visual Editor*.
</comp.Annotation>
<comp.Annotation number="2">
  This function accesses the file `/content/pages/about.md`, regardless of what is currently open in the *Visual Editor*.
</comp.Annotation>
</comp.CodeBlock>

<comp.Notice info_type="info">
  Not every page in the *Visual Editor* has an associated file. If `currentFile()` is called for a page without an associated file, it may throw an error or fail silently. Wrap `currentFile()` in a `try`/`catch` block to handle this gracefully.
</comp.Notice>

## Access a Collection

A [Collection](/documentation/developer-articles/configure-your-collections/) is a folder of files with a similar format, defined under `collections_config` in your *CloudCannon Configuration File*. Calling `api.collection()` allows you to access a *Collection* by passing its key (the key must match the entry you configured under `collections_config`).

To access a *Collection*:

1. Open your website files in your local development environment.
2. Open the JavaScript file where your *Visual Editor API* integration runs.
3. Call `api.collection('example-key')`, where `example-key` is a *Collection* key under `collections_config` in your *CloudCannon Configuration File*.

<comp.CodeBlock language="javascript" source="script.js">
```
const posts = api.collection('posts'); /*1*/
```
<comp.Annotation number="1">
  This function accesses the *Collection* `posts`, as defined in the *CloudCannon Configuration File*.
</comp.Annotation>
</comp.CodeBlock>

## Access a *Dataset*

A [Dataset](/documentation/developer-articles/define-your-data/) is a structured data file or folder defined under `data_config` in your *CloudCannon Configuration File* — for example, navigation links, locale strings, or site settings. Calling `api.dataset()` allows you to access a *Dataset* by passing its key (the key must match the entry you configured under `data_config`).

To access a *Dataset*:

1. Open your website files in your local development environment.
2. Open the JavaScript file where your *Visual Editor API* integration runs.
3. Call `api.dataset('example-key')`, where `example-key` is a *Dataset* key under `data_config` in your *CloudCannon Configuration File*.

<comp.CodeBlock language="javascript" source="script.js">
```
const locales = api.dataset('locales'); /*1*/
```
<comp.Annotation number="1">
  This function accesses the *Dataset* `locales`, as defined in the *CloudCannon Configuration File*.
</comp.Annotation>
</comp.CodeBlock>
