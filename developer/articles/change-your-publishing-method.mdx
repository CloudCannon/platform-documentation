<comp.Notice info_type="permissions">
  **Permissions required**

  Members of the *Owners* and *Developers* [Default Permission Groups](/documentation/user-articles/what-are-default-permission-groups/), or [Custom Permission Groups](/documentation/developer-articles/what-are-custom-permission-groups/) with the `site:settings:git:write` permission, can change the *Publishing Method* for a *Site*.
</comp.Notice>

The *Publishing Method* controls how CloudCannon moves changes from a <comp.GlossaryTerm term="/user/glossary/s/site.yml">Site</comp.GlossaryTerm> to its <comp.GlossaryTerm term="/user/glossary/p/publish-branch.yml">Publish Branch</comp.GlossaryTerm>. CloudCannon offers two *Publishing Methods*: *Merge Immediately* and *Pull Request*. For more information, please read our documentation on [what publishing is](/documentation/user-articles/what-is-publishing/) and [what Publishing Methods are](/documentation/user-articles/what-are-publishing-methods/).

These instructions assume you have already [connected a Publish Branch](/documentation/developer-articles/connect-a-publish-branch/) to your *Site*.

To change the *Publishing Method* for a *Site*:

1. From your *Site*, click *Site Settings* in the *Site Navigation*.
2. Click *Publishing* in the *Section Navigation*.
3. Under *Publishing Method*, select *Merge changes immediately* or *Create a Pull Request*.
4. Click *Update Publishing Settings*.

CloudCannon will save your *Publishing Method* preference for the *Site*.

<comp.DocShot docshot_key="CloudCannon-Documentation-Publishing-Connected" alt="A screenshot of the Publishing page in Site Settings shows the Publishing Method options for Merge changes immediately and Create a Pull Request." title="The Publishing page in Site Settings" type="screenshot" />

To set a default *Publishing Method* for new branched *Sites* in a *Project*, please read our documentation on [configuring Branch Defaults for your Project](/documentation/developer-articles/configure-branch-defaults-for-your-project/).
