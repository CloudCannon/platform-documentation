The CloudCannon JavaScript API v1 is a frontend API you can use inside the *Visual Editor* runtime to read and update content programmatically.

You can use it to build custom editing experiences, such as live previews, custom data panels, and advanced editing controls on top of your site UI.

<comp.Notice info_type="important">
  **Which API do you need?**

  Use the JavaScript API v1 when your code runs in the browser inside CloudCannon's *Visual Editor*.

  If you need backend API access, contact support to request private beta access.
</comp.Notice>

## How JavaScript API v1 works

When your page loads in the *Visual Editor*, CloudCannon exposes an API router on `window.CloudCannonAPI`. You select v1 from that router and then work with file, collection, and dataset objects.

At a high level, v1 gives you:

- File access with `currentFile()` and `file(path)`.
- Structured data access through `file.data`.
- Body content access through `file.content`.
- Collection and dataset access with `collection(key)` and `dataset(key)`.
- Reactive updates with `addEventListener`.

## Why use v1

v1 uses an object-based model that maps directly to editing concepts in CloudCannon. This makes complex workflows easier to compose than legacy global function patterns.

You can use v1 to:

- Build custom interfaces for editors.
- Keep live previews in sync with content updates.
- Edit related files from a single view.
- Reuse strongly typed APIs in TypeScript projects.

## Legacy context

CloudCannon previously supported older live-editing integrations that relied on legacy API patterns. For new implementation work, use JavaScript API v1 only.
