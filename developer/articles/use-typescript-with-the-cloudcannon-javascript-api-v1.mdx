TypeScript support helps you build JavaScript API v1 integrations with autocomplete, safer refactors, and clearer contracts for tooling code.

## Install package types

Install the JavaScript API package in your project:

<comp.CodeBlock language="bash" source="Terminal">
```
npm install --save-dev @cloudcannon/javascript-api
```
</comp.CodeBlock>

## Add API types to window usage

Import the editor window and API interfaces, then use them where you initialize your integration.

<comp.CodeBlock language="typescript" source="src/cloudcannon-api.ts">
```
import type {
  CloudCannonEditorWindow,
  CloudCannonJavaScriptV1API
} from '@cloudcannon/javascript-api';

export function getCloudCannonApi(): CloudCannonJavaScriptV1API | null {
  const win = window as CloudCannonEditorWindow;
  return win.CloudCannonAPI?.useVersion('v1') ?? null;
}
```
</comp.CodeBlock>

## Build typed editor helpers

Use typed wrappers for reusable operations in previews, widgets, and custom controls.

<comp.CodeBlock language="typescript" source="src/file-helpers.ts">
```
import type { CloudCannonJavaScriptV1APIFile } from '@cloudcannon/javascript-api';

export async function updateTitle(
  file: CloudCannonJavaScriptV1APIFile,
  title: string
): Promise<void> {
  await file.data.set({ slug: 'title', value: title });
}
```
</comp.CodeBlock>

## Use typed APIs in tooling workflows

Typed helper functions make internal tooling easier to maintain across larger codebases.

<comp.CodeBlock language="typescript" source="src/content-audit.ts">
```
import type { CloudCannonJavaScriptV1API } from '@cloudcannon/javascript-api';

export async function listDraftFiles(api: CloudCannonJavaScriptV1API): Promise<string[]> {
  const files = await api.files();
  const drafts: string[] = [];

  for (const file of files) {
    const data = await file.data.get();
    if (data?.draft === true) drafts.push(file.path);
  }

  return drafts;
}
```
</comp.CodeBlock>
