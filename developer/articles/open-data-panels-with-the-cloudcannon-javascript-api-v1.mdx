This article shows two panel patterns in JavaScript API v1:

- Open built-in field editing with `file.data.edit`.
- Open custom panels with `createCustomDataPanel`.

## Prerequisites

Before you begin, this article assumes you already initialize v1 as shown in [Get started with the CloudCannon JavaScript API v1](/documentation/developer-articles/get-started-with-the-cloudcannon-javascript-api-v1/).

## Open a field editor

Use `file.data.edit()` when you want CloudCannon to open the standard editor for a field.

<comp.CodeBlock language="javascript" source="src/open-field-editor.js">
```
const file = api.currentFile();

button.addEventListener('click', (event) => {
  const rect = event.currentTarget.getBoundingClientRect();
  file.data.edit({
    slug: 'title',
    style: 'panel',
    position: {
      x: rect.left,
      y: rect.bottom,
      left: rect.left,
      width: rect.width,
      top: rect.top,
      height: rect.height
    }
  });
});
```
</comp.CodeBlock>

## Open a custom data panel

Use `createCustomDataPanel()` when you need a custom editing workflow with your own data and input configuration.

<comp.CodeBlock language="javascript" source="src/open-custom-panel.js">
```
await api.createCustomDataPanel({
  title: 'Card settings',
  data: {
    heading: 'Welcome',
    showImage: true
  },
  config: {
    _inputs: {
      heading: {
        type: 'text'
      },
      showImage: {
        type: 'boolean'
      }
    }
  },
  onChange: (updatedData) => {
    applySettings(updatedData);
  }
});
```
</comp.CodeBlock>
