Bynder is an enterprise Digital Asset Manager (DAM) with a rich asset browser and metadata support. Connecting Bynder to CloudCannon allows your team members to select from your existing assets and upload new ones from any editing interface that accepts assets.

A Bynder DAM is created on an Organization and connected to each Site. This lets you choose which Sites can interface with Bynder and reduces configuration across branches.

## Getting your Bynder credentials

CloudCannon authenticates with Bynder using an OAuth 2.0 application configured for the client credentials flow. In your Bynder portal, go to your OAuth app settings and create a new application:

* Set the **grant type** to **Client Credentials**, and choose an **Assigned user** — the token inherits that user's permissions, so the user needs access to the assets and brands you want available in CloudCannon.
* Under **Select scopes**, enable **asset:read** and **asset:write** so team members can both browse and upload assets.
* If you plan to use the Bynder widget (Universal Compact View) rather than the CloudCannon asset browser, add `https://cloudcannon.com` to the allowed domains under **Set HTTP access control (CORS)**. The widget runs inside CloudCannon, so if it can't load your assets, a missing CORS entry for `https://cloudcannon.com` is the most likely cause.

Once you save the application, note the following details for CloudCannon:

* **Portal URL** — the domain of your Bynder portal, for example `https://your-company.bynder.com`.
* **Client ID** and **Client Secret** — generated when you create the OAuth application.
* **Brand ID** (optional) — only needed if you want new uploads saved to a specific sub-brand. Leave it blank and CloudCannon saves uploads to your account's primary brand. See [Finding your Brand ID](#finding-your-brand-id) below if you need a specific brand.

### Finding your Brand ID

You only need a Brand ID if you want uploads saved to a specific brand or sub-brand rather than your account's primary brand. Bynder does not display brand IDs in the portal interface, so you need to look them up through the API.

The quickest way is to sign in to Bynder in your browser, then open the brands endpoint for your portal in the same browser — your session authenticates the request:

```
https://your-company.bynder.com/api/v4/brands/
```

The page returns an array of brands, each with an `id` and `name` (and any `subBrands`, each with their own `id` and `name`). Copy the `id` of the brand or sub-brand you want new uploads saved to — this value is your Default Brand ID.

If you would rather use the API directly (for example, to script this), request an access token with your Client ID and Client Secret, then call the same endpoint with a bearer token:

```bash
curl -X POST https://your-company.bynder.com/v6/authentication/oauth2/token \
  -d grant_type=client_credentials \
  -d client_id=YOUR_CLIENT_ID \
  -d client_secret=YOUR_CLIENT_SECRET \
  -d scope="asset:read"

curl https://your-company.bynder.com/api/v4/brands/ \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Authenticating Bynder

First, authenticate Bynder as a DAM in your *Organization Settings / Assets* page. You can then link the authenticated DAM to any Site from *Site Settings / Assets*.

<comp.DocShot docshot_key="CloudCannon-Documentation-Site-Settings-Assets-Page" title="The Site Settings Assets page" alt="The Site Settings Assets page, showing options to link a DAM to your Site." type="screenshot" />

To authenticate a new Bynder DAM, navigate to *Organization Settings / Assets* and click *Authenticate a new DAM*.

Select **Bynder** from the list of DAM providers and enter a **Name** — this is used to identify your DAM in CloudCannon. Then fill out the **Portal URL**, **Client ID**, and **Client Secret** you collected from your Bynder portal. Leave **Default Brand ID** blank to use your account's primary brand, or set it to target a specific sub-brand.

<comp.DocShot docshot_key="CloudCannon-Documentation-Site-Settings-Assets-New-DAM-Modal-Bynder" title="The New DAM modal with Bynder selected" alt="The New DAM modal with Bynder selected as the DAM provider, showing fields for Name, Portal URL, Client ID, Client Secret, and Default Brand ID." type="screenshot" />

Under **Advanced options**, the **Asset browser** option determines how your Bynder assets are displayed and navigated in CloudCannon.

* If you *Use Bynder widget*, the Bynder Universal Compact View opens. This gives you access to the advanced filtering and metadata options that Bynder provides, and your team members sign in through your Bynder portal.
* If you *Use CloudCannon asset browser*, the standard CloudCannon browser opens, populated with your Bynder assets.

Which option should you choose?

* Choose the **Bynder widget** if your team already works in Bynder and wants its full search, filtering, and metaproperty tools. Because the widget signs in through your Bynder portal, each team member needs their own Bynder login, and your portal must allow cross-domain requests from CloudCannon (see the CORS step above).
* Choose the **CloudCannon asset browser** for a simpler, consistent experience that matches every other DAM in CloudCannon. Team members do not need their own Bynder logins — CloudCannon connects using the credentials you configured — which is a good fit when your content authors don't otherwise use Bynder.

Also under **Advanced options**, the **Custom Delivery Hostname** field lets you set a base URL that is prepended to your asset paths — leave it blank to default to your Portal URL. If you later change your Portal URL, this value updates with it (unless you have set a custom hostname). The **Maximum size for uploaded files** option lets you set an upper limit, in bytes, for files uploaded to this DAM.

Once you have filled out all the fields, click **Authenticate** to save and close the modal.

## Link Bynder to your Site

Once authenticated, your new Bynder DAM will be selectable in the DAM dropdown on your *Site Settings / Assets* page. Click **Link DAM** to finish connecting the DAM to your Site.

This new Bynder DAM can be connected to as many Sites as you like. This is useful for different branches or environments of the same Site that need to interface with the same assets.

In your Site settings, you can click the context menu on your linked DAM and select **Settings** to configure extra Site-level options, including the **Uploads locked** checkbox, which prevents uploads to your DAM from within CloudCannon.
