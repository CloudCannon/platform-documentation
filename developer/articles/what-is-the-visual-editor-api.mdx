An API (Application Programming Interface) is a set of methods and rules that allow one piece of software to interact with another using code to call specific functions for reading data, triggering actions, or responding to events, rather than a graphical interface. You can use CloudCannon's public *Visual Editor API* to read and write content, listen for changes, and build custom editing interfaces in the <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm> — all from code running inside your site.

The *Visual Editor API* is designed for developers who want to build their own custom integrations that go beyond what CloudCannon provides out of the box.

## How the Visual Editor works

When you open a page in the *Visual Editor*, CloudCannon loads your site inside a sandboxed iframe. The *Visual Editor API* bridges communication between your site's JavaScript and the editor application that surrounds it.

CloudCannon exposes an API router on `window.CloudCannonAPI`. You request the version you want with `useVersion()`, and the router returns an API object you can use to interact with the editor.

The API uses a `postMessage`-based protocol under the hood, but you don't need to worry about that — the API object handles all communication for you.

## How the API relates to other Visual Editing features

CloudCannon provides several ways to enable visual editing, each suited to different needs:

- *Editable Regions* use HTML attributes like `data-editable` to mark elements as inline-editable. They are declarative and require no JavaScript.
- *Data Bindings* use attributes like `data-cms-bind` to connect front matter fields to elements on the page. They are also declarative.
- The *Visual Editor API* is imperative and programmatic. It gives you full control over how your site interacts with the editor, and is the foundation that powers the other features.

These approaches are not mutually exclusive. *Editable Regions* and *Data Bindings* cover common use cases without code. The *Visual Editor API* is for when you need to build something custom.

## CloudCannon's own features use this API

CloudCannon's *Editable Regions* are built on the same *Visual Editor API* that you have access to. For example, `createTextEditableRegion()` is the method that powers the `data-editable` attribute — it makes any HTML element inline-editable and connects it to the editor's content model.

This means the API is not an afterthought or a secondary interface. It is the same mechanism CloudCannon uses internally to deliver its own editing features.

## What you can build

The *Visual Editor API* provides methods for:

- **Reading and writing data** — access front matter, body content, and raw file source for any file in your <comp.GlossaryTerm term="/user/glossary/s/site.yml">Site</comp.GlossaryTerm>.
- **Listening for changes** — subscribe to `change` and `delete` events on files, <comp.GlossaryTerm term="/user/glossary/c/collection.yml">Collections</comp.GlossaryTerm>, and *Datasets* to keep your interface in sync.
- **Working with Collections and Datasets** — enumerate files in a *Collection* or *Dataset* and react to changes across them.
- **Creating custom editing interfaces** — make any element inline-editable, open custom data panels with CloudCannon's full input support, and control the editor's loading state.
- **File operations** — upload files, claim and release file locks, and access prefetched files.

## Real-world example: the Rosey CloudCannon Connector

The <a href="https://github.com/CloudCannon/rosey-cloudcannon-connector" target="_blank" rel="noopener">Rosey CloudCannon Connector</a> is a custom integration built entirely with the *Visual Editor API*. It adds a locale switcher, inline translation editing, and stale translation detection to sites using <comp.GlossaryTerm term="/user/glossary/r/rosey.yml">Rosey</comp.GlossaryTerm> for <comp.GlossaryTerm term="/user/glossary/i/internationalization.yml">internationalization</comp.GlossaryTerm> — all without requiring changes to CloudCannon itself.

For more about the Connector, see [What is the Rosey CloudCannon Connector?](/documentation/developer-articles/what-is-the-rosey-cloudcannon-connector/).

## Who the API is for

The *Visual Editor API* is designed for experienced CloudCannon users and power-user <comp.GlossaryTerm term="/user/glossary/p/partner.yml">Partners</comp.GlossaryTerm> who are already invested in the platform and want to build custom integrations. It is not aimed at new users or those setting up CloudCannon for the first time.

To get started, see [Use the Visual Editor API](/documentation/developer-articles/use-the-visual-editor-api/).
