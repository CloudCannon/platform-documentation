An access key lets the CloudCannon CLI or an SDK-based program authenticate as you. For more information about what access keys are and how they work, read [What are access keys?](/documentation/developer-articles/what-are-access-keys/).

<comp.Notice info_type="important">
  An access key inherits your account's full permissions, and CloudCannon shows your *Access Key Secret* only once. Store the secret somewhere safe, such as a password manager, and revoke the key if it is ever exposed.
</comp.Notice>

There are two ways to create an access key. You can create one yourself in *Account Settings* and supply it to your program, or, if you are setting up the CloudCannon CLI, you can let the CLI create and store one for you through your browser.

## Create an access key in Account Settings

To create an access key:

1. Log in to CloudCannon.
2. Navigate to the *Access Keys* page under *Account Settings*.
3. Click the *Create Access Key* button. CloudCannon will open the *Create Access Key* modal.
4. Enter a name for your access key in the *Name* field. We recommend choosing a name that indicates which program will use the key.
5. Click the *Create* button.
6. Copy your *Access Key ID* and *Access Key Secret* using the *Copy to Clipboard* button beside each value, then store them somewhere safe.
7. Close the modal.

You have created an access key. Because CloudCannon shows the *Access Key Secret* only once, you cannot retrieve it after you close the modal. If you lose the secret, revoke the key and create a new one.

Your new key appears in the *Active access keys* list, where CloudCannon shows its name, its Access Key ID, and the date you created it.

## Log in with the CloudCannon CLI

If you are setting up the CloudCannon CLI, you do not need to create an access key by hand. The CLI can create one for you through your browser and configure itself automatically.

To log in with the CloudCannon CLI:

1. Run the login command in the CloudCannon CLI. The CLI will open the *CloudCannon CLI Login* page in your browser.
2. Log in to CloudCannon if you are not already logged in.
3. On the *CloudCannon CLI Login* page, click the *Copy to Clipboard* button to copy your login code. You can click the visibility button to reveal the code before you copy it.
4. Paste the login code into your terminal.

The CLI stores your credentials and can now act on your behalf. CloudCannon adds a matching access key named *CloudCannon CLI* to your *Active access keys* list, which you can revoke at any time.

Each time you log in with the CLI, CloudCannon creates a new access key. If you log in repeatedly, you will see several *CloudCannon CLI* entries in your *Active access keys* list, and you can revoke any you no longer need.

## Revoke an access key

When a program no longer needs access, or when you suspect a secret has been exposed, you can revoke the key. CloudCannon will reject any request signed with a revoked key.

To revoke an access key:

1. Navigate to the *Access Keys* page under *Account Settings*.
2. Find the key you want to remove in the *Active access keys* list.
3. Open the key's actions menu, then select *Revoke*.

CloudCannon will move the key to the *Revoked access keys* list and reject any further requests signed with it. CloudCannon deletes revoked keys permanently 30 days after revocation.
