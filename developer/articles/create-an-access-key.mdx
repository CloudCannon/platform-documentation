An *Access Key* lets the CloudCannon CLI or an SDK-based program authenticate as you. For more information about what *Access Keys* are and how they work, please read our documentation on [what Access Keys are](/documentation/developer-articles/what-are-access-keys/).

<comp.Notice info_type="important">
  An *Access Key* inherits your account's full permissions, and CloudCannon shows your *Access Key Secret* only once. Store the secret somewhere safe, such as a password manager, and revoke the key if it is ever exposed.
</comp.Notice>

There are two ways to create an *Access Key*. You can create one yourself in *Account Settings* and supply it to your program, or, if you are setting up the CloudCannon CLI, you can let the CLI create and store one for you through your browser.

## Create an *Access Key* through your *Account Settings*

To create an *Access Key* in app:

1. Click on your *Avatar* at the bottom of the *App Sidebar* and select *Account Settings* from the *Account Menu*.
2. Navigate to the *Access Keys* page.
3. Click the *Create Access Key* button. CloudCannon will open the *Create Access Key* modal.
4. Enter a name for your *Access Key* in the *Name* field. We recommend choosing a name that indicates which program will use the key.
5. Click the *Create* button.
6. Copy your *Access Key ID* and *Access Key Secret* using the *Copy to Clipboard* button beside each value, then store them somewhere safe.
7. Close the modal.

The *Access Key* is now ready for use. Your new key appears in the *Active Access Keys* list, where CloudCannon shows its name, its Access Key ID, and the date you created it.

Because CloudCannon shows the *Access Key Secret* only once, you cannot retrieve it after you close the *Create Access Key* modal. If you lose the secret, revoke the key and create a new one.

## Log in with the CloudCannon CLI

If you are setting up the CloudCannon CLI, you do not need to create an *Access Key* through the app. The CLI can create one for you through your browser and configure itself automatically.

These instructions assume you already have the CloudCannon CLI installed.

To create an *Access Key* with the CloudCannon CLI:

1. Run the `cloudcannon login` command in your terminal. The CLI will open the *CloudCannon CLI Login* page in your browser.
2. Log in to CloudCannon if you are not already logged in.
3. On the *CloudCannon CLI Login* page, click the *Copy to Clipboard* button to copy your login code. You can click the visibility button to reveal the code before you copy it.
4. Paste the login code into your terminal.

The CLI stores your credentials and can now act on your behalf. CloudCannon adds a matching *Access Key* named *CloudCannon CLI* to your *Active Access Keys* list, which you can revoke at any time.

Each time you log in with the CLI, CloudCannon creates a new *Access Key*. In general, you will only need to login once per machine. However, if you log in repeatedly, you will see several *CloudCannon CLI* entries in your *Active Access Keys* list, and you can revoke any you no longer need.

## Revoke an *Access Key*

When a program no longer needs access, or when you suspect a the *Access Key Secret* has been exposed, you can revoke the key. CloudCannon will reject any request signed with a revoked key.

To revoke an *Access Key*:

1. Click on your Avatar at the bottom of the App Sidebar and select Account Settings from the Account Menu.

   Navigate to the Access Keys page.

2. Identify the key you want to remove in the *Active Access Keys* list and click the *Context Menu*.
3. Select the *Revoke* option from the *Context Menu*.

CloudCannon will revoke the *Access Key* immediately. Revoked keys appear in the *Revoked Access Keys* list and reject any further requests signed with it. CloudCannon deletes revoked keys permanently 30 days after revocation.
