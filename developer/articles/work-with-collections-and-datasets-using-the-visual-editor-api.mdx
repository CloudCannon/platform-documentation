<comp.Notice info_type="important">
  The *Visual Editor API* is designed for advanced CloudCannon users. If you need help with custom integrations or your CloudCannon setup in general, please contact our friendly [support team](/support/).
</comp.Notice>

CloudCannon's *Visual Editor API* allows you to read and write content, listen for changes, and build custom editing interfaces in the <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm>. This article covers how to work with <comp.GlossaryTerm term="/user/glossary/c/collection.yml">Collections</comp.GlossaryTerm> and *Datasets* using the *Visual Editor API*.

This article assumes you know how to [initialize the Visual Editor API](/documentation/developer-articles/initialize-the-visual-editor-api/) and [access a Collection or Dataset](/documentation/developer-articles/access-files-collections-and-datasets-with-the-visual-editor-api/).

## List files

You can programatically list all file in a *COllection* or *Dataset

You can call `*.items()` on a *Collection* to return an array of file objects.

<comp.CodeBlock language="javascript" source="script.js">
```
const posts = api.collection('posts'); /*1*/
const files = await posts.items(); /*2*/

for (const file of files) { /*3*/
  const data = await file.data.get();
  console.log(file.path, data?.title);
}
```
<comp.Annotation number="1">
  Create a `posts` variable to access the `posts` *Collection*.
</comp.Annotation>
<comp.Annotation number="2">
  Call `items()` to get an array of file objects in the *Collection*.
</comp.Annotation>
<comp.Annotation number="3">
  Iterate over the files. Each file object has the same interface as `currentFile()` — you can call `data.get()`, `data.set()`, `content.get()`, and all other file methods on it.
</comp.Annotation>
</comp.CodeBlock>

## List files in a *Dataset*

Calling `items()` on a dataset object returns either a single file object or an array, depending on how the *Dataset* is configured.

<comp.CodeBlock language="javascript" source="script.js">
```
const locales = api.dataset('locales'); /*1*/
const items = await locales.items(); /*2*/
```
<comp.Annotation number="1">
  Create a `locales` variable to access the `locales` *Dataset*.
</comp.Annotation>
<comp.Annotation number="2">
  Call `items()` to get the files in the *Dataset*.
</comp.Annotation>
</comp.CodeBlock>

## List all files and Collections

The API provides methods to enumerate all files and *Collections* available in your *Site*.

<comp.CodeBlock language="javascript" source="script.js">
```
const allFiles = await api.files(); /*1*/
const allCollections = await api.collections(); /*2*/

for (const collection of allCollections) { /*3*/
  console.log(collection.collectionKey);
}
```
<comp.Annotation number="1">
  Call `api.files()` to get all files in your *Site* as an array.
</comp.Annotation>
<comp.Annotation number="2">
  Call `api.collections()` to get all *Collections* in your *Site* as an array.
</comp.Annotation>
<comp.Annotation number="3">
  Each collection object has a `collectionKey` property matching the key in your CloudCannon configuration.
</comp.Annotation>
</comp.CodeBlock>


## Filter Collection files

Since `items()` returns an array of file objects, you can filter them using standard JavaScript.

<comp.CodeBlock language="javascript" source="script.js">
```
const posts = api.collection('posts'); /*1*/
const files = await posts.items(); /*2*/
const published = []; /*3*/

for (const file of files) {
  const data = await file.data.get();
  if (data?.published) published.push(file); /*4*/
}
```
<comp.Annotation number="1">
  Create a `posts` variable to access the `posts` *Collection*.
</comp.Annotation>
<comp.Annotation number="2">
  Get all files in the *Collection* as an array.
</comp.Annotation>
<comp.Annotation number="3">
  Create an empty array to hold the filtered results.
</comp.Annotation>
<comp.Annotation number="4">
  Read the structured data for each file, and add files with a `published` field set to `true` to the results array.
</comp.Annotation>
</comp.CodeBlock>


## Type guards

The API provides type guard functions to check whether an object is a file, *Collection*, or *Dataset*.

<comp.CodeBlock language="javascript" source="script.js">
```
if (api.isAPIFile(obj)) { /*1*/
  const data = await obj.data.get();
}

if (api.isAPICollection(obj)) { /*2*/
  const items = await obj.items();
}

if (api.isAPIDataset(obj)) { /*3*/
  const items = await obj.items();
}
```
<comp.Annotation number="1">
  `isAPIFile()` returns `true` if `obj` is a file object, allowing you to safely call file methods on it.
</comp.Annotation>
<comp.Annotation number="2">
  `isAPICollection()` returns `true` if `obj` is a collection object.
</comp.Annotation>
<comp.Annotation number="3">
  `isAPIDataset()` returns `true` if `obj` is a dataset object.
</comp.Annotation>
</comp.CodeBlock>

These are useful when working with mixed results or when you need to narrow the type of an object before calling methods on it.
