The *Main Branch* of your <comp.GlossaryTerm term="/user/glossary/p/project.yml">Project</comp.GlossaryTerm> determines which branch new *Sites* are created from and which *Site* represents your *Project* in the *Projects Browser*. You can set the *Main Branch* when you [create a Project](/documentation/developer-articles/create-a-project/), or update it at any time in *Project Settings*. For more information about how the main branch affects your workflow, read [What is a Project?](/documentation/developer-articles/what-is-a-project/).

To set or update the *Main Branch*:

1. Click the *Projects* button in the *App Sidebar*. CloudCannon will open the *Projects Browser*.
2. Select your *Project* by clicking on the *Project Card*.
3. Click the *Project Settings* tab.
4. Open the *Repository* menu.
5. Enter the name of the branch in the *Main Branch* field. This must match the name of a branch in your *Git Repository*.
6. Click the *Update Project* button.

CloudCannon will update your *Project* to use the new *Main Branch*. New branch *Sites* created from this *Project* will branch from this branch by default.

## Allow all branching

By default, team members can only create branch *Sites* from the *Main Branch*. If you need a more flexible workflow, you can enable the *Allow all branching* toggle in the *Repository* menu of *Project Settings*. When this is enabled, team members can create a branch *Site* from any existing *Site* in the *Project*.

For more information about branching workflows, read [What is a Project?](/documentation/developer-articles/what-is-a-project/).
