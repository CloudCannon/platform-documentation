<comp.Notice info_type="important">
  **Which API do you need?**

  Use the JavaScript API v1 for frontend code that runs in CloudCannon's *Visual Editor*.

  If you need backend API access, contact support to request private beta access.
</comp.Notice>

This article shows how to initialize JavaScript API v1 and safely access the active file.

## Prerequisites

Before you begin, this article assumes your code runs on pages that load inside CloudCannon's *Visual Editor*.

## Initialize JavaScript API v1

To initialize JavaScript API v1:

1. Check whether `window.CloudCannonAPI` is already available.
2. If it is available, call `useVersion('v1')`.
3. If it is not available, wait for the `cloudcannon:load` event and then call `useVersion('v1')`.

<comp.CodeBlock language="javascript" source="src/cloudcannon-api.js">
```
export function onCloudCannonApiReady(callback) {
  const setup = (apiRouter) => {
    const api = apiRouter?.useVersion('v1');
    if (api) callback(api);
  };

  if (window.CloudCannonAPI) {
    setup(window.CloudCannonAPI);
    return;
  }

  document.addEventListener(
    'cloudcannon:load',
    (event) => {
      setup(event.detail?.CloudCannonAPI);
    },
    { once: true }
  );
}
```
</comp.CodeBlock>

## Access the current file

After initialization, you can access the active file in the *Visual Editor*.

<comp.CodeBlock language="javascript" source="src/init.js">
```
import { onCloudCannonApiReady } from './cloudcannon-api';

onCloudCannonApiReady(async (api) => {
  try {
    const file = api.currentFile();
    const data = await file.data.get();
    console.log('Current file path:', file.path);
    console.log('Current file data:', data);
  } catch (error) {
    console.warn('No active file is available for this page.', error);
  }
});
```
</comp.CodeBlock>
