</comp.Notice>

CloudCannon's *Visual Editor API* exposes `createTextEditableRegion()`, which makes any supported HTML element directly editable in the <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm> page preview. This article covers how to create, configure, and update custom inline editable regions.

This article assumes you know how to [initialize the Visual Editor API](/documentation/developer-articles/initialize-the-visual-editor-api/) and [access a Collection or Dataset](/documentation/developer-articles/access-files-collections-and-datasets-with-the-visual-editor-api/). For creating a floating *Data Panel*, read [Create a custom data panel with the Visual Editor API](/documentation/developer-articles/create-a-custom-data-panel-with-the-visual-editor-api/).

## Custom inline editing

Custom inline editing lets users change content directly in the page preview. Clicking an element opens a rich text toolbar and makes the element's content editable in place, without opening a separate panel. This is useful for building a live preview experience, where you can update content in context with your webpage design.

<comp.Notice info_type="info">
  Regions created with `createTextEditableRegion()` use the same underlying mechanism as CloudCannon's `data-editable` attribute, but give you programmatic control. For example, you can register elements dynamically, pass a custom callback, or configure the toolbar options at runtime.
  
  For more information about *Text Editable Regions*, please read our documentation on [Text Editable Regions](/documentation/developer-articles/configure-text-editable-regions/).
</comp.Notice>

### Create a text editable region

Calling `createTextEditableRegion()` allows you to make a supported HTML element inline-editable. This function takes three arguments: your HTML element, a callback which fires whenever the content changes and receives the updated content as a string, and an optional configuration object.

<comp.CodeBlock language="javascript" source="script.js">
```
const element = document.querySelector('#hero-heading'); /*1*/

const region = await api.createTextEditableRegion( /*2*/
  element,
  (content) => { /*3*/
    console.log('Content updated:', content);
  }
);
```
<comp.Annotation number="1">
  Select the HTML element you want to make inline-editable.
</comp.Annotation>
<comp.Annotation number="2">
  Call `createTextEditableRegion()` to register the element as an editable region. It returns a region object you can use to update the content programmatically.
</comp.Annotation>
<comp.Annotation number="3">
  Pass a callback as the second argument. The callback fires whenever the content changes and receives the updated content as a string.
</comp.Annotation>
</comp.CodeBlock>

### Configure a text editable region

Passing an options object as the third argument to `createTextEditableRegion()` customizes how CloudCannon treats the region.

`elementType` tells CloudCannon which family of HTML tags you are editing (`text`, `block`, `span`, `image`, or `link`). That choice affects toolbar behavior and validation. If you omit it, CloudCannon guesses from the tag name (for example, `h2` as `text`, `div` as `block`).

`editableType` accepts the same values as `elementType` (`text`, `block`, `span`, `image`, or `link`), or `'content'`. As a rule of thumb, set `editableType` to the same value as `elementType`, unless you are editing body content, in which case set it to `'content'`.

`extension` tells CloudCannon which file extension to use when persisting the edited fragment (for example, `'.html'` with `'html'`, or `'.md'` with `'markdown'`). Match it to the format of the underlying file so saves serialize the way your *Site* expects.

You can also pass `inputConfig` to control which rich text toolbar options are available.

The following shows common HTML elements and their typical `elementType`. This is not a strict mapping — for example, a `p` element could be configured as `span` if you only want to allow inline content:

* `text` — `p`, `h1`–`h6`, `blockquote`, `li`, `dd`, `dt`
* `block` — `div`, `section`, `article`, `aside`, `main`, `footer`, `header`, `nav`
* `span` — `span`, `small`, `strong`, `em`, `i`, `b`, `sub`, `sup`, `td`, `th`, `cite`
* `image` — `img`
* `link` — `a`

If omitted, CloudCannon infers `elementType` from the element's tag name — for example, an `h2` is inferred as `text`, a `div` as `block`, and a `span` as `span`.

<comp.CodeBlock language="javascript" source="script.js">
```
const region = await api.createTextEditableRegion( /*1*/
  element,
  (content) => { /* handle update */ },
  { /*2*/
    elementType: 'text', /*3*/
    editableType: 'html', /*4*/
    extension: '.html' /*5*/
  }
);
```
<comp.Annotation number="1">
  Pass the same element and callback as before.
</comp.Annotation>
<comp.Annotation number="2">
  Pass an options object as the third argument to customize the editable region. All options are optional.
</comp.Annotation>
<comp.Annotation number="3">
  `elementType` sets the element type, which controls the editing behavior. If omitted, CloudCannon infers it from the element's tag name.
</comp.Annotation>
<comp.Annotation number="4">
  `editableType` controls the editing mode. Set it to the same value as `elementType`, or `'content'` when editing body content.
</comp.Annotation>
<comp.Annotation number="5">
  `extension` sets the file extension used when saving the content.
</comp.Annotation>
</comp.CodeBlock>

The following lists all options accepted by `createTextEditableRegion()`:

<comp.DataReference>
  <comp.DataReferenceRow label="elementType" type_markdown="String">
    The family of HTML element being edited (`text`, `block`, `span`, `image`, or `link`). CloudCannon infers this from the tag name if omitted.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="editableType" type_markdown="String">
    The editing mode. Match to `elementType`, or use `'content'` for body content.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="extension" type_markdown="String">
    The file extension used when saving the edited fragment (e.g. `'.html'` or `'.md'`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="inputConfig" type_markdown="Object">
    Controls which rich text toolbar options are available.
  </comp.DataReferenceRow>
</comp.DataReference>

### Update an inline editable region

Calling `setContent()` on the region object returned by `createTextEditableRegion()` allows you to update the content of the editable region programmatically.

<comp.CodeBlock language="javascript" source="script.js">
```
region.setContent('<p>New content</p>'); /*1*/
```
<comp.Annotation number="1">
  Pass a string to `setContent()` to replace the content of the editable region.
</comp.Annotation>
</comp.CodeBlock>
