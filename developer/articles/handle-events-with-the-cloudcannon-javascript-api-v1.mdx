JavaScript API v1 includes event listeners so you can keep your UI synchronized with changes in the *Visual Editor*.

## Prerequisites

Before you begin, this article assumes you already initialize v1 as shown in [Get started with the CloudCannon JavaScript API v1](/documentation/developer-articles/get-started-with-the-cloudcannon-javascript-api-v1/).

## Listen at API level

Use API-level listeners to react to broad site changes.

<comp.CodeBlock language="javascript" source="src/events-api.js">
```
const refresh = async () => {
  const files = await api.files();
  console.log('Files changed. Current count:', files.length);
};

api.addEventListener('change', refresh);
api.addEventListener('delete', refresh);
```
</comp.CodeBlock>

## Listen at file level

Use file-level listeners for targeted updates to the active file.

<comp.CodeBlock language="javascript" source="src/events-file.js">
```
const file = api.currentFile();

const onDataChange = async () => {
  const latest = await file.data.get();
  renderPreview(latest);
};

file.data.addEventListener('change', onDataChange);
```
</comp.CodeBlock>

## Clean up listeners

Always remove listeners when your component or page section unmounts.

<comp.CodeBlock language="javascript" source="src/events-cleanup.js">
```
const file = api.currentFile();

const onChange = () => {
  console.log('File changed');
};

file.addEventListener('change', onChange);

function cleanup() {
  file.removeEventListener('change', onChange);
}
```
</comp.CodeBlock>
