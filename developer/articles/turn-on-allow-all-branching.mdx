<comp.Notice info_type="pricing">
  ***Allow all branching* is available on our [Team or Enterprise plan](/pricing/).**

  Want to chat about whether this feature is right for you? Our support team is always <a href="mailto:success@cloudcannon.com?subject=A%20question%20about%20branching" rel="noreferrer">happy to hear from you</a>.
</comp.Notice>

<comp.Notice info_type="permissions">
  **Permissions required**

  Members of the Owners and Developers [Default Permission Groups](/documentation/user-articles/what-are-default-permission-groups/), or [Custom Permission Groups](/documentation/developer-articles/what-are-custom-permission-groups/) with the `project:settings:git:write` permission, can turn *Allow all branching* on or off.
</comp.Notice>

By default, *Team Members* can only create branched *Sites* from your <comp.GlossaryTerm term="/user/glossary/p/project.yml">Project's</comp.GlossaryTerm> <comp.GlossaryTerm term="/user/glossary/m/main-branch.yml">Main Branch</comp.GlossaryTerm>. With *Allow all branching* turned on, *Team Members* can branch from any *Site* in your *Project*.

This is useful when *Team Members* need to build on each other's in-progress work. For more information, please read our documentation on [what branching is](/documentation/user-articles/what-is-branching/).

These instructions assume you have [created a Project](/documentation/developer-articles/create-a-project/).

To turn on *Allow all branching*:

1. Click *Projects* in the *App Sidebar*. CloudCannon will open the *Projects Browser*.
2. Select the *Project* you want to configure. CloudCannon will open the *Project Sites Browser*.
3. Click the *Project Settings* tab.
4. Select *Repository* from the list of sections.
5. Switch the *Allow all branching* toggle on.
6. Click the *Update Project* button.

CloudCannon will save the setting, and *Team Members* will be able to branch from any *Site* in your *Project*.

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Settings-Repository" alt="A screenshot of the Repository section in Project Settings, showing the Git Provider, Repository, Main Branch, and Allow all branching toggle." title="The Repository section in Project Settings" type="screenshot" />

You can use the same method to turn *Allow all branching* off, which limits *Team Members* to branching from your *Main Branch*. Existing branched *Sites* are not affected.
