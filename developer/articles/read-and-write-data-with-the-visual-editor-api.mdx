<comp.Notice info_type="important">
  The *Visual Editor API* is designed for advanced CloudCannon users. If you need help with custom integrations or your CloudCannon setup in general, please contact our friendly [support team](/support/).
</comp.Notice>

CloudCannon's *Visual Editor API* allows you to read and write content, listen for changes, and build custom editing interfaces in the <comp.GlossaryTerm term="/user/glossary/v/visual-editor.yml">Visual Editor</comp.GlossaryTerm>. This article covers how to read and write file data using the *Visual Editor API*.

This article assumes you know how to [initialize the Visual Editor API](/documentation/developer-articles/initialize-the-visual-editor-api/) and [access a file](/documentation/developer-articles/access-files-collections-and-datasets-with-the-visual-editor-api/).

## Read and write file data

You can read and write to your website files with the *Visual Editor API* using the following functions:

- Use `*.data.get()` and `*.data.set()` for structured data in the <comp.GlossaryTerm term="/user/glossary/f/front-matter.yml">Front Matter</comp.GlossaryTerm>.
- Use `*.content.get()` and `*.content.set()` for file body content.
- Use `*.get()` and `*.set()` for raw source.
- Use `*.metadata()` for file metadata (you cannot write to file metadata).

### Structured data in Front Matter

Calling `*.data.get()` allows you to read the accessed file, and the *Visual Editor API* returns a JavaScript object. 

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
const frontmatter = await file.data.get(); /*2*/
console.log(frontmatter.title); /*3*/
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Create a `frontmatter` variable to read the structured data in `file`.
</comp.Annotation>
<comp.Annotation number="3">
  Log the value of the structured data item `title` by reading the structured data in `file`.
</comp.Annotation>
</comp.CodeBlock>

You can also read a specific structured data field by passing a `slug` option. When you supply `{ slug: 'author' }`, CloudCannon returns only that field's value instead of the entire object.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile();
const author = await file.data.get({ slug: 'author' }); /*1*/
console.log(author);
```
<comp.Annotation number="1">
  Pass the `{ slug: 'author'}` option to `file.data.get()` to read the specific structured data field `author`.
</comp.Annotation>
</comp.CodeBlock>

Calling `*.data.set()` allows you to write to a structured data field in an accessed file by passing the field's `slug` and the new `value`. This updates the *Visual Editor* data model and marks the file as having unsaved changes — the change will be visible in any open *Editing Interfaces* (not just the *Visual Editor*).

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
await file.data.set({ /*2*/
  slug: 'title', /*3*/
  value: 'Updated title' /*4*/
});
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Call `file.data.set()` to write to a structured data field in `file`.
</comp.Annotation>
<comp.Annotation number="3">
  Pass the `{ slug: 'title'}` option to specify the structured data field to update.
</comp.Annotation>
<comp.Annotation number="4">
  Pass the `{ value: 'Updated title'}` option with the new value for the field.
</comp.Annotation>
</comp.CodeBlock>

### Body content

Body content is the part of a file that comes after the front matter (for example, the Markdown body of a blog post).

Calling `*.content.get()` allows you to read the body content of an accessed file as a string.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
const content = await file.content.get(); /*2*/
console.log(content); /*3*/
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Create a `content` variable to read the body content of `file` as a string.
</comp.Annotation>
<comp.Annotation number="3">
  Log the body content of `file`.
</comp.Annotation>
</comp.CodeBlock>

Calling `*.content.set()` allows you to replace the body content of an accessed file by passing a string. This updates the *Visual Editor* data model and marks the file as having unsaved changes — the change will be visible in any open *Editing Interfaces*.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
const content = await file.content.get(); /*2*/
await file.content.set(`${content}\n\nAppended paragraph.`); /*3*/
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Read the current body content of `file` as a string.
</comp.Annotation>
<comp.Annotation number="3">
  Call `file.content.set()` to replace the body content of `file`, appending a new paragraph to the existing content (e.g., if content was "Hello world", the result passed to `set()` would be "Hello world\n\nAppended paragraph").
</comp.Annotation>
</comp.CodeBlock>

### Raw source

Calling `*.get()` allows you to read the entire raw source of an accessed file as a string, including both front matter and body content. Use this when you need access to the complete file rather than structured data or body content separately.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
const raw = await file.get(); /*2*/
console.log(raw); /*3*/
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Create a `raw` variable to read the entire raw source of `file` as a string.
</comp.Annotation>
<comp.Annotation number="3">
  Log the raw source of `file`.
</comp.Annotation>
</comp.CodeBlock>

Calling `*.set()` allows you to replace the entire raw source of an accessed file by passing a string. This is particularly useful for programmatically updating a specific file — such as a shared data file — regardless of which page is currently open in the *Visual Editor*. This updates the *Visual Editor* data model and marks the file as having unsaved changes — the change will be visible in any open *Editing Interfaces*.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.file('_data/navigation.yml'); /*1*/
const raw = await file.get(); /*2*/
await file.set(raw.replace('old text', 'new text')); /*3*/
```
<comp.Annotation number="1">
  Use `api.file()` with a source path to access a specific file, regardless of which page is currently open in the *Visual Editor*.
</comp.Annotation>
<comp.Annotation number="2">
  Read the entire raw source of `file` as a string.
</comp.Annotation>
<comp.Annotation number="3">
  Call `file.set()` to replace the raw source of `file` with the updated string.
</comp.Annotation>
</comp.CodeBlock>

### File metadata

Calling `*.metadata()` allows you to read metadata about an accessed file, such as its path and collection. File metadata is read-only — you cannot write to it using the *Visual Editor API*.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.file('content/pages/about.md'); /*1*/
const meta = await file.metadata(); /*2*/
console.log(meta); /*3*/
```
<comp.Annotation number="1">
  Use `api.file()` with a source path to access a specific file. You can also use `api.currentFile()` to read the metadata of the file currently open in the *Visual Editor*.
</comp.Annotation>
<comp.Annotation number="2">
  Create a `meta` variable to read the metadata of `file`.
</comp.Annotation>
<comp.Annotation number="3">
  Log the metadata of `file`.
</comp.Annotation>
</comp.CodeBlock>

## Open the *Visual Editor* *Data Panel*

You can use *Data Panels* in CloudCannon's *Visual Editor* to edit the value of structured data fields in the *Front Matter*.

<comp.DocShot docshot_key="CloudCannon-Documentation-Visual-Editor-Bookshop-Sendit-Home-Data-Panel" alt="A screenshot of the Visual Editor displays a webpage preview with a data panel containing inputs open over top." title="The Visual Editor with data panel" type="screenshot"/>

By calling `*.data.edit()` and passing a `slug` option, you can open a *Data Panel* for a specific field. This is useful when you want to use CloudCannon's built-in interface for your custom integration.

Optionally, pass a position object to anchor the *Data Panel* to the HTML element that triggered the action (for example, the element the user clicked).

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
file.data.edit({ /*2*/
  slug: 'hero_image', /*3*/
  position: { /*4*/
    x: event.clientX,
    y: event.clientY,
    left: rect.left,
    width: rect.width,
    top: rect.top,
    height: rect.height
  }
});
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Call `file.data.edit()` to open the editing panel for a field in `file`.
</comp.Annotation>
<comp.Annotation number="3">
  Pass the `{ slug: 'hero_image'}` option to specify which field to open in the *Data Panel*.
</comp.Annotation>
<comp.Annotation number="4">
  Optional. Pass a `position` object to tell CloudCannon where to anchor the editing panel relative to the element.
</comp.Annotation>
</comp.CodeBlock>

## Edit Array Inputs

You can use the *Visual Editor API* to add, remove, and reorder items in an [Array Input](/documentation/user-articles/what-is-an-array-input/) within your *Front Matter*.

### Add an item

Calling `*.data.addArrayItem()` allows you to add an item to an array field in an accessed file.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
await file.data.addArrayItem({ /*2*/
  slug: 'items', /*3*/
  index: null, /*4*/
  value: { title: 'New item', description: '' } /*5*/
});
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Call `file.data.addArrayItem()` to add an item to an array field in `file`.
</comp.Annotation>
<comp.Annotation number="3">
  Pass the `{ slug: 'items'}` option to specify the array field to add an item to.
</comp.Annotation>
<comp.Annotation number="4">
  Pass `index: null` to append the new item to the end of the array, or pass a number to insert it at a specific position.
</comp.Annotation>
<comp.Annotation number="5">
  Pass a `value` option with the content of the new item.
</comp.Annotation>
</comp.CodeBlock>

### Remove an item

Calling `*.data.removeArrayItem()` allows you to remove an item from an array field in an accessed file.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
await file.data.removeArrayItem({ /*2*/
  slug: 'items', /*3*/
  index: 2 /*4*/
});
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Call `file.data.removeArrayItem()` to remove an item from an array field in `file`.
</comp.Annotation>
<comp.Annotation number="3">
  Pass the `{ slug: 'items'}` option to specify the array field to remove an item from.
</comp.Annotation>
<comp.Annotation number="4">
  Pass an `index` option to specify the position of the item to remove.
</comp.Annotation>
</comp.CodeBlock>

### Move an item

Calling `*.data.moveArrayItem()` allows you to reorder items within an array field, or move items between different array fields, in an accessed file.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.currentFile(); /*1*/
await file.data.moveArrayItem({ /*2*/
  fromSlug: 'items', /*3*/
  fromIndex: 0, /*4*/
  toIndex: 3 /*5*/
});
```
<comp.Annotation number="1">
  Create a `file` variable to access the file currently open in the *Visual Editor* using `currentFile()`.
</comp.Annotation>
<comp.Annotation number="2">
  Call `file.data.moveArrayItem()` to move an item within or between array fields in `file`.
</comp.Annotation>
<comp.Annotation number="3">
  Pass a `fromSlug` option to specify the array field containing the item to move. Pass a `toSlug` option to move the item to a different array field.
</comp.Annotation>
<comp.Annotation number="4">
  Pass a `fromIndex` option to specify the current position of the item.
</comp.Annotation>
<comp.Annotation number="5">
  Pass a `toIndex` option to specify the new position of the item.
</comp.Annotation>
</comp.CodeBlock>

## Lock editing on a file

File locking prevents other editors from making changes to a file while your custom integration is working with it. Use `api.file()` to access the specific file you want to lock, then call `claimLock()` before making changes and `releaseLock()` when you are done.

<comp.CodeBlock language="javascript" source="script.js">
```
const file = api.file('_data/settings.yml'); /*1*/
const lock = await file.claimLock(); /*2*/

if (!lock.readOnly) { /*3*/
  await file.data.set({ slug: 'status', value: 'in-progress' });
}

await file.releaseLock(); /*4*/
```
<comp.Annotation number="1">
  Use `api.file()` to access the specific file you want to lock. File locking is most useful when your integration needs to modify a shared file that other editors might also be working with.
</comp.Annotation>
<comp.Annotation number="2">
  Call `claimLock()` to request an editing lock on `file`. Returns an object with a `readOnly` property.
</comp.Annotation>
<comp.Annotation number="3">
  Only make changes if the lock was granted. If another editor holds the lock, `readOnly` will be `true`.
</comp.Annotation>
<comp.Annotation number="4">
  Call `releaseLock()` to release the lock when you are done making changes.
</comp.Annotation>
</comp.CodeBlock>
