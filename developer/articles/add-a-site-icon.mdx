<comp.Notice info_type="permissions">
  <p><strong>Permissions required</strong></p>
  <p>Members of all <a href="/documentation/user-articles/what-are-default-permission-groups/">Default Permission Groups</a>, or <a href="/documentation/developer-articles/what-are-custom-permission-groups/">Custom Permission Groups</a> with the <code>site:file:write</code> permission, can add a Site Icon.</p>
</comp.Notice>

A [Site Icon](/documentation/developer-articles/what-is-a-site-icon/) is a PNG image that helps identify your *Site* in the CloudCannon interface. You can add a *Site Icon* from the *Site Header* using <comp.GlossaryTerm term="/user/glossary/c/configuration-mode.yml">Configuration Mode</comp.GlossaryTerm>, or by creating the file directly in your local development environment.

## Add or update your *Site Icon*

To add a *Site Icon* using *Configuration Mode*:

1. Navigate to your *Site*.
2. Click the *Configuration Mode* switch on the right of the *Site Header* to turn it on.
3. Click the edit button on the *Site Icon* placeholder in the *Site Header*. CloudCannon will open a file picker.
4. Select a PNG file from your computer. CloudCannon will add the file to your editing session.
5. Click the *Save* button to commit the changes to your *Git Repository*.

CloudCannon will save your `siteicon.png` file to the root of your repository and display it across the interface.

<comp.DocShot docshot_key="CloudCannon-Documentation-Site-Header-Config-Mode-Site-Icon" alt="A screenshot of the Site Header with Configuration Mode on shows the edit button on the Site Icon placeholder." title="The Site Header in Configuration Mode" type="screenshot"/>

Alternatively, you can also add a *Site Icon* from your local development environment by adding a PNG image named `siteicon.png` at the root of your Git repository.

<comp.Notice info_type="info">
  Your *Site Icon* must be a PNG file named exactly `siteicon.png` at the root of your repository. For image recommendations, please read our documentation on [What is a Site Icon?](/documentation/developer-articles/what-is-a-site-icon/).
</comp.Notice>

If you remove your *Site Icon*, by deleting the `siteicon.png` file from the root of your repository, CloudCannon will revert to the default icon.
