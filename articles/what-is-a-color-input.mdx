A Color input is an editing interface for color information. You can use this input in the Data Editor, Content Editor, or Visual Editor to edit structured data in data files or front matter.

There is only one type of Color input:

* Color

## Color input types

### Color

The Color input provides an editing interface for color values.

You can edit the value of a Color input by typing in the input text field.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/color/2024-07/CloudCannon-Documentation-RGB-Color-Input.png" alt="A screenshot of two Color inputs in the Data Editor, showing RGB and RGBA data." title="The Color input" type="ui-snippet"/>

## Color input appearance and behavior

Depending on how your developer configured the input, your Color input may have color transparency controls.

<comp.DocShot docshot_key="" alt="" title=""/>

Your Color input may also have input validation, which requires the value to match predefined criteria before you can save your changes.

CloudCannon will show a red `*` next to the name of your Color input if a value is required.

<comp.DocShot docshot_key="" alt="" title=""/>

If you enter a value under the minimum or over the maximum character count, CloudCannon will display an error message in red text under the input field.

<comp.DocShot docshot_key="" alt="" title=""/>

If your developer has configured a pattern for your input value, CloudCannon will warn you when the value does not match the pattern. CloudCannon will show any error messages underneath the input field in red.

<comp.DocShot docshot_key="" alt="" title=""/>

For more information, please read our developer documentation on [configuring a Color input](/documentation/articles/configure-a-color-input/) for configuration options, valid input values, and unconfigured input behavior.
