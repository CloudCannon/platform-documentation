Rich text editors are editing interfaces that enable you to create and format content. Rich text editors are WYSIWYG editors, so the content you create will look like it does on your web page. CloudCannon has three kinds of rich text editors: the [Content Editor](/documentation/articles/the-content-editor/), [editable regions](/documentation/articles/define-editable-regions-in-your-html/) in the [Visual Editor](/documentation/articles/the-visual-editor/), and [Rich Text inputs](/documentation/articles/rich-text-inputs/).

This article covers all keys available in CloudCannon for configuring your rich text editors. For more information, please read our documentation on [rich text editors](/documentation/articles/what-are-rich-text-editors/) in general, [configuring rich text editors](/documentation/articles/configure-your-rich-text-editors/), and [best practices for custom markup in rich text](/documentation/articles/best-practices-for-rich-text/).

<comp.MultiCodeBlock language="YAML" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
``````````

_editables:
  text:
    italic: true
  block:
    format: p h3
    undo: true
    redo: true
  content:
    blockquote: true
    bold: true
    format: p h1 h2 h3 h4 h5 h6 pre address div
    italic: true
    strike: true
    subscript: true
    superscript: true
    underline: true
    link: true
    bulletedlist: true
    indent: true
    numberedlist: true
    outdent: true
    code: true
    embed: true
    horizontalrule: true
    image: true
    table: true
    undo: true
    redo: true
    removeformat: true
    copyformatting: true
    styles: /css/editor.css
    left: align-left
    center: align-center
    right: align-right
    justify: align-justify
    image_size_attributes: true
    snippet: true

``````````
</comp.MultiCodeBlock>

## WYSIWYG toolbar

Not all WYSIWYG tools are available for every rich text editor.

### Text options

The following options are available in the WYSIWYG toolbar for text formatting:

<comp.DataReference>
  <comp.DataReferenceRow label="blockquote" type_markdown="Boolean">
    This key enables a tool to wrap selected blocks of text in a block quote.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="bold" type_markdown="Boolean">
    This key enables a tool to bold selected text.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="format" type_markdown="String">
    This key enables a dropdown menu tool for structured text. Valid values include for `p`, `h1`, `h2`, `h3`, `h4`, `h5`, `h6`, `pre`, `address`, and `div`. Include a space between each values, e.g. `format: p h1 h2`.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="italic" type_markdown="Boolean">
    This key enables a tool to italicize selected text.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="strike" type_markdown="Boolean">
    This key enables a tool to strike through selected text.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="subscript" type_markdown="Boolean">
    This key enables a tool to subscript selected text.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="superscript" type_markdown="Boolean">
    This key enables a tool to superscript selected text.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="underline" type_markdown="Boolean">
    This key enables a tool to underline selected text.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="link" type_markdown="Boolean">
    This key enables a tool to create a hyperlink around selected text.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
</comp.DataReference>

### Style and alignment options

The following options are available in the WYSIWYG toolbar for text style and alignment:

<comp.DataReference>
  <comp.DataReferenceRow label="styles" type_markdown="String">
    This key enables a dropdown menu to apply a style to selected text. Styles are the combination of an element and class name. The value for this option is the path (either source or build output) for the CSS file containing styles.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="left" type_markdown="String">
    This key enables a tool to left align selected text by toggling a class name for that text block. The value is the class name to align the text. The styles for this class must be in the `styles` file to work outside of the Visual Editor.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="center" type_markdown="String">
    This key enables a tool to center align selected text by toggling a class name for that text block. The value is the class name to align the text. The styles for this class must be in the `styles` file to work outside of the Visual Editor.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="right" type_markdown="String">
    This key enables a tool to right align selected text by toggling a class name for that text block. The value is the class name to align the text. The styles for this class must be in the `styles` file to work outside of the Visual Editor.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="justify" type_markdown="String">
    This key enables a tool to justify selected text by toggling a class name for that text block. The value is the class name to align the text. The styles for this class must be in the `styles` file to work outside of the Visual Editor.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
</comp.DataReference>

Add predefined styles in plain CSS for your clients and team members to use in the Visual Editor and Content Editor.

The `styles` CSS file can be either a source or output file. The source CSS file is used if both exist. In either case, it must contain plain CSS.

Selectors must specify an element and one class in order to be included in the styles drop down. Styles with incompatible selectors are applied to the editor, but not shown as options.

The alignment classes for `left`, `right`, `center` and `justify` must be defined in the `styles` CSS file to give the editor access to those styles.

<comp.CodeBlock language="css" source="css/editor.css">
```
/* Can be applied to blocks of content */
p.callout {
  margin: 10px;
  border: 1px solid #f5f5f5;
  background-color: #eee;
}

/* Can be applied to inline content */
span.big-blue-text {
  font-size: 2rem;
  color: blue;
}

/* Applied to content, excluded from style dropdown */
h2 {
  font-family: cursive;
}

/* Applied to content, excluded from style dropdown */
.align-left { text-align: left; }
.align-center { text-align: center; }
.align-right { text-align: right; }
.align-justify { text-align: justify; }
```
</comp.CodeBlock>

<comp.Notice info_type="info">
  Custom styles in the *Visual Editor* require the same styles on your live site. If you don’t have the same styles on your live site, the class will be there but it won’t change the styling.
</comp.Notice>

### Image options

The following options are available in the WYSIWYG toolbar for image formatting:

<comp.DataReference>
  <comp.DataReferenceRow label="image_size_attributes" type_markdown="Boolean">
    Instructs the editor to save `width` and `height` on the image elements added to this editable. This can [prevent pop-in](/documentation/articles/why-cloudcannon-injects-width-and-height-onto-images/) as a page loads. Defaults to `false`.

    <comp.Notice info_type="important">
                  This defaults to `true` for sites created before June 23 2023, in order to keep those sites working. Use the [Image Size Attributes Default](/documentation/articles/activating-flags/#image-size-attributes-default-behaviour) flag to update this for your site, if needed.
                </comp.Notice>

    Available for `_editables.image`, `_editables.block` and `_editables.content`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="width" type_markdown="Integer">
    Defines the width of the bounding box used in the resizing process defined with `resize_style`.

    Available for `_editables.image`, `_editables.block` and `_editables.content`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="height" type_markdown="Integer">
    Defines the height of the bounding box used in the resizing process defined with `resize_style`.

    Available for `_editables.image`, `_editables.block` and `_editables.content`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="resize_style" type_markdown="String">
    Sets how uploaded image files are resized with a bounding box defined by `width` and `height` prior to upload. Defaults to `contain`. Must be `contain`, `cover` or `stretch`\:

    * `cover` keeps aspect ratio and ensures the image covers the bounding box
    * `contain` keeps aspect ratio and ensures the image fits inside the bounding box
    * `stretch` ignores aspect ratio and resizes the image to the bounding box

    Has no effect when selecting existing images, or if `width` and `height` are not set.



    Available for `_editables.image`, `_editables.block` and `_editables.content`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="mime_type" type_markdown="String">
    Sets what format the image is converted to prior to upload. The extension of the file is updated to match. Defaults to keeping the mime type of the uploaded file. Must be one of `image/jpeg` or `image/png`.

    Available for `_editables.image`, `_editables.block` and `_editables.content`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="expandable" type_markdown="Boolean">
    Controls whether or not images can be upscaled to fit the bounding box during resize prior to upload. Defaults to `false`. Has no effect if files are not resized.

    Available for `_editables.image`, `_editables.block` and `_editables.content`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="allowed_sources" type_markdown="Array">
    If you have one or more DAMs connected to your site, you can use this key to list which asset sources can be uploaded to and selected from.

    You can refer to a DAM with the **Reference Key** field from the site-level DAM configuration.

    You can refer to your site files with `site-files`. If this is omitted from the array, site files will not be available as an asset source for this input.
  </comp.DataReferenceRow>
</comp.DataReference>

### Block options

The following options are available in the WYSIWYG toolbar for adding media and full-width content elements:

<comp.DataReference>
  <comp.DataReferenceRow label="code" type_markdown="Boolean">
    This key enables two tools to turn selected text into inline code and selected text into a code block.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="embed" type_markdown="Boolean">
    This key enables a tool to insert a region of raw HTML for Youtube, Vimeo, social media posts, or other media. Embedded content is sanitized to mitigate XSS risks, which includes removing `style` tags. Embeds containing `script` tags are not loaded a rich text editor.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="horizontalrule" type_markdown="Boolean">
    This key enables a tool to insert a horizontal line.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="image" type_markdown="Boolean">
    This key enables a tool to insert an image and enter alternative text and title text. Select an existing image, upload an image, or use an external link.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="table" type_markdown="Boolean">
    This key enables a tool to insert a table. Further table options are available from the table context menu in the rich text editor.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="snippet" type_markdown="Boolean">
    This key enables a tool to insert a [Snippet](/documentation/articles/what-is-a-snippet/), if any are available.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
</comp.DataReference>

### Action options

The following options are available in the WYSIWYG toolbar for user actions when editing and formatting content:

<comp.DataReference>
  <comp.DataReferenceRow label="undo" type_markdown="Boolean">
    This key enables a tool to undo recent edits. Undo is always enabled through standard OS-specific keyboard shortcuts.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="redo" type_markdown="Boolean">
    This key enables a tool to redo recent edits undone with `undo`. Undo is always enabled through standard OS-specific keyboard shortcuts.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="removeformat" type_markdown="Boolean">
    This key enables a tool to remove formatting from selected text. Applies to `bold`, `italic`, `underline`, `strike`, `subscript`, and `superscript` formatting. Does not remove other styles or formatting.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="copyformatting" type_markdown="Boolean">
    This key enables a tool to copy formatting from selected text. Applies to `bold`, `italic`, `underline`, `strike`, `subscript`, and `superscript` formatting. Does not copy other styles or formatting.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
</comp.DataReference>

## Paths

The following options are available for determining where new asset files upload to, the file names for newly uploaded assets, and the default path for selecting existing assets:

<comp.DataReference>
  <comp.DataReferenceRow label="paths" type_markdown="Object">
    This key enables you to define paths for your rich text editors. Each path is relative to global site `source`. If undefined, the paths default to those configured in the [global configuration file](/documentation/articles/create-a-global-configuration-file/). The following nested keys are available:

    * `dam_static`
    * `dam_uploads`
    * `dam_uploads_filename`
    * `static`
    * `uploads`
    * `uploads_filename`

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Image editable region (`_editables.image`), the Link editable region (`_editables.link`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="paths.dam_static" type_markdown="String">
    This key enables you to define the path for the location of statically copied assets for [DAM](/documentation/articles/introduction-to-assets-and-dams/) files.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Image editable region (`_editables.image`), the Link editable region (`_editables.link`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="paths.dam_uploads" type_markdown="String">
    This key enables you to define the path for the default location of newly uploaded [DAM](/documentation/articles/introduction-to-assets-and-dams/) files.

    You can use [dynamic placeholders](/documentation/articles/configure-your-template-strings/) for `uploads` and `dam_uploads`. For more information, please read our documentation on [adjusting file upload paths](/documentation/articles/adjusting-the-uploads-path/).

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Image editable region (`_editables.image`), the Link editable region (`_editables.link`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).

  <comp.DataReferenceRow label="paths.dam_uploads_filename" type_markdown="String">
    This key enables you to define the path for the name of the uploaded file, when uploading [DAM](/documentation/articles/introduction-to-assets-and-dams/) files.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Image editable region (`_editables.image`), the Link editable region (`_editables.link`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="paths.static" type_markdown="String">
    This key enables you to define the path for the location of statically copied assets.

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Image editable region (`_editables.image`), the Link editable region (`_editables.link`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="paths.uploads" type_markdown="String">
    This key enables you to define the paths for the default location of newly uploaded site files.

    You can use [dynamic placeholders](/documentation/articles/configure-your-template-strings/) for `uploads` and `dam_uploads`. For more information, please read our documentation on [adjusting file upload paths](/documentation/articles/adjusting-the-uploads-path/).

    Available for the Content Editor (`_editables.content`), the Block editable region (`_editables.block`), the Image editable region (`_editables.image`), the Link editable region (`_editables.link`), the Text editable region (`_editables.text`), and Rich Text inputs (`_inputs.key_name.options`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="paths.uploads_filename" type_markdown="String">

## Custom markup

More options for handling HTML or Markdown structure that cannot be recreated in the rich text editor.

The following options are available for

<comp.DataReference>
  <comp.DataReferenceRow label="allow_custom_markup" type_markdown="Boolean">
    Enables you to edit custom markup in rich text editors and explicitly accept the risk of unintentionally deleting elements that cannot be recreated. By default, `allow_custom_markup` is `false` for inputs and `true` for editable regions.

    When using `allow_custom_markup` outside of editable regions, content that uses JavaScript will not be editable for security reasons. These inputs will appear as uneditable snippets.

    For more information, please read our documentation on best practices for HTML in [rich text](/documentation/articles/best-practices-for-rich-text/) and [editable regions](/documentation/articles/define-editable-regions-in-your-html/).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="remove_custom_markup" type_markdown="Boolean">
    Enables you to strip custom markup from content edited in a rich text editor, allowing you to clean files of unnecessary HTML content. By default, `remove_custom_markup` is `false` for inputs and editable regions.

    The `remove_custom_markup` key is disabled if `allow_custom_markup` is enabled.

    For more information, please read our documentation on best practices for HTML in [rich text](/documentation/articles/best-practices-for-rich-text/) and [editable regions](/documentation/articles/define-editable-regions-in-your-html/).
  </comp.DataReferenceRow>
</comp.DataReference>
