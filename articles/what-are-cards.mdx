A Card is a square, user interface element used to represent your data in CloudCannon. Cards are clickable elements that preview the data they represent.

CloudCannon uses Cards to represent your data in many places across the app, including:

* Files in your [Collection](/documentation/articles/define-your-collections/) browser (but not the File browser)
* The [Object](/documentation/articles/object-inputs/), [Array](/documentation/articles/array-inputs/), and [Select and Multiselect](/documentation/articles/select-and-multiselect-inputs/) inputs
* [Structures](/documentation/articles/what-is-a-structure/), including the [Structures modal](/documentation/articles/configure-your-structure-previews/)
* [Snippets](/documentation/articles/what-is-a-snippet/), including the Snippets modal

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/cards/2024-08/CloudCannon-Documentation-Collection-Browser-File-Cards.png" alt="A screenshot of the Collection browser showing four cards to represent four files." title="The Collection browser" type="ui-snippet"/>

You can [configure the appearance of your Cards](/documentation/articles/configure-your-card-previews/) to customize a better experience for your team members, and specify a different appearance for each data type (e.g., Collections may preview their Collection items using different Card configurations).

Cards have five main elements: the title, subtitle, metadata, icon, and image.

CloudCannon uses cards to represent your data. A card is a configurable UI element that gives you a preview of that data. Depending on the type of data stored, this could include an image, title, subtitle, icon, and metadata.

<comp.DocsImage path="ye_olde_images/preview-options.svg" alt="The anatomy of a card preview" title="" type="regular"/>

Cards appear in different places across the app, including:

1. Snippets
   * [Hugo Shortcodes](/documentation/articles/snippets-using-hugo-shortcodes/)
   * [Eleventy Shortcodes](/documentation/articles/snippets-using-eleventy-shortcodes/)
2. Snippet Picker
3. [Structures](/documentation/articles/what-is-a-structure/)
4. [Structure Picker](/documentation/articles/create-a-structure/)
5. [Select-style inputs](/documentation/articles/select-and-multiselect-inputs/)
6. [Object inputs](/documentation/articles/object-inputs/)
7. [Collection](/documentation/articles/define-your-collections/) file lists

These cards are configurable using the `preview` key.

<comp.Notice info_type="info">
  Select-style inputs, Object inputs and Structure and non-structure inputs inside Array inputs support a subset of these options. The specifics are documented on each page.
</comp.Notice>
