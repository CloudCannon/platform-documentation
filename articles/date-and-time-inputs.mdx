A Date or Time input lets your team members add date or time values to your data files or the front matter of markup files. Date inputs have a calendar to select date values.

There are three types of Date and Time input:

* Datetime
* Date
* Time

For each inputs, your can configure the timezone and null value. You can also use the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs.

## Datetime

The Datetime input provides an editing interface for date and time values.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/date-and-time/2024-07/CloudCannon-Documentation-Datetime-Input.png" alt="A screenshot of the Datetime input in the Data Editor, showing a calendar for date values." title="The Datetime input" type="ui-snippet"/>

You can configure a Datetime input using the `type` key with the value `datetime`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  published___2___:
    type___3___: datetime

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Datetime input is called `published`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `datetime` input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Datetime input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="blog.md">
```
---
published: 2024-07-16T12:00:00Z
---

Blog content goes here.
```
</comp.CodeBlock>

## Datetime

Combination date picker and 12-hour time input.

Datetime inputs are shown for inputs configured with the type `datetime`, for date input values, or for input keys matching:

* `date`
* `datetime`
* `*_datetime`
* `at`
* `*_at`

## Date

Date picker input for date values with year, month, and day.

Date inputs are shown for inputs configured with the type date, or for inputs keys matching:

* `*_date`

<comp.DocsImage path="ye_olde_images/date-open.png" alt="Screenshot of date input field focused" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```

sale_start_date: 2021-01-09T00:00:00Z

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

due_by: 2021-01-09T00:00:00Z
_inputs:
  due_by:
    type: date

```
</comp.MultiCodeBlock>

<comp.Notice info_type="info">
  The time value is always added for Date inputs as `00:00:00` for consistency with Datetime inputs.
</comp.Notice>

## Time

Single line 12-hour time input for strings of time values.

Time inputs are shown for inputs configured with the type `time`, or for input keys matching:

* `time`
* `*_time`

## Options

Date, Datetime and Time input options are configured with `options` inside an `_inputs` entry.

<comp.DataReference>
  <comp.DataReferenceRow label="timezone" type_markdown="String">
    Specifies the time zone that dates are displayed and edited in. Also changes the suffix the date is persisted to the file with. Must be in the [tz database name](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones) format (e.g. `Pacific/Auckland` or `America/New_York`). Defaults to the `timezone` configured in your [global configuration file](/documentation/articles/create-a-global-configuration-file/), which defaults to `Etc/UTC`.

    Available on Date and Datetime inputs, has no effect on Time inputs.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="empty_type" type_markdown="string">
    Set how an ‘empty’ value will be saved. Does not apply to existing empty values. Can be one of the following:

    * `string` - an empty value for this input will be stored as `""`.
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.
  </comp.DataReferenceRow>
</comp.DataReference>

## Valid values

Documented below are the valid formats for date, datetime, and time inputs.

The lists of examples are non-exhaustive.

<comp.Notice info_type="info">
  * Date, time, and datetime inputs surrounded by quotation marks will be automatically unquoted when loaded into the Data Editor.

  * 24hr times will be converted to 12hr times when loaded into the Data Editor.
</comp.Notice>

#### Date and Datetime

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:

    * `datetime: `

    Empty string:

    * `datetime: ""`
    * `datetime: ''`

    ISO 8601 formatted datetime (quoted or unquoted):

    * `datetime: 2020-07-15T12:00:00Z`
    * `datetime: '2020-07-15T12:00:00Z'`
    * `datetime: "2020-07-15T12:00:00Z"`
  </comp.Tab>
  <comp.Tab name="TOML">
    Empty string:

    * `datetime = ""`
    * `datetime = ''`

    ISO 8601 formatted datetime (quoted or unquoted):

    * `datetime = 2020-07-15T12:00:00Z`
    * `datetime = "2020-07-15T12:00:00Z"`
    * `datetime = '2020-07-15T12:00:00Z'`

    ISO 8601 formatted datetime with space instead of `T` (quoted or unquoted):

    * `datetime = 2020-07-15 12:00:00Z`
    * `datetime = "2020-07-15 12:00:00Z"`

    The space will be converted to a T when loaded into the Data Editor
  </comp.Tab>
  <comp.Tab name="JSON">
    Null value:
    * `"datetime": null`

    Empty string:
    * `"datetime": ""`

    ISO 8601 formatted datetime:
    * `"datetime": "2020-07-15T12:00:00Z"`
  </comp.Tab>
</comp.Tabs>
