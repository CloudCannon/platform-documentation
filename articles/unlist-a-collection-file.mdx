Hide specific files in a collection so your team members don't accidentally edit them. These files are still used in your site build.

This option is set in the front matter of specific files.

<comp.DataReference>
  <comp.DataReferenceRow label="_unlisted" type_markdown="Boolean">
    Hides specific files in the collection list section (still accessible from [Visual Editor](/documentation/articles/the-visual-editor/) and direct links).
  </comp.DataReferenceRow>
</comp.DataReference>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Front Matter">
```

_unlisted: true

```
</comp.MultiCodeBlock>

<comp.Notice info_type="info">
  By default, CloudCannon does not read data files during your build. In order to unlist data files, instruct CloudCannon to read it by [defining your data](/documentation/articles/define-your-data/).
</comp.Notice>
