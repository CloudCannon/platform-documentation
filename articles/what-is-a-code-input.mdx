A Code input is an editing interface for code or mono-spaced plain text. You can use this input in the Data Editor, Content Editor, or Visual Editor to edit structured data in data files or front matter.

There is only one type of Code input:

* Code

## Code input types

### Code

The Code input provides an editing interface for code or mono-spaced plain text content.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/code/2024-07/CloudCannon-Documentation-Code-Input.png" alt="A screenshot of a Code input in the Data Editor, showing ruby code." title="The Code input" type="ui-snippet"/>

You can configure a Code input using the `type` key with the value `code`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  example_ruby___2___:
    type___3___: code

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Code input is called `example_ruby`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `code` input.
</comp.Annotation>
</comp.MultiCodeBlock>

When you add your input key name to a data or markup file, your configured Code input will appear in the Data Editor, Visual Editor, and Content Editor.

<comp.CodeBlock language="YAML" source="data.yml">
```

example_ruby: |
  def say_hello
    puts 'Hi there!'
  end

  say_hello

```
</comp.CodeBlock>

## Input configuration options

General configuration options are available for all input types. You can define the label, comment, and context box for your Boolean input, whether it is hidden or disabled, and how CloudCannon should handle configuration at multiple levels of the configuration cascade.

Specific configuration options for Code inputs include defining the height of the code area, tab size, theme color, gutter visibility, syntax highlighting, and how CloudCannon handles empty values. You can also add input validation to require a value, specify the minimum and maximum value length, or match a regular expression.

Here is an example of a Code input using some of the most commonly used configuration keys.

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs:
  example_ruby:
    type: code
    label: Example Ruby Code
    comment: Enter the example code for the feature
    options:
      max_visible_lines: 25
      min_visible_lines: 15
      show_gutter: false
      syntax: ruby
      theme: darcula
      required: true

```
</comp.MultiCodeBlock>

For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/) in general, or our [inputs reference](/documentation/articles/inputs-reference/) for a complete list of available configuration keys.

## Valid values

Code inputs can have multiple valid values for empty, single-line, and multiline content. Here are some examples of valid values for the key `code`.

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `code: `

    Any valid string (quoted or unquoted):
    * `code: ""`
    * `code: ''`
    * `code: any string`
    * `code: "any string"`
    * `code: 'any string'`

    Any valid multiline string:
    * <code>code: ><br/>  multiline string</code>
    * <code>code: >-<br/>  multiline string</code>
    * <code>code: >+<br/>  multiline string</code>
    * <code>code: |<br/>  multiline string</code>
    * <code>code: |-<br/>  multiline string</code>
    * <code>code: |+<br/>  multiline string</code>
  </comp.Tab>
  <comp.Tab name="TOML">
    Any valid string:
    * `code = ""`
    * `code = "any string"`

    Any valid escaped string:
    * `code = ''`
    * `code = 'any string (literal)'`

    Any valid multiline string:
    * <code>code = """<br/>  multiline string"""</code>
    * <code>code = """\\<br/>  multiline string (trimmed)<br/>  \\"""</code>
    * <code>code = '''<br/>  literal multiline string'''</code>
  </comp.Tab>
  <comp.Tab name="JSON">
    Null value:
    * `"code": null`

    Any valid string:
    * `"code": ""`
    * `"code": "any string"`

    Any valid multiline string:
    * `"code": "multiline \n string"`
  </comp.Tab>
</comp.Tabs>

## Unconfigured Code inputs

In some cases, CloudCannon can still detect a Code input even if you have not configured it.

CloudCannon will interpret any unconfigured input with the key name `code_block`, or that ends in `_code_block` as a Code input.

<comp.CodeBlock language="Markdown" source="about.md">
```

---
code_block: |
  Some content is better in monospace.

  1 + 1 = 2
  2 + 2 = 4
---

```
</comp.CodeBlock>

This behavior is convenient if you have simple inputs or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.

<comp.Notice info_type="info">
  We recommend configuring your inputs for greater control over their functionality and appearance.
</comp.Notice>
