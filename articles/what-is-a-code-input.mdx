A Code input lets your team members write code or mono-spaced plain text in your data files or the front matter of markup files. Code inputs have a code area for your content.

For each input, you can configure the height of the code area, tab size, theme color, gutter visibility, syntax highlighting, and how CloudCannon handles empty values. You can also use the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs.

## Code

The Code input provides an editing interface for code or mono-spaced plain text content.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/code/2024-07/CloudCannon-Documentation-Code-Input.png" alt="A screenshot of a Code input in the Data Editor, showing ruby code." title="The Code input" type="ui-snippet"/>

You can configure a Code input using the `type` key with the value `code`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  example_ruby___2___:
    type___3___: code

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Code input is called `example_ruby`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `code` input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Code input will appear in the Data Editor or the Visual or Content Editor sidebar when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="YAML" source="data.yml">
```

example_ruby: |
  def say_hello
    puts 'Hi there!'
  end

  say_hello

```
</comp.CodeBlock>

## Options

You can configure Code inputs using the `options` key under your input key, inside of `_inputs`.

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs:
  snippet:
    type: code
    options:
      max_visible_lines: 25
      min_visible_lines: 15
      show_gutter: false
      syntax: sh
      tab_size: 4
      theme: solarized_dark
      empty_type: null

```
</comp.MultiCodeBlock>


## Unconfigured Code inputs

In some cases, CloudCannon can still detect a Code input even if you have not configured it.

CloudCannon will interpret any unconfigured input with the key name `code_block`, or that ends in `_code_block` as a Code input.

<comp.CodeBlock language="Markdown" source="about.md">
```

---
code_block: |
  Some content is better in monospace.

  1 + 1 = 2
  2 + 2 = 4
---

```
</comp.CodeBlock>

This behavior is convenient if you have simple inputs or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.

<comp.Notice info_type="info">
  We recommend configuring your inputs for greater control over their functionality and appearance.
</comp.Notice>
