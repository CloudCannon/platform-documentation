A URL input lets your team members add relative, absolute and fully qualified URLs to your data files or the front matter of markup files.

For each input, you can configure how CloudCannon handles null values. You can also use the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs.

## URL

The URL input provides an editing interface for relative, absolute and fully qualified URLs.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/url/2024-07/CloudCannon-Documentation-URL-Input-External.png" alt="" title="The URL input" type="ui-snippet"/>

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/url/2024-07/CloudCannon-Documentation-URL-Input-Internal.png" alt="" title="The URL input" type="ui-snippet"/>

You can configure a URL input using the `type` key with the value `url`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  associate_link___2___:
    type___3___: url

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This URL input is called `associate_link`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `url` input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the URL input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="blog.md">
```
---
associate_link: example.com
---

Content goes here.
```
</comp.CodeBlock>

Previews for URLs without a protocol are requested prefixed with `http://`

## Options

You can configure URL inputs using the `options` key under your input key, inside of `_inputs`.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  background:
    type: url
    options:
      empty_type: null

```
</comp.MultiCodeBlock>

URL inputs have the following options available:

<comp.DataReference>
  <comp.DataReferenceRow label="empty_type" type_markdown="string">
    This key determines how CloudCannon handles an empty value. This key does not apply to existing empty values.

    Value must be one of the following:

    * `string` - an empty value for this input will be stored as "".
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.
  </comp.DataReferenceRow>
</comp.DataReference>

## Unconfigured URL inputs

In some cases, CloudCannon can still detect a URL input even if you have not configured it.

CloudCannon will interpret any unconfigured input with the key name `url` or `link`, or that ends in `_url` or `_link`, as a URL input.

<comp.CodeBlock language="YAML" source="data.yml">
``````````
external_link: https://cloudcannon.com/
``````````
</comp.CodeBlock>

This behavior is convenient if you have simple inputs, or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.

<comp.Notice info_type="info">
  We recommend configuring your inputs for greater control over their functionality and appearance.
</comp.Notice>

## Valid values

Here are some examples of valid values for the key `url`.

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `url: `

    Any valid string (quoted or unquoted):
    * `url: ""`
    * `url: ''`
    * `url: example.com`
    * `url: "example.com"`
    * `url: 'example.com'`
    * `url: "https://example.com"`

    URLs containing `:` must be surrounded by quotes
  </comp.Tab>
  <comp.Tab name="TOML">
    Any valid string:
    * `url = ""`
    * `url = ''`
    * `url = "example.com"`
    * `url = 'example.com'`
    * `url = "https://example.com"`
  </comp.Tab>
  <comp.Tab name="JSON">
    Null value:
    * `"url": null`

    Any valid string:
    * `"url": ""`
    * `"url": "example.com"`
    * `"url": "https://example.com"`
  </comp.Tab>
</comp.Tabs>
