A Boolean input is an editing interface for true or false values. You can use this input in the Data Editor, Content Editor, or Visual Editor to edit structured data in data files or front matter.

There are two types of Boolean input:

* Checkbox
* Switch

## Boolean input types

### Checkbox

The Checkbox Boolean input provides an editing interface for true or false values. True values have a checked box, while false values have an unchecked box.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/boolean/2024-07/CloudCannon-Documentation-Checkbox-Boolean-Input.png" alt="A screenshot of two Checkbox Boolean inputs in the Data Editor, showing one true and one false value." title="The Checkbox Boolean Input" type="ui-snippet"/>

You can configure a Checkbox input using the `type` key with the value `checkbox`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  blog_published___2___:
    type___3___: checkbox

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Boolean input is called `blog_published`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `checkbox` Boolean input.
</comp.Annotation>
</comp.MultiCodeBlock>

When you add your input key name to a data or markup file, your configured Checkbox input will appear in the Data Editor, Visual Editor, or Content Editor.

<comp.CodeBlock language="Markdown" source="blog.md">
```

---
blog_published: true
---

Blog content goes here.

```
</comp.CodeBlock>

### Switch

The Switch Boolean input provides an editing interface for true or false values. True values show the switch to the right with the text "ON", while false values show the switch to the left with the text "OFF".

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/boolean/2024-07/CloudCannon-Documentation-Switch-Boolean-Input.png" alt="A screenshot of two Switch Boolean inputs in the Data Editor, showing one true and one false value." title="The Switch Boolean Input" type="ui-snippet"/>

You can configure a Switch input using the `type` key with the value `switch`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  featured___2___:
    type___3___: switch

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Boolean input is called `featured`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `switch` Boolean input.
</comp.Annotation>
</comp.MultiCodeBlock>

When you add your input key name to a data or markup file, your configured Switch input will appear in the Data Editor, Visual Editor, or Content Editor.

<comp.CodeBlock language="YAML" source="data.yml">
```

featured: false

```
</comp.CodeBlock>

## Input configuration options

General configuration options are available for all input types. You can define the label, comment, and context box for your Boolean input, whether it is hidden or disabled, and how CloudCannon should handle configuration at multiple levels of the configuration cascade.

Specific configuration options for Boolean inputs include adding input validation, where CloudCannon requires a value for your Boolean input.

Here is an example of a Boolean input using some of the most commonly used configuration keys.

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs:
  blog_published:
    type: checkbox
    label: Published
    comment: This blog will appear on our website when this box is checked
    options:
      required: true

```
</comp.MultiCodeBlock>

For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/) in general, or our [inputs reference](/documentation/articles/inputs-reference/) for a complete list of available configuration keys.

## Valid values

Here are some examples of valid values for the key `available`. These work for both Checkbox and Switch inputs.

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `available: `

    Boolean values (without any quotation marks):
    * `available: true`
    * `available: false`
  </comp.Tab>
  <comp.Tab name="TOML">
    Boolean values (without any quotation marks):
    * `available = true`
    * `available = false`
  </comp.Tab>
  <comp.Tab name="JSON">
    Null value:
    * `"available": null`

    Boolean values (without any quotation marks):
    * `"available": true`
    * `"available": false`
  </comp.Tab>
</comp.Tabs>

## Unconfigured Boolean inputs

In some cases, CloudCannon can still detect a Boolean input even if you have not configured it.

CloudCannon will interpret any unconfigured input with a `true` or `false` value as a Checkbox Boolean input. CloudCannon cannot create a Switch input without configuration.

<comp.CodeBlock language="Markdown" source="article.md">
```

---
public: true
---

Content goes here.

```
</comp.CodeBlock>

This behavior is convenient if you have simple inputs or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.

<comp.Notice info_type="info">
  We recommend configuring your inputs for greater control over their functionality and appearance.
</comp.Notice>
