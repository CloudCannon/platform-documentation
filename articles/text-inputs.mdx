A text input lets your team members add plain text to your data files or front matter.

There are three types of Text input:

* Text
* Textarea
* Email

For each input, you can configure a character counter (Textarea only) and determine how CloudCannon handles null values. You can also use the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs.

## Text

The Text input provides a simple editing interface for plain text.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/text/2024-07/CloudCannon-Documentation-Text-Input.png" alt="A screenshot of the Text input in the Data Editor." title="The Text input" type="ui-snippet"/>

You can configure a Text input using the `type` key with the value `text`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  blog_title___2___:
    type___3___: text

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Document input is called `download_newsletter`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `document` File input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Text input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="about.md">
```
---
blog_title: The benefits of static sites
---

Page content goes here.
```
</comp.CodeBlock>

## Text

Single line input for short text values.

Text inputs are shown for inputs configured with the type `text`, or for inputs not matching any other naming convention.

<comp.DocsImage path="/documentation/static/inputs/text/text.png" alt="Screenshot of text input field" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```

tagline: Hand-crafted clothing.

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

footer_message: Look up!
_inputs:
  footer_message:
    type: text

```
</comp.MultiCodeBlock>

## Textarea

Expanded text input for longer and/or multi-line text.

Textarea inputs are shown for inputs configured with the type `textarea`, or for input keys matching:

* `textarea`
* `*_textarea`
* `description`
* `*_description`

<comp.DocsImage path="/documentation/static/inputs/text/textarea.png" alt="Screenshot of large text input field" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```

description: Products are crafted in-house by hand, making each piece a labor of love and an exercise in craftsmanship.

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

footer_copy: |-
  We hope you love our products.
  We are proud of them!
_inputs:
  footer_copy:
    type: textarea

```
</comp.MultiCodeBlock>

## Email

Single line input for email addresses.

Email inputs are shown for inputs configured with the type `email`, or for input keys matching:

* `email`
* `*_email`
* `email_address`
* `*_email_address`

<comp.DocsImage path="/documentation/static/inputs/text/email.png" alt="Screenshot of two email input fields using the email and email_address keys" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```

email: support@cloudcannon.com
email_address: support@cloudcannon.com

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

to: support@cloudcannon.com
_inputs:
  to:
    type: email

```
</comp.MultiCodeBlock>

## Disabled

Single line input that cannot be edited for any value.

Disabled inputs are shown for inputs configured with an unknown `type`, or for inputs with values that cannot be edited with the configured `type`.

<comp.Notice info_type="info">
  Text is stored unquoted if possible. Unquoted text containing only numbers with commas may require a [workaround in Jekyll](/documentation/articles/patching-yaml-number-parsing-for-jekyll/).
</comp.Notice>

### Options

Text inputs have the following available options:

<comp.DataReference>
  <comp.DataReferenceRow label="empty_type" type_markdown="string">
    Set how an ‘empty’ value will be saved. Does not apply to existing empty values. Can be one of the following:

    * `string` - an empty value for this input will be stored as `""`.
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.
  </comp.DataReferenceRow>
</comp.DataReference>

## Valid values

Documented below are the valid formats for **text**, **textarea**, and **email** inputs.

The lists of examples are non-exhaustive.

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `text: `

    Any valid string (quoted or unquoted):
    * `text: ""`
    * `text: ''`
    * `text: any string`
    * `text: "any string"`
    * `text: 'any string'`

    Any valid multiline string:
    * <code>text: ><br/>  multiline string</code>
    * <code>text: >-<br/>  multiline string</code>
    * <code>text: >+<br/>  multiline string</code>
    * <code>text: |<br/>  multiline string</code>
    * <code>text: |-<br/>  multiline string</code>
    * <code>text: |+<br/>  multiline string</code>
  </comp.Tab>
  <comp.Tab name="TOML">
    Any valid string:
    * `text = ""`
    * `text = "any string"`

    Any valid escaped string:
    * `text = ''`
    * `text = 'any string (literal)'`

    Any valid multiline string:
    * <code>text = """<br/>  multiline string"""</code>
    * <code>text = """\\<br/>  multiline string (trimmed) \\<br/>  """</code>
    * <code>text = '''<br/>  literal multiline string'''</code>
  </comp.Tab>
  <comp.Tab name="JSON">
    Null value:
    * `"text": null`

    Any valid string:
    * `"text": ""`
    * `"text": "any string"`

    Any valid multiline string:
    * `"text": "multiline \n string"`
  </comp.Tab>
</comp.Tabs>
