A Boolean input lets your team add true or false values to your data files or front matter.

There are two types of Boolean input:

* Checkbox
* Switch

Boolean inputs are simple. Beyond configuring the type (checkbox or switch) and the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs, there are no more configuration keys.

## Checkbox

The Checkbox Boolean input provides an editing interface for true or false values. True values have a checked box, while false values have an unchecked box.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/boolean/2024-07/CloudCannon-Documentation-Checkbox-Boolean-Input.png" alt="" title="The Checkbox Boolean Input" type="ui-snippet"/>

You can configure a Checkbox input using the `type` key with the value `checkbox`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  blog_published___2___:
    type___3___: checkbox

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Boolean input is called `blog_published`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `checkbox` Boolean input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Checkbox Boolean input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="blog.md">
```
---
blog_published: true
---

Blog content goes here.
```
</comp.CodeBlock>

## Switch

The Switch Boolean input provides an editing interface for true or false values. True values show the switch to the right with the text "ON", while false values show the switch to the left with the text "OFF".

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/boolean/2024-07/CloudCannon-Documentation-Switch-Boolean-Input.png" alt="" title="The Switch Boolean Input" type="ui-snippet"/>

You can configure a Switch input using the `type` key with the value `switch`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  featured___2___:
    type___3___: switch

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Boolean input is called `featured`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `switch` Boolean input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Switch Boolean input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="YAML" source="data.yml">
```
featured: false
```
</comp.CodeBlock>

## Unconfigured Boolean inputs

In some cases, CloudCannon can still detect a Boolean input even if you have not configured it.

CloudCannon will interpret any unconfigured input with a `true` or `false` value as a Checkbox Boolean input.

## Valid values

Here are some examples of valid values for the key `is_true`. These work for both Checkbox and Switch inputs.

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `is_true: `

    Boolean values (without any quotation marks):
    * `is_true: true`
    * `is_true: false`
  </comp.Tab>
  <comp.Tab name="TOML">
    Boolean values (without any quotation marks):
    * `is_true = true`
    * `is_true = false`
  </comp.Tab>
  <comp.Tab name="JSON">
    Null value:
    * `"is_true": null`

    Boolean values (without any quotation marks):
    * `"is_true": true`
    * `"is_true": false`
  </comp.Tab>
</comp.Tabs>

---

These inputs are for editing true/false values in your data.

## Checkbox

Checkbox input for values that can be toggled.

Checkbox inputs are shown for inputs configured with the type `checkbox`, or for the inputs values `true` and `false`.

<comp.DocsImage path="/documentation/static/inputs/boolean/checkbox.png" alt="Screenshot of checkbox input field" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Value convention">
```

show_feature: true

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

featured: true
_inputs:
  featured:
    type: checkbox

```
</comp.MultiCodeBlock>

## Switch

Switch input for values that can be toggled.

Switch inputs are shown for inputs configured with the type `switch`.

<comp.DocsImage path="/documentation/static/inputs/boolean/switch.png" alt="Screenshot of switch input field" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

featured: true
_inputs:
  featured:
    type: switch

```
</comp.MultiCodeBlock>
