A Number input

There are two types of Number input:

* Number
* Range

For each input, you can configure

You can also use the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs.

## Number

The Number input provides an editing interface for

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/number/2024-07/CloudCannon-Documentation-Number-Input.png" alt="A screenshot of the Number input in the Data Editor." title="The Number input" type="ui-snippet"/>

You can configure a Number input using the `type` key with the value `number`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  series___2___:
    type___3___: number

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Number input is called `series`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `number` input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Number input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="guide.md">
```
---
series: 5
---

Content goes here.
```
</comp.CodeBlock>

## Range

The Range input provides an editing interface for

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/number/2024-07/CloudCannon-Documentation-Range-Input.png" alt="A screenshot of the Range input in the Data Editor, showing a sliding scale between 0 and 20." title="The Range input" type="ui-snippet"/>

You can configure a Range input using the `type` key with the value `range`. Define the `type` key under your input key name within `_inputs`. For this input to function, you must also define the `min`, `max`, and `step` options.

For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  priority___2___:
    type___3___: range
    options___4___:
      min: 1
      max: 5
      step: 1

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Range input is called `priority`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `range` Number input.
</comp.Annotation>
<comp.Annotation number="3">
  Unlike many other input types, some options must be defined for the input to work. The Range input requires `min`, `max`, and `step` options.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Range input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="guide.md">
```
---
priority: 4
---

Content goes here.
```
</comp.CodeBlock>

These inputs are for editing numerical values in your data.

## Number

Single line input field for integers or decimals.

Number inputs are shown for inputs configured with the type `number`, or input keys matching:

* `number`
* `*_number`

<comp.DocsImage path="ye_olde_images/number-2.png" alt="Screenshot of a number input field with decimals" title="" type="screenshot-snippet"/>

<comp.DocsImage path="ye_olde_images/number-1.png" alt="Screenshot of a number input field" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```

number: 3.14
sort_number: 2

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

order: 1
_inputs:
  order:
    type: number

```
</comp.MultiCodeBlock>

## Range

A combination number and range input field for integers or decimals.

Range inputs are shown for inputs configured with the type `range`. This input requires `step`, `min` and `max` options.

<comp.DocsImage path="ye_olde_images/range-input.png" alt="Screenshot of a range input field" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

order: 12
_inputs:
  order:
    type: range
    options:
      step: 2
      max: 20
      min: 0

```
</comp.MultiCodeBlock>

## Options

Number input options are configured with `options` inside an `_inputs` entry.

<comp.DataReference>
  <comp.DataReferenceRow label="min" type_markdown="Float">
    The lowest value in the range of permitted values.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="max" type_markdown="Float">
    The greatest value in the range of permitted values.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="step" type_markdown={"Float or \"any\""}>
    A number that specifies the granularity that the value must adhere to, or the special value `any`, which allows any decimal value between `max` and `min`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="empty_type" type_markdown="string">
    Set how an 'empty' value will be saved. Does not apply to existing empty values. Can be one of the following:

    * `number` - an empty value for this input will be stored as `0` (default).
    * `null` - an empty value for this input will be stored as a null value. This does not apply to TOML files.
  </comp.DataReferenceRow>
</comp.DataReference>

## Valid values

Documented below are the valid formats for **number** and **range** inputs.

The lists of examples are non-exhaustive.

<comp.Notice info_type="info">
  Binary and hexadecimal numbers will be converted to base-10 integers when loaded into the Data Editor
</comp.Notice>

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `number: `

    Whole base-10 integers (unquoted):
    * `number: 123456`
    * `number: -123456`
    * `number: 123456e78`
    * `number: 123456e+78`
    * `number: 123456e-78`

    Floating point numbers (unquoted):
    * `number: 123.456`
    * `number: -123.456`
    * `number: 123.456e78`
    * `number: 123.456e+78`
    * `number: 123.456e-78`

    Binary integers (unquoted):
    * `number: 0b11110001001000000`

    Hexadecimal integers (unquoted):
    * `number: 0x1E240`
  </comp.Tab>
  <comp.Tab name="TOML">

    Hexadecimal numbers (unquoted):
