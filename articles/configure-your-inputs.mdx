You can configure your [inputs](/documentation/articles/what-are-inputs/) anywhere in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/). In this article, we will show you how to configure an input at the global level in your [global configuration file](/documentation/articles/create-a-global-configuration-file/).

To configure your inputs:

1. Navigate to your global configuration file and open it in the [Source Editor](/documentation/articles/the-source-editor/).
2. Identify the `_inputs` key, or create one at the top level of your configuration file.
3. Add the key name of your input under `_inputs`.
4. Add any keys you want to configure under your input key.
5. [Save your changes](/documentation/articles/save-your-changes/) to the global configuration file.

## Example

Let’s walk through an example.

We want to create a tag system for our blog articles so that users can filter our articles by tag for better search results. For this to work, we want to add a list containing relevant tags to the front matter of each blog file. When a content creator writes a new blog, they can select which tags apply using a [Multiselect input](/documentation/articles/select-and-multiselect-inputs/) in the sidebar of the Content Editor.

Here's out input configuration:

<comp.MultiCodeBlock language="YAML" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
``````````

_inputs___1___:
  blog_tags___2___:
    type___3___: multiselect
    label___4___: Blog type
    comment___5___: Select a blog type
    context___6___:
      open: false
      title: Help
      icon: help
      content: |
        Blog tags help our users filter articles by topic.
    options___7___:
      values___8___:
        - Opinion
        - Feature
        - Resource

``````````
<comp.Annotation number="1">
  Configure all inputs under the `_inputs` key anywhere in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  Enter the key name of the input here. In this example, the input name is `blog_tags`.
</comp.Annotation>
<comp.Annotation number="3">
  This input is a Multiselect input. For more information, please read our documentation on [Multiselect inputs](/documentation/articles/select-and-multiselect-inputs/).
</comp.Annotation>
<comp.Annotation number="4">
  The title of this input is “Blog type”. If no label is configured, CloudCannon will use the key name in title case.
</comp.Annotation>
<comp.Annotation number="5">
  The subtitle of this input is “Select a blog type”.
</comp.Annotation>
<comp.Annotation number="6">
  To help our team members use this input, we have configured a context dropdown with extra information.
</comp.Annotation>
<comp.Annotation number="7">
  Configuration options for inputs are nested keys under `options`.
</comp.Annotation>
<comp.Annotation number="8">
  This input has three possible values in an array under the values key: “Opinion”, “Feature”, and “Resource”.
</comp.Annotation>
</comp.MultiCodeBlock>

Once we add the above code to our global configuration file, we need to add the input to our blog file front matter.

<comp.CodeBlock language="Markdown" source="blog.md">
``````````
---
blog_tags:
---

The content of the blog goes here.
``````````
</comp.CodeBlock>

Once we save our changes to both files, opening our blog file will show the input `blog_tags` we configured in the Data Editor or in the sidebar of the Content Editor.
