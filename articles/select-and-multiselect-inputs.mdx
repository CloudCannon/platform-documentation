## Select

The Select input provides an editing interface for

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/select/2024-07/CloudCannon-Documentation-Select-Input.png" alt="A screenshot of the Select input in the Data Editor." title="The Select input" type="ui-snippet"/>

You can configure a Select input using the `type` key with the value `select`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  blog_author___2___:
    type___3___: select

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Select input is called `blog_author`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `select` input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Select input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="guide.md">
```
---
blog_author: Author A
---

Content goes here.
```
</comp.CodeBlock>

## Options

You can configure Select and Multiselect inputs using the `options` key under your input key, inside of `_inputs`.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  tags:
    type: multichoice
    options:
      allow_create:
      allow_empty:
      empty_type: null
      preview:
        text:
        icon:
        image:
      values:
      value_key:

```
</comp.MultiCodeBlock>

Number inputs have the following options available:

<comp.DataReference>
  <comp.DataReferenceRow label="allow_create" type_markdown="Boolean">
    This key determines whether you are allowed to enter a new text value that does not appear in the data set. CloudCannon does not add new text values to the data set. Defaults to `false`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="allow_empty" type_markdown="Boolean">
    This key toggles the presence of an empty option alongside the options listed under `values`. Defaults to `true`. When `false`, CloudCannon will use the first value in the data set by default.

    This key has no effect on Multiselect inputs.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="empty_type" type_markdown="string">
    This key determines how CloudCannon handles an empty value. This key does not apply to existing empty values.

    Value must be one of the following:

    * `string` - an empty value for this input will be stored as "".
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.

    Available for Multiselect and Multichoice inputs.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview" type_markdown="Object">
    This key enables you to define the appearance of input values in the Data Editor or sidebar of the Visual or Content Editor. The following nested keys are available:

    * `text`
    * `icon` (Choice/Multichoice only)
    * `image` (Choice/Multichoice only)

    Defaults to:

    <comp.CodeBlock language="YAML" source="cloudcannon.config.yml">
    ``````````
    preview:
      text:
        - key: title
        - key: name
      icon:
        - key: icon
    ``````````
    </comp.CodeBlock>

    For more information about previews, please read our documentation on [configuring card previews](/documentation/articles/configure-your-card-previews/).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.text" type_markdown="Array, string or boolean">
    Controls the main text shown for selected and available options. Has no effect unless the available or selected option is an object.

    If the value is a primitive type, it is used directly.

    If set to `false`, no text is displayed.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.icon" type_markdown="Array, string or boolean">
    Controls the icon shown for selected and available options. Must match a [Material Icon](https://material.io/tools/icons/) name.

    If set to `false`, no icon is displayed.

    Available to Choice and Multichoice inputs.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="values" type_markdown="Array, object or string">
    This key determines the available values for your input. If this key contains an object or array, CloudCannon will use the listed values. If this key contains a string, CloudCannon will fetch the values from the [data set](#data-sets) referenced.

    Defaults to fetching values from a data set following the naming convention (e.g. `colors` or `my_colors` for [data set](#data-sets) `colors`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="value_key" type_markdown="String or array of strings">
    Defines the key used for mapping between saved values and objects in `values`. This changes how the input saves selected values to match. Defaults to `['id', 'uuid', 'path', 'title', 'name']`. Has no effect unless `values` is an array of objects, the key is used instead for objects, and the value itself is used for primitive types.

    If set as an array, keys are tried in order, until a non-empty value is found. If no keys are set for a value available, the value available is not included.

    If set as a string, this key is used directly. Values available are excluded if they don't have a value for this key.
  </comp.DataReferenceRow>
</comp.DataReference>

<comp.Notice info_type="important">
  Sites created before 25 November 2021 have a different default for saving collection items in order to keep existing sites working.
  Create a new site, or use the [Legacy Collections Value Key](/documentation/articles/activating-flags/#legacy-collections-value-key) flag to update this for your site.
</comp.Notice>

## Select

Single value select input for values from a set of options.

Select inputs are shown for inputs configured with the type `select`, or for input keys matching or ending with **the singular form** of a data set key (e.g. `color` or `my_color` for data set `colors`).

<comp.DocsImage path="/documentation/static/inputs/select/color-select.png" alt="Color select input" title="" type="screenshot-snippet"/>

<comp.DocsImage path="/documentation/static/inputs/select/ssg-select.png" alt="SSG select input" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```

color: Red
ssg: jekyll
_select_data:
  colors:
    - Red
    - Blue
  ssgs:
    jekyll: Jekyll
    hugo: Hugo
    11ty: Eleventy

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">

## Options

Select, Multiselect, Choice and Multichoice input options are configured with `options` inside an `_inputs` entry.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

items:
  - P1
_inputs:
  items:
    type: multiselect
    options:
      values: _select_data.inventory
      value_key: key
      preview:
        text:
          - key: display_name
_select_data:
  inventory:
    - key: P1
      display_name: Potion
      kind: consumable

```
</comp.MultiCodeBlock>

## Data sets

There are currently three sources for populating the options for a select or multiselect input:

* Fixed data sets
* Collection items
* Data files

You can populate the available options for a Select, Multiselect, Choice or Multichoice input with a naming convention or set it explicitly with the input configuration `options.values`.

The saved value when an option is chosen in the editor depends on your data set and the `value_key` options.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

spare_part: Bolt
_inputs:
  spare_part:
    type: select
    options:
      values:
        - Bolt
        - Nut
        - Gear

```
</comp.MultiCodeBlock>

In addition to having `options.values` contain values directly as seen above, you can refer to one of the three available sources. Here's how you can refer to a collection called `bits_and_pieces`:

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

spare_part: Bolt
_inputs:
  spare_part:
    type: select
    options:
      values: collections.bits_and_pieces

```
</comp.MultiCodeBlock>

<comp.Notice info_type="info">
  You can refer to fixed data sets and data files in a similar way by changing the prefix. For example: `data.inventory` and `_select_data.spare_parts`.
</comp.Notice>

### Fixed data sets

Fixed data sets are defined in `_select_data` from the [configuration cascade](/documentation/articles/using-the-configuration-cascade/). Best suited when providing options that do not change as content is added or removed.

### Collection items

Collection items are files from your [defined collections](/documentation/articles/define-your-collections/). Best suited when you need to select from pages or other content files created by team members.

### Data files

Data files are similar to collections, usually a reserved collection for your SSG. Best suited when you need the available options editable by team members.

Processed data file sets must be arrays of strings, or objects. The value saved is either the string for arrays, or the key for objects.

By default, CloudCannon does not read data files during your build. To use this option source, instruct CloudCannon to read it by [defining your data](/documentation/articles/define-your-data/).
