A  Select or Multiselect input lets your team members select a value (or values) for data from a predefined list of options. Select inputs allow a single value, whereas Multiselect inputs allow multiple values.

There are four types of Select input:

* Select
* Multiselect
* Choice
* Multichoice

For each input, you can configure whether users can create their own values, if inputs can be empty, the appearance of input values, a list of options for data values, add ID keys to each value option, and how CloudCannon handles empty values. You can also use the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs.

You must configure how the value options in your Select or Multiselect input are populated for the input to function. There are three ways to populate the list of options for a Select or Multiselect input:

* Fixed data sets
* Collection items
* Data files

## Select

The Select input provides an editing interface for data with multiple predefined options, but where only one value is allowed at a time. In the Data Editor, predefined options appear in a dropdown menu.

This input has the same function as a Choice input, but a different appearance.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/select/2024-07/CloudCannon-Documentation-Select-Input.png" alt="A screenshot of the Select input in the Data Editor." title="The Select input" type="ui-snippet"/>

You can configure a Select input using the `type` key with the value `select`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  blog_author___2___:
    type___3___: select

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Select input is called `blog_author`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `select` input.
</comp.Annotation>
</comp.MultiCodeBlock>

Once configured, the Select input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="guide.md">
```
---
blog_author: Author A
---

Content goes here.
```
</comp.CodeBlock>

## Populating the value options

Without a list of options for the input value, Select and Multiselect inputs become misconfigured.

You can configure a fixed data set, or reference a Collection or data file to populate the list of value options.

### Fixed data sets

A fixed data set is an array of predefined value options for your Select or Multiselect input. Fixed data sets are best suited to inputs with values that do not change often.

There are two ways to define a fixed data set: within the input configuration using the `options.values` key or anywhere in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/) using the `_select_data` key.

To define the fixed data set within your input configuration, and your array of value options within the `values` key in the input `options`.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  pets___1____:
    type: multichoice
    options:
      values___2___:
        - Dog
        - Cat
        - Rabbit

```
<comp.Annotation number="1">
  This Multichoice input is called `pets`.
</comp.Annotation>
<comp.Annotation number="2">
  You can configure a fixed data set directly inside your input under the `options.values` key. This fixed data set contains the options "Dog", "Cat", and "Rabbit".
</comp.Annotation>
</comp.MultiCodeBlock>

To define your fixed data set elsewhere in the configuration cascade, create a nested key and array of value options within `_select_data`.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_select_data___1___:
  animals___2___:
    - Dog
    - Cat
    - Rabbit

```
<comp.Annotation number="1">
  You can configure a fixed data set anywhere in the configuration cascade using the `_select_data` key.
</comp.Annotation>
<comp.Annotation number="2">
  The fixed data set called `animals` contains the options "Dog", "Cat", and "Rabbit".
</comp.Annotation>
</comp.MultiCodeBlock>

You can reference a fixed data set defined elsewhere in the configuration cascade using the `values` key in the input `options`. Use the value `_select_data.key_name`, where `key_name` is replaced with the key name of your data set.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  pets___1____:
    type: multichoice
    options:
      values___2___: _select_data.animals

```
<comp.Annotation number="1">
  This Multichoice input is called `pets`.
</comp.Annotation>
<comp.Annotation number="2">
  You can reference a fixed data set configured elsewhere in the configuration cascade using the `options.values` key. This input references a fixed data set called `animals`.
</comp.Annotation>
</comp.MultiCodeBlock>

### Collections

A [Collection](/documentation/articles/define-your-collections/) is a single folder of files with a similar format. You can find your Collections in the *Site Navigation* sidebar. Using a Collection to populate your value options is useful when you want to reference pages or other content files created by team members (e.g., blogs, staff pages, or products).

You can reference a Collection for your Select or Multiselect input using the `values` key in the input `options`. Use the value `collections.key_name`, where `key_name` is replaced with the key name of your Collections.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  product_carousel___1____:
    type: multiselect
    options:
      values___2___: collections.products

```
<comp.Annotation number="1">
  This Multiselect input is called `product_carousel`.
</comp.Annotation>
<comp.Annotation number="2">
  This input references a Collection called `products`.
</comp.Annotation>
</comp.MultiCodeBlock>

<comp.Notice info_type="important">
  You must have [defined your Collections](/documentation/articles/define-your-collections/) for this configuration to work.
</comp.Notice>

### Data files

You can populate your input using a dedicated data file which you can open and edit this file using the CloudCannon file browser. Using a data file to populate your value options is useful when you want you team members to be able to edit the available options.

<comp.Notice info_type="info">
  Your Site may encounter issues if a team member removes an option from your data file that is in use by an input.
</comp.Notice>

Data files must contain arrays of strings or objects. The Select or Multiselect input will use the strings if the data file contains an array, and the key names if the data file contains objects.

<comp.CodeBlock language="YAML" source="data.yml">
``````````
- Blue
- Green
- Red
``````````
</comp.CodeBlock>

You can reference a data file for your Select or Multiselect input using the `values` key in the input `options`. Use the value `data.key_name`, where `key_name` is replaced with the key name of your Collections.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  product_carousel___1____:
    type: select
    options:
      values___2___: collections.products

```
<comp.Annotation number="1">
  This Select input is called `blog_author`.
</comp.Annotation>
<comp.Annotation number="2">
  The value of the `type` key determines the input type. This is a `select` input.
</comp.Annotation>
</comp.MultiCodeBlock>

<comp.Notice info_type="important">
  You must have [defined your Data](/documentation/articles/define-your-data/) for this configuration to work.
</comp.Notice>

## Options

You can configure Select and Multiselect inputs using the `options` key under your input key, inside of `_inputs`.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  tags:
    type: multichoice
    options:
      allow_create: true
      allow_empty: true
      empty_type: string
      preview:
        text:
        icon:
        image:
      values:
        - Pink
        - Blue
        - Orange
      value_key:

```
</comp.MultiCodeBlock>

Number inputs have the following options available:

<comp.DataReference>
  <comp.DataReferenceRow label="allow_create" type_markdown="Boolean">
    This key determines whether you are allowed to enter a new text value that does not appear in the data set. CloudCannon does not add new text values to the data set. Defaults to `false`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="allow_empty" type_markdown="Boolean">
    This key toggles the presence of an empty option alongside the options listed under `values`. Defaults to `true`. When `false`, CloudCannon will use the first value in the data set by default.

    This key has no effect on Multiselect inputs.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="empty_type" type_markdown="string">
    This key determines how CloudCannon handles an empty value. This key does not apply to existing empty values.

    Value must be one of the following:

    * `string` - an empty value for this input will be stored as "".
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.

    Available for Multiselect and Multichoice inputs.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview" type_markdown="Object">
    This key enables you to define the appearance of input values in the Data Editor or sidebar of the Visual or Content Editor. The following nested keys are available:

    * `text`
    * `icon` (Choice/Multichoice only)
    * `image` (Choice/Multichoice only)

    Defaults to:

    <comp.CodeBlock language="YAML" source="cloudcannon.config.yml">
                                                        ``````````
                                                        preview:
                                                          text:
                                                            - key: title
                                                            - key: name
                                                          icon:
                                                            - key: icon
                                                        ``````````
                                                        </comp.CodeBlock>

    For more information about previews, please read our documentation on [configuring card previews](/documentation/articles/configure-your-card-previews/).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.text" type_markdown="Array, string or boolean">
    Controls the main text shown for selected and available options. Has no effect unless the available or selected option is an object.

    If the value is a primitive type, it is used directly.

    If set to `false`, no text is displayed.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.icon" type_markdown="Array, string or boolean">
    Controls the icon shown for selected and available options. Must match a [Material Icon](https://material.io/tools/icons/) name.

    If set to `false`, no icon is displayed.

    Available to Choice and Multichoice inputs.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="values" type_markdown="Array, object or string">
    This key determines the available values for your input. If this key contains an object or array, CloudCannon will use the listed values. If this key contains a string, CloudCannon will fetch the values from the [data set](#data-sets) referenced.

    Defaults to fetching values from a data set following the naming convention (e.g. `colors` or `my_colors` for [data set](#data-sets) `colors`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="value_key" type_markdown="String or array of strings">
    Defines the key used for mapping between saved values and objects in `values`. This changes how the input saves selected values to match. Defaults to `['id', 'uuid', 'path', 'title', 'name']`. Has no effect unless `values` is an array of objects, the key is used instead for objects, and the value itself is used for primitive types.

    If set as an array, keys are tried in order, until a non-empty value is found. If no keys are set for a value available, the value available is not included.

    If set as a string, this key is used directly. Values available are excluded if they don't have a value for this key.
  </comp.DataReferenceRow>
</comp.DataReference>

<comp.Notice info_type="important">
  Sites created before 25 November 2021 have a different default for saving collection items in order to keep existing sites working.
  Create a new site, or use the [Legacy Collections Value Key](/documentation/articles/activating-flags/#legacy-collections-value-key) flag to update this for your site.
</comp.Notice>

## Unconfigured Select and Multiselect inputs

In some cases, CloudCannon can still detect an Object input even if you have not configured it.

CloudCannon will interpret any unconfigured input with nested keys as an Object input. You cannot create structured or mutable Object inputs without configuration.

<comp.CodeBlock language="YAML" source="data.yml">
```
company_photo:
  image:
  date:
  description:
```
</comp.CodeBlock>

This behavior is convenient if you have simple inputs or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.

<comp.Notice info_type="info">
  We recommend configuring your inputs for greater control over their functionality and appearance.
</comp.Notice>

## Select

Single value select input for values from a set of options.

Select inputs are shown for inputs configured with the type `select`, or for input keys matching or ending with **the singular form** of a data set key (e.g. `color` or `my_color` for data set `colors`).

## Multiselect

Multiple value select input for values from a set of options.

Multiselect inputs are shown for inputs configured with the type `multiselect`, or for input keys matching or ending with a data set key (e.g. `colors` or `my_colors` for data set `colors`).
