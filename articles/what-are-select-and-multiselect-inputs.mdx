Select and Multiselect inputs are editing interfaces for selecting values from predefined lists. You can use these inputs in the Data Editor, Content Editor, or Visual Editor to edit structured data in data files or front matter.

There are four types of Select and Multiselect inputs:

* Select
* Multiselect
* Choice
* Multichoice

## Select input types

### Select

The Select input provides an editing interface for data with multiple predefined options. Only one value is allowed at a time. In the Data Editor, predefined options appear in a dropdown menu.

This input has the same function as a Choice input but a different appearance.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/select/2024-07/CloudCannon-Documentation-Select-Input.png" alt="A screenshot of the Select input in the Data Editor." title="The Select input" type="ui-snippet"/>

You can configure a Select input using the `type` key with the value `select`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  blog_author___2___:
    type___3___: select

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Select input is called `blog_author`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `select` input.
</comp.Annotation>
</comp.MultiCodeBlock>

When you add your input key name to a data or markup file, your configured Select input will appear in the Data Editor, Visual Editor, or Content Editor.

<comp.CodeBlock language="Markdown" source="blog.md">
```

---
blog_author: Author A
---

Content goes here.

```
</comp.CodeBlock>

## Input configuration options

General configuration options are available for all input types. You can define the label, comment, and context box for your Select or Multiselect input, whether it is hidden or disabled, and how CloudCannon should handle configuration at multiple levels of the configuration cascade.

Specific configuration options for Select and Multiselect inputs include defining the list of input options, the appearance of option previews, whether you can create new values, whether the input can be empty, and how CloudCannon handles empty values. You can also add input validation to require a value, specify the minimum and maximum number of items, and whether CloudCannon should require unique values.

Here is an example of a Select input using some of the most commonly used configuration keys.

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs:
  blog_author:
    type: select
    label: Blog Author
    comment: Select the author for this blog post
    options:
      values:
        - name: Karen Key
          role: Project Manager
        - name: Holly James
          role: Engineering Lead
      value_key: name
      allow_empty: false
      required: true

```
</comp.MultiCodeBlock>

For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/) in general, or our [inputs reference](/documentation/articles/inputs-reference/) for a complete list of available configuration keys.

## Populating input value options

All Select and Multiselect inputs must have a predefined list of value options from which to select. There are three ways to populate the list of options:

* Fixed data sets
* Collection items
* Data files

### Fixed data sets

A fixed data set is an array of predefined value options for your Select or Multiselect input. Fixed data sets are best suited to inputs with values that do not change often.

### Collections

A [Collection](/documentation/articles/what-is-a-collection/) is a single folder of files with a similar format. You can find your Collections in the *Site Navigation* sidebar. Using a Collection to populate your value options is useful when you want to reference pages or other content files created by team members (e.g., blogs, staff pages, or products).

You can reference a Collection for your Select or Multiselect input using the `values` key in the input `options`. Use the value `collections.key_name`, replacing `key_name` with the key name of your Collections.

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs:
  product_carousel___1___:
    type: multiselect
    options:
      values___2___: collections.products

```
<comp.Annotation number="1">
  This Multiselect input is called `product_carousel`.
</comp.Annotation>
<comp.Annotation number="2">
  This input references a Collection called `products`.
</comp.Annotation>
</comp.MultiCodeBlock>

<comp.Notice info_type="important">
  You must have [configured your Collections](/documentation/articles/configure-your-collections/) for this configuration to work.
</comp.Notice>

## Valid values

Here are some examples of valid values for Select and Multiselect inputs.

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `select_value: `

    Single value (Select/Choice):
    * `select_value: option1`
    * `select_value: "option1"`

    Multiple values (Multiselect/Multichoice):
    ```yaml
    multiselect_value:
      - option1
      - option2
      - option3
    ```
  </comp.Tab>
  <comp.Tab name="TOML">
    Single value (Select/Choice):
    * `select_value = "option1"`

    Multiple values (Multiselect/Multichoice):
    * `multiselect_value = ["option1", "option2", "option3"]`
  </comp.Tab>
  <comp.Tab name="JSON">
    Empty/null value:
    * `"select_value": null`

    Single value (Select/Choice):
    * `"select_value": "option1"`

    Multiple values (Multiselect/Multichoice):
    ```json
    {
                                  "multiselect_value": [
                                    "option1",
                                    "option2",
                                    "option3"
                                  ]
                                }
    ```
  </comp.Tab>
</comp.Tabs>

## Unconfigured Select and Multiselect inputs

In some cases, CloudCannon can still detect a Select or Multiselect input even if you have not configured it.

CloudCannon will interpret any key as a Select input if the key name ends in the singular form of a fixed data set you have defined under `_select_data` (e.g., `author` or `blog_author` for the fixed data set `authors`).

<comp.CodeBlock language="Markdown" source="blog.md">
```

---
blog_author: Karen Key
---

Blog content goes here.

```
</comp.CodeBlock>

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_select_data:
  authors:
    - Karen Key
    - Billy Mills
    - Holly James

```
</comp.MultiCodeBlock>

This behavior is convenient if you have simple inputs or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.
