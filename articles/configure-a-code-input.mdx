<comp.Notice info_type="permissions">
  **Permissions required**

  Members of all [Default Permission Groups](/documentation/articles/what-are-default-permission-groups/), or [Custom Permission Groups](/documentation/articles/what-are-custom-permission-groups/) with the `site:file:write` permission, can configure inputs in all locations in the configuration cascade. You can limit permission to specific locations in the configuration using [file globs](/documentation/articles/what-are-custom-permission-groups/#specify-a-file-glob).
</comp.Notice>

A Boolean input is an editing interface for true or false values. You can use this input in the Data Editor, Content Editor, or Visual Editor, to edit structured data in data files or front matter.

These instructions assume that you know where in the configuration cascade you want to configure your Code input. For more information, please read our documentation on [inputs](/documentation/articles/what-are-inputs/) in general and the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).

To configure a Code input:

1. Open your website files in your local development environment, or log in to CloudCannon and select the Site for which you want to configure your input.
2. Navigate to the location in the configuration cascade where you want to configure your input. This can be the root of your CloudCannon Configuration File, within the `collections_config.[*]` key in your CloudCannon Configuration File, a Schema file, a markup file, or any where you configure a Structure.
3. Identify the `_inputs` key, or create one at that level of the configuration cascade.
4. Add an input name key for your input under the `_inputs` key (a.k.a., `_inputs.[*]`). We recommend naming your input something simple that indicates the function or context.
5. Add the `type` key under your input name key, with the `code` value.
6. (Optional.) Add any other general configuration keys (e.g., `label`, `comment`, `context`, `options`) under your input name key.
7. (Optional.) Add any specific configuration keys under `_inputs.[*].options` (e.g., `required`).

CloudCannon will now apply this Code input configuration to all markup files that use your input name key, without needing to save your configuration. This allows you to make changes to your input configuration and see those changes affect inputs live.

When you are happy with your input configuration, you must save your input configuration.

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  blog_published___2___:
    type___3___: checkbox
    label: string
    comment: string
    context:
      open: boolean
      title: string
      icon: string
      content: string
    hidden: boolean, or string
    disabled: boolean, or string
    cascade: boolean
    options:
      required: boolean

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Boolean input is called `blog_published`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `checkbox` Boolean input.
</comp.Annotation>
</comp.MultiCodeBlock>

## Valid values

Here are some examples of valid values for the key `available`. These work for both Checkbox and Switch inputs.

<comp.Tabs label="Languages">
  <comp.Tab name="YAML">
    Empty/null value:
    * `available: `

    Boolean values (without any quotation marks):
    * `available: true`
    * `available: false`
  </comp.Tab>
  <comp.Tab name="TOML">
    Boolean values (without any quotation marks):
    * `available = true`
    * `available = false`
  </comp.Tab>
  <comp.Tab name="JSON">
    Null value:
    * `"available": null`

    Boolean values (without any quotation marks):
    * `"available": true`
    * `"available": false`
  </comp.Tab>
</comp.Tabs>
