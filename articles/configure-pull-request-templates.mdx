Pull Request Templates allow you to set default values for the title and body of any pull request made from a Site. You can also configure the title and body [inputs](/documentation/articles/what-are-inputs/ "What are inputs?") that are used when creating a pull request.

You can have multiple pull request templates configured on a single site. In this case, users will be able to select which one they would like to use.

## Configure a pull request template

To configure a pull request template:

1. Navigate to your global configuration file and open it in the Source Editor.
2. Identify the `pull_request_templates` key, or create one at the top level of your configuration file.
3. Add any of the keys listed below, within an item in the `pull_request_templates` array.
4. Save your changes to the global configuration file.

<comp.DataReference>
  <comp.DataReferenceRow label="label" type_markdown="<p>String</p>">
    <p>If you have multiple pull request templates defined, users will see a select box to choose their template when creating a pull request. CloudCannon will use this label to help identify which is which.</p>
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="title" type_markdown="<p>String</p>">
    <p>Define the default value of the title of the pull request.</p>
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="body" type_markdown="<p>String</p>">
    <p>Define the default value of the body of the pull request.</p>
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="template_path" type_markdown="<p>String</p>">
    <p>Sets the path for a file containing your pull request template. The file path should be relative to your root directory. CloudCannon will use the contents of this file as the default value for the pull request body.</p><p>This value will not be used if <code>body</code> is set on this pull request template.</p>
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="_inputs" type_markdown="<p>Object</p>">
        <p>This key allows you to define _inputs configuration for the title and body inputs. The default value is:</p>
        <comp.CodeBlock language="yaml">
        ``````````

        _inputs:
          body:
            type: textarea

        ``````````
        </comp.CodeBlock>
        <p>See <a href="/documentation/articles/what-are-inputs/">this article</a> for more information about configuring inputs.</p>
  </comp.DataReferenceRow>
</comp.DataReference>

Here is some example configuration:

<comp.MultiCodeBlock language="yaml" translate_into={["json"]} source="cloudcannon.config.yml">
``````````

pull_request_templates:
  - label: General changes ___1___
    _inputs:
      title:
        options:
          required: true ___2___
      body:
        type: markdown
        options: ___3___
          bold: true
          italic: true
          link: true
          format: p h1 h2 h3
  - label: Breaking changes
    title: Breaking changes
    _inputs:
      title:
        options:
          disabled: true ___4___
      template_path: .github/templates/bug-fix.md ___5___

``````````
<comp.Annotation number="1">
  Since multiple pull request templates are defined, users will see a select box to choose their template when creating a pull request. CloudCannon will use this label in the select box.
</comp.Annotation>
<comp.Annotation number="2">
  This ensures the `title` field must be filled out before the pull request can be submitted.
</comp.Annotation>
<comp.Annotation number="3">
  These options allow the `body` to be edited as Markdown, with a stripped-back set of editing options.
</comp.Annotation>
<comp.Annotation number="4">
  The `disabled` option is used to ensure the default value of `title` cannot be changed.
</comp.Annotation>
<comp.Annotation number="5">
  The default value for the pull request body will be filled from the specified file.
</comp.Annotation>
</comp.MultiCodeBlock>
