A Number input is an editing interface for numeric values. You can use this input in the Data Editor, Content Editor, or Visual Editor to edit structured data in data files or front matter.

There are two types of Number input:

* Number
* Range

## Number input types

### Number

The Number input provides an editing interface for numeric values with optional step controls.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/number/2024-07/CloudCannon-Documentation-Number-Input.png" alt="A screenshot of two Number inputs in the Data Editor, showing integer and decimal values." title="The Number input" type="ui-snippet"/>

You can configure a Number input using the `type` key with the value `number`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  step___2___:
    type___3___: number

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Number input is called `step`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `number` input.
</comp.Annotation>
</comp.MultiCodeBlock>

When you add your input key name to a data or markup file, your configured Number input will appear in the Data Editor, Visual Editor, or Content Editor.

<comp.CodeBlock language="Markdown" source="product.md">
```

---
step: 4
---

Guide content goes here.

```
</comp.CodeBlock>

### Range

The Range input provides a slider interface for selecting a numeric value within a defined range.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/number/2024-07/CloudCannon-Documentation-Range-Input.png" alt="A screenshot of two Range inputs in the Data Editor, showing sliders with different ranges." title="The Range input" type="ui-snippet"/>

You can configure a Range input using the `type` key with the value `range`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  priority___2___:
    type___3___: range

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Range input is called `priority`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is a `range` input.
</comp.Annotation>
</comp.MultiCodeBlock>

When you add your input key name to a data or markup file, your configured Range input will appear in the Data Editor, Visual Editor, or Content Editor.

<comp.CodeBlock language="YAML" source="data.yml">
```

priority: 10

```
</comp.CodeBlock>

## Input configuration options

General configuration options are available for all input types. You can define the label, comment, and context box for your Number input, whether it is hidden or disabled, and how CloudCannon should handle configuration at multiple levels of the configuration cascade.

Specific configuration options for Number inputs include defining the minimum and maximum values, step size, and how CloudCannon handles empty values. You can also add input validation to require a value or match a regular expression.

Here is an example of a Number input using some of the most commonly used configuration keys.

<comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
```

_inputs:
  available:
    type: number
    label: Product Quantity
    comment: Enter the number of items in stock
    options:
      min: 0
      max: 1000
      step: 1
      required: true

```
</comp.MultiCodeBlock>

For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/) in general, or our [inputs reference](/documentation/articles/inputs-reference/) for a complete list of available configuration keys.

## Unconfigured Number inputs

In some cases, CloudCannon can still detect a Number input even if you have not configured it.

CloudCannon will interpret any unconfigured input with a numeric value as a Number input. CloudCannon cannot create a Range input without configuration.

<comp.CodeBlock language="YAML" source="data.yml">
``````````

page_number: 7
days: 50

``````````
</comp.CodeBlock>

This behavior is convenient if you have simple inputs or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.

<comp.Notice info_type="info">
  We recommend configuring your inputs for greater control over their functionality and appearance.
</comp.Notice>
