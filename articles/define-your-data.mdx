Defining data allows you to populate [select and multiselect inputs](/documentation/articles/select-and-multiselect-inputs/). Each data set corresponds to a file or folder in your site files.

By default, CloudCannon does not read the contents of data files during your build. If required, include data with the instructions for your SSG:

<comp.Conditional ssgs={["Jekyll"]}>
  To include all data sets, add the following to your global configuration file:

  <comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
  ```

  data_config: true

  ```
  </comp.MultiCodeBlock>

  Include only select data sets with specific keys:

  <comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
  ```

  data_config:
    authors: true
    offices: true

  ```
  </comp.MultiCodeBlock>
</comp.Conditional>

<comp.Conditional not_ssgs={["Jekyll", "Eleventy", "Hugo"]}>
  To include data sets with specific keys, [reader](https://github.com/CloudCannon/reader) parses files and folders listed in `data_config`. In the following
  example global configuration there are two data sets:

  1. `authors` which is read from the contents of a single file
  1. `offices` which is read from the contents of each file in a directory

  <comp.MultiCodeBlock language="YAML" translate_into={["json"]} source="cloudcannon.config.yml">
  ```

  data_config:
    authors:
      path: data/authors.csv
    offices:
      path: data/offices
      parser: json

  ```
  </comp.MultiCodeBlock>
</comp.Conditional>
