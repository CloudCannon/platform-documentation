Markdown files can be opened in Visual Editor if these conditions are met:

* The page has a layout defined and is output on the built site
* In the layout, there must be an element surrounding the output of the Markdown content, and it canâ€™t have any other children.

For example, Markdown files that use this layout **can** be opened in the Visual Editor:



<comp.Conditional ssgs={["Jekyll"]}>
  <comp.CodeBlock language="html" source="page.html">
  ```
  <div class="content">
    {{ content }}
  </div>
  ```
  </comp.CodeBlock>
</comp.Conditional>



<comp.Conditional ssgs={["Hugo"]}>
  <comp.CodeBlock language="html" source="single.html">
  ```
  <div class="content">
    {{ .Content }}
  </div>
  ```
  </comp.CodeBlock>
</comp.Conditional>



<comp.Conditional ssgs={["Eleventy"]}>
  <comp.CodeBlock language="html" source="page.njk">
  ```
  <div class="content">
    {{ content | safe }}
  </div>
  ```
  </comp.CodeBlock>
</comp.Conditional>



<comp.Conditional not_ssgs={["Jekyll", "Hugo", "Eleventy"]}>
  There's no official support to automatically create an Editable Region for content in other SSGs as the way they render content varies greatly.
</comp.Conditional>



Markdown files that use this layout **can not** be opened in the Visual Editor:



<comp.Conditional ssgs={["Jekyll"]}>
  <comp.CodeBlock language="html" source="page.html">
  ```
  <div class="content">
    {{ content }}
    <p> Some other content</p>
  </div>
  ```
  </comp.CodeBlock>
</comp.Conditional>



<comp.Conditional ssgs={["Hugo"]}>
  <comp.CodeBlock language="html" source="single.html">
  ```
  <div class="content">
    {{ .Content }}
    <p> Some other content</p>
  </div>
  ```
  </comp.CodeBlock>
</comp.Conditional>



<comp.Conditional ssgs={["Eleventy"]}>
  <comp.CodeBlock language="html" source="page.njk">
  ```
  <div class="content">
    {{ content | safe }}
    <p> Some other content</p>
  </div>
  ```
  </comp.CodeBlock>
</comp.Conditional>



<comp.Conditional not_ssgs={["Jekyll", "Hugo", "Eleventy"]}>
  Not applicable.
</comp.Conditional>
