These inputs are for editing plain text in your data.

## Text

Single line input for short text values.

Text inputs are shown for inputs configured with the type `text`, or for inputs not matching any other naming convention.

<comp.DocsImage path="ye_olde_images/text.png" alt="Screenshot of text input field" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```
tagline: Hand-crafted clothing.
```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```
footer_message: Look up!
_inputs:
  footer_message:
    type: text
```
</comp.MultiCodeBlock>

## Textarea

Expanded text input for longer and/or multi-line text.

Textarea inputs are shown for inputs configured with the type `textarea`, or for input keys matching:

* `textarea`
* `*_textarea`
* `description`
* `*_description`
<comp.DocsImage path="ye_olde_images/textarea.png" alt="Screenshot of large text input field" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```
description: Products are crafted in-house by hand, making each piece a labor of love and an exercise in craftsmanship.
```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```
footer_copy: |-
  We hope you love our products.
  We are proud of them!
_inputs:
  footer_copy:
    type: textarea
```
</comp.MultiCodeBlock>

## Email

Single line input for email addresses.

Email inputs are shown for inputs configured with the type `email`, or for input keys matching:

* `email`
* `*_email`
* `email_address`
* `*_email_address`
<comp.DocsImage path="ye_olde_images/email.png" alt="Screenshot of an email input field using the email key" type="screenshot-snippet"/>
<comp.DocsImage path="ye_olde_images/email-2.png" alt="Screenshot of an email input field using the email_address key" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Naming convention">
```
email: support@cloudcannon.com
email_address: support@cloudcannon.com
```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```
to: support@cloudcannon.com
_inputs:
  to:
    type: email
```
</comp.MultiCodeBlock>

## Disabled

Single line input that cannot be edited for any value.

Disabled inputs are shown for inputs configured with an unknown `type`, or for inputs with values that cannot be edited with the configured `type`.

<comp.Notice info_type="info">
  Text is stored unquoted if possible. Unquoted text containing only numbers with commas may require a [workaround in Jekyll](/documentation/articles/patching-yaml-number-parsing-for-jekyll/).
</comp.Notice>

### Options

Text inputs have the following available options:

<comp.DataReference>
 <comp.DataReferenceRow label="empty_type" type_markdown="string">
    Set how an ‘empty’ value will be saved. Does not apply to existing empty values. Can be one of the following:

    * `string` - an empty value for this input will be stored as `""`.
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.
 </comp.DataReferenceRow>
</comp.DataReference>

## Valid values

Documented below are the valid formats for **text**, **textarea**, and **email** inputs.

The lists of examples are non-exhaustive.

<comp.Tabs label="Languages">
 <comp.Tab name="YAML">
    Empty/null value:
    * `text: `

    Any valid string (quoted or unquoted):
    * `text: ""`
    * `text: ''`
    * `text: any string`
    * `text: "any string"`
    * `text: 'any string'`

    Any valid multiline string:
    * <code>text: ><br/>  multiline string</code>
    * <code>text: >-<br/>  multiline string</code>
    * <code>text: >+<br/>  multiline string</code>
    * <code>text: |<br/>  multiline string</code>
    * <code>text: |-<br/>  multiline string</code>
    * <code>text: |+<br/>  multiline string</code>
 </comp.Tab>
 <comp.Tab name="TOML">
    Any valid string:
    * `text = ""`
    * `text = "any string"`

    Any valid escaped string:
    * `text = ''`
    * `text = 'any string (literal)'`

    Any valid multiline string:
    * <code>text = """<br/>  multiline string"""</code>
    * <code>text = """\\<br/>  multiline string (trimmed) \\<br/>  """</code>
    * <code>text = '''<br/>  literal multiline string'''</code>
 </comp.Tab>
 <comp.Tab name="JSON">
    Null value:
    * `"text": null`

    Any valid string:
    * `"text": ""`
    * `"text": "any string"`

    Any valid multiline string:
    * `"text": "multiline \n string"`
 </comp.Tab>
</comp.Tabs>
