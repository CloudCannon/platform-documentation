Inputs

CloudCannon supports twelve types of input: [Text](/documentation/articles/text-inputs/), [Rich text](/documentation/articles/rich-text-inputs/), [URL](/documentation/articles/url-inputs/), [File](/documentation/articles/file-inputs/), [Select and Multiselect](/documentation/articles/select-and-multiselect-inputs/), [Boolean](/documentation/articles/boolean-inputs/), [Code](/documentation/articles/code-inputs/), [Color](/documentation/articles/color-inputs/), [Number](/documentation/articles/number-inputs/), [Date and Time](/documentation/articles/date-and-time-inputs/), [Object](/documentation/articles/object-inputs/), and [Array](/documentation/articles/array-inputs/).

This article covers all keys available in CloudCannon for configuring your inputs. For more information, please read our documentation on [inputs](/documentation/articles/what-are-inputs/) in general, [configuring your inputs](/documentation/articles/configure-your-inputs/), and each input type.

## General input options

You can configure inputs under your input key, inside of `_inputs`.

<comp.MultiCodeBlock language="YAML" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
``````````

_inputs:
  example:
    type: datetime
    label: Publication date
    comment: The
    context:
      open: false
      title: Help
      icon: help
      content: This field will automatically populate when you create an article.
    hidden: false
    instance_value: NOW
    cascade:
    options:

``````````
</comp.MultiCodeBlock>

All inputs have the following options available:

<comp.DataReference>
  <comp.DataReferenceRow label="type" type_markdown="String">
    Changes the editing interface for an input. Each input appears and behave differently, takes different `options`, and processes different values. See more details in the [type section](#type) below.

    For unknown types, or if the input contains an unsupported value (e.g. an array value in a text input), the `disabled` type is used instead.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="label" type_markdown="String">
    Changes the text above an input. Defaults to generating the label from the input key (e.g. "Page Title" for `page_title`).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="comment" type_markdown="String">
    Changes the subtext below the `label`. Has no default. Supports a limited set of Markdown: links, bold, italic, subscript, superscript and inline code elements are allowed.

    Useful for adding reminders or additional context for your team members on a specific input.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="context" type_markdown="Object">
    x
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="context.open" type_markdown="Boolean">
    x
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="context.title" type_markdown="String">
    x
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="context.icon" type_markdown="String">
    x
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="context.content" type_markdown="String">
    x
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="options" type_markdown="Object">
    Configuration settings passed on to the input when displayed. These are specific to each input. The [available types and their options](#types-reference) are listed below.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="instance_value" type_markdown="String">
    Controls if and how the value of this input is instantiated when created. This occurs when creating files, or adding array items containing the configured input. Has no default. Must be one of `UUID` or `NOW`.

    * `UUID` generates a uuidv4 ([extremely unlikely to generate duplicates)](https://softwareengineering.stackexchange.com/a/130298), useful for identifying unique items (e.g. `6ec0bd7f-11c0-43da-975e-2a8ad9ebae0b`)
    * `NOW` generates the current datetime in the site's configured timezone
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="hidden" type_markdown="Boolean or string">
    Toggles the visibility of this input. Useful for when you need an input to exist, but don't want it edited or seen. Defaults to the naming convention, `true` for input keys starting with an underscore and `false` otherwise.

    If set as a boolean, the input is hidden based if `true`.

    If set as a string, the input is hidden based on the value of another input. This can start with a `!` to reverse the value. For reverse values in `YAML` files, the string should be in `'` or `"` quotation marks.

    * `published` hides an input when the sibling input `published` is truthy
    * `!published` hides an input when the sibling input `published` is falsy
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="cascade" type_markdown="Boolean">
    Specifies whether or not this input configuration should be merged with any matching, less specific configuration. Defaults to `true`.

    The [configuration cascade](/documentation/articles/using-the-configuration-cascade/) works by finding the most specific `_inputs` entry. Usually, once an option is found in the cascade, there's no reason to keep looking.

    When this is `true`, the cascade continues looking and each entry found is merged. This allows you to define a `comment` globally, then a different `label` for inputs in a collection without redefining the `comment`. You can stop the cascade at any given point by setting `cascade` to `false`.
  </comp.DataReferenceRow>
</comp.DataReference>

### Rich text input options

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="cloudcannon.config.yml">
```

_inputs:
  description:
    type: markdown
    options:
      allow_resize: false
      empty_type: "null"
      initial_height: 200
      paths:
        dam_uploads: assets/images
        dam_uploads_filename: "[filename].[extension]"

```
</comp.MultiCodeBlock>

Rich Text inputs have the following options available:

<comp.DataReference>
  <comp.DataReferenceRow label="allow_resize" type_markdown="Boolean">
    Shows or hides the resize handler to vertically resize the input. Defaults to `true`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="empty_type" type_markdown="string">
    Set how an ‘empty’ value will be saved. Does not apply to existing empty values. Can be one of the following:

    * `string` - an empty value for this input will be stored as `""`.
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="initial_height" type_markdown="Integer">
    Defines the initial height of this input in pixels (`px`). Defaults to `320`. Supports values between `135` and `1200`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="paths" type_markdown="Object">
    Paths to common folders. Each path is relative to [global](/documentation/articles/create-a-global-configuration-file/) site `source`. If unset, the paths default to those configured in the [global configuration](/documentation/articles/create-a-global-configuration-file/).

    The available keys are:

    * `dam_static` for the location of statically copied assets for [DAM](/documentation/articles/introduction-to-assets-and-dams/) files.
    * `dam_uploads` for the default location of newly uploaded [DAM](/documentation/articles/introduction-to-assets-and-dams/) files.
    * `dam_uploads_filename` for the name of the uploaded file, when uploading [DAM](/documentation/articles/introduction-to-assets-and-dams/) files.
    * `static` for the location of statically copied assets.
    * `uploads` for the default location of newly uploaded site files.
    * `uploads_filename` for the name of the uploaded file.

    These paths control where new asset files are uploaded to. They also set the default path when choosing existing images, and linking to existing files.

    You can use [dynamic placeholders](/documentation/articles/configure-your-template-strings/) for `uploads` and `dam_uploads`. For more information, please read our documentation on [adjusting file upload paths](/documentation/articles/adjusting-the-uploads-path/).
  </comp.DataReferenceRow>
</comp.DataReference>
