CloudCannon supports embedding rich Snippets in Markdown content when using the CloudCannon Content Editor. Once configured, Snippets in your content will be presented as blocks in rich text views, with the ability to add them as Snippets via the toolbar:

<comp.DocsImage
  path="ye_olde_images/snippet-toolbar.png"
  alt={
    'The CloudCannon Content Editor toolbar with a cursor hovering over the "Snippet" button'
  }
  type="regular"
/>

To start configuring Components in your MDX content, a Snippet configuration must be imported using the `_snippets_imports` key in your CloudCannon global configuration file.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "javascript"]} source="cloudcannon.config.yml">
```
# Import the MDX Components Snippets:
_snippets_imports:
  mdx: true
  docusaurus_mdx: true

```
</comp.MultiCodeBlock>

Docusaurus uses both regular MDX components along with custom extensions, so we import both the `mdx` Snippets and the `docusaurus_mdx` Snippets. You can find detailed documentation on the base `mdx` snippets on the [Editing with MDX Components](/documentation/articles/editing-with-mdx-components/) page.

## Enabling snippets in the toolbar
By default, CloudCannon will show the `snippet` toolbar action in the content editor if snippets are available.

<comp.Notice info_type="important">
  If you have already customized which options are available via `_editables` in your CloudCannon config, you will need to include `snippet: true` for Snippets to be available. See the [Editables options documentation](/documentation/articles/defining-editable-regions-in-your-html/#options) for more details.
</comp.Notice>

## Supported extensions

### Admonitions

With the Admonition snippet you can use Docusaurus [admonition syntax](https://docusaurus.io/docs/markdown-features/admonitions) to highlight rich content in the Content Editor.

<comp.CodeBlock source="example.mdx" language='markdown'>
```
:::note Your Title

Some **content** with _Markdown_ `syntax`.

:::
```
</comp.CodeBlock>

### Code Blocks

The Code Block snippet allows you to create [Docusaurus code blocks](https://docusaurus.io/docs/markdown-features/code-blocks) with syntax highlighting, titles, and line numbers.

<comp.CodeBlock source="example.mdx" language='markdown'>
````
```jsx title="/src/components/HelloCodeTitle.js" showLineNumbers
function HelloCodeTitle(props) {
  return <h1>Hello, {props.name}</h1>;
}
```
````
</comp.CodeBlock>

### Content Tabs

The Content Tabs snippet provides support for adding tab seperated content using the [Docusaurus Tabs component](https://docusaurus.io/docs/markdown-features/tabs). If you're using
the Content Tab snippet we recommend registering it to the global scope by wrapping the MDX Component file.
You can find [documentation on Docusaurus component scope](https://docusaurus.io/docs/markdown-features/react#mdx-component-scope) here.

<comp.CodeBlock source="example.mdx" language='markdown'>
```
<Tabs>
  <TabItem value="apple" label="Apple" default>
    This is an apple üçé
  </TabItem>
  <TabItem value="orange" label="Orange">
    This is an orange üçä
  </TabItem>
  <TabItem value="banana" label="Banana">
    This is a banana üçå
  </TabItem>
</Tabs>
```
</comp.CodeBlock>

### Truncate

The Truncate snippet allows you to mark which content should be included in the blog post summary when using the [Docusaurus blog feature](https://docusaurus.io/docs/blog).

<comp.CodeBlock source="example.mdx" language='markdown'>
```
<!--truncate-->
```
</comp.CodeBlock>
