<comp.DataReference>
  <comp.DataReferenceRow label="data-editable" type_markdown="attibute">
    This attribute defines the type of data inside an Editable.
    
    Value can be `array`, `array-item`, `component`, `image`, `source`, or `text`.

    Alternatively, you can wrap elements in the equivalent web component: `<array-editable>`, `<array-item>`, `<component-editable>`, `<image-editable>`, `<source-editable>`, or `<text-editable>`. If you use the web component, you do not need to define the `data-editable` attribute.

    <comp.Example>
      In this example, we have configured the `page_description` Input as a markdown Rich Text Input using the `type` key.

      <comp.CodeBlock language="HTML" source="index.html">
            ``````````
            <h1 data-editable="text" data-prop="title" >{title}</h1>

            ``````````
            </comp.CodeBlock>

      In this example, we have wrapped the `<slot />` element in the `<text-editable>` web component.

      <comp.CodeBlock language="HTML" source="blog-layout.html">
            ``````````
            <text-editable data-prop="@content">
              <slot />
            </text-editable>

            ``````````
            </comp.CodeBlock>
    </comp.Example>
  </comp.DataReferenceRow>
</comp.DataReference>
