The Source Editor is an in-browser code editor that is great for making minor changes to the source code of your files. This editing interface is useful for all text-based files.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/editing-interfaces/2024-04/CloudCannon-Documentation-SOurce-Editor-Navigation.png" alt="" title="" type="screenshot"/>

<comp.Notice info_type="info">
  Depending on your SSG and site configuration, you may not have access to the Data Editor for all files. An editing interface may not be available because:

  * Some permission levels cannot access the Source Editor (e.g., unavailable to [Editors](/documentation/articles/managing-your-team-members/) or [Clients](/documentation/articles/giving-clients-access-to-update-a-site/)).
  * A developer on your team has [specified which editing interfaces are enabled](/documentation/articles/set-a-default-editing-interface/) for a particular [collection](/documentation/articles/define-your-collections/) or [schema.](/documentation/articles/create-a-schema/)
</comp.Notice>

## Source Editor features

The Source Editor displays the raw content of text-based files. Syntax highlighting in the Source Editor is automatically detected based on the file extension of the file you opened.

For more information, including a walk-through of the *Editing interface navigation* and the *Context menu*, read our documentation on [editing your files](/documentation/articles/edit-your-files).

### Appearance and behavior

You can configure the appearance and behavior of the Source Editor using the `source_editor` key in your [global configuration file](/documentation/articles/create-a-global-configuration-file/).

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/editing-interfaces/2024-04/CloudCannon-Documentation-Source-Editor-Configure-Appearance.png" alt="" title="" type="screenshot"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "javascript"]} source="cloudcannon.config.yml">
```
source_editor:
  tab_size: 2
  show_gutter: false
  theme: pastel_on_dark
```
</comp.MultiCodeBlock>

The Source Editor has the following configuration options available:

<comp.DataReference>
  <comp.DataReferenceRow label="tab_size" type_markdown="Integer">
    Defines how many spaces lines are auto indented by, and how many spaces tabs are shown as. Optional, defaults to `2`.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="theme" type_markdown="String">
    Changes the colors scheme for syntax highlighting in the editor. Optional, defaults to `monokai`. Must be one of:

    `ambiance` `chaos` `chrome` `clouds` `clouds_midnight` `cobalt` `crimson_editor` `dawn` `dracula` `dreamweaver` `eclipse` `github` `gob` `gruvbox` `idle_fingers` `iplastic` `katzenmilch` `kr_theme` `kuroir` `merbivore` `merbivore_soft` `mono_industrial` `monokai` `nord_dark` `pastel_on_dark` `solarized_dark` `solarized_light` `sqlserver` `terminal` `textmate` `tomorrow` `tomorrow_night` `tomorrow_night_blue` `tomorrow_night_bright` `tomorrow_night_eighties` `twilight` `vibrant_ink` `xcode`
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="show_gutter" type_markdown="Boolean">
    Toggles displaying line numbers and code folding controls in the editor. Optional, defaults to `true`.
  </comp.DataReferenceRow>
</comp.DataReference>
