## What is a snippet?

x

<comp.Notice info_type="info">
  We recommend using snippets to create custom styles in your Markdown, rather than [adding custom markup](/documentation/articles/best-practices-for-rich-text/).
</comp.Notice>

Once you have configured your snippets, your team members can easily add them to any Markdown content by clicking the *Insert snippet* button in the *WYSIWYG toolbar* and selecting from the searchable pop-up modal.

### Select a snippet type

CloudCannon supports the following snippet templates:

* [Docusaurus Components](/documentation/articles/snippets-using-docusaurus-components)
* [Eleventy Shortcodes](/documentation/articles/snippets-using-eleventy-shortcodes/)
* [Hugo Shortcodes](/documentation/articles/snippets-using-hugo-shortcodes/)
* [MDX Components](/documentation/articles/snippets-using-mdx-components/)
* [Python Markdown](/documentation/articles/snippets-using-python-markdown/)

### Enabling the snippet button in the WYSIWYG toolbar

By default, CloudCannon will show the *Insert snippet* button in the *WYSIWYG toolbar* if you have at least one snippet. The *WYSIWYG toolbar* appears in the [Content Editor](/documentation/articles/introducing-the-content-editor/) and in [rich text inputs](/documentation/articles/using-rich-text-inputs-to-edit-your-data/).

However, if you have already customized which options are available in your *WYSIWYG toolbar* using the `_editables`key in your [global configuration file](/documentation/articles/create-a-global-configuration-file/), you will need to ensure that `snippet: true` is included.

<comp.MultiCodeBlock language="YAML" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```
_editables:
  content:
    snippet: true
```
</comp.MultiCodeBlock>

For more information, read our documentation on [configuring your rich text toolbars](/documentation/articles/configuring-toolbars/).
