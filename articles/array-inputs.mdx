An Array input lets your team members create a list of inputs, or grouped inputs. Array inputs contain nested inputs of any type, including other Arrays and [Objects](/documentation/articles/object-inputs/). There is no limit to how many nested layers you can create within an Array. Creating a list of inputs, or groups of inputs, is useful for organizing your data.

You can create several predefined [structures](/documentation/articles/create-a-structure/) for your team to use

For each input you can configure which structures are associated with the input, the appearance of the preview card, and how CloudCannon handles empty values. You can also use the [general configuration options](/documentation/articles/configure-your-inputs/#general-configuration-options) available for all inputs.

## Array

The Array input provides a user interface for lists of nested inputs. Array inputs appear as a series of [cards](/documentation/articles/configure-your-card-previews/) in the Data Editor or sidebar of the Visual or Content Editor. Each card previews some of the content the input, or input group, contains.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/array/2024-08/CloudCannon-Documentation-Array-Input-Closed.png" alt="A screenshot of the Array input in the Data Editor, showing a preview of the content within multiple entries." title="The Array input" type="ui-snippet"/>

You can reorder, clone, and add new entries in the list using the *Context menu*. You can also add new entries using the *\+ Add* button below the Array — the text on this button will differ depending on the key name of your input.

By default, adding a new entry to an Array input will clone the last input in the Array.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/array/2024-08/CloudCannon-Documentation-Array-Input-Context-Menu.png" alt="" title="" type="ui-snippet"/>

When you click on a card in the Array, it will open to show the nested inputs within that Array entry. The *Back* button at the top of the input will return you to the parent scope.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/array/2024-08/CloudCannon-Documentation-Array-Input-Open.png" alt="" title="" type="ui-snippet"/>

Array inputs can contain a mixture of inline and nested values simultaneously. By default, text, number, and boolean values are inline fields. Other values, including nested inputs, Objects, or Arrays appear as cards.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/array/2024-08/CloudCannon-Documentation-Array-Input-Value-Types.png" alt="" title="" type="ui-snippet"/>

You can configure an Array input using the `type` key with the value `array`. Define the `type` key under your input key name within `_inputs`. For more information, please read our documentation on [configuring your inputs](/documentation/articles/configure-your-inputs/).

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs___1___:
  staff___2___:
    type___3___: array

```
<comp.Annotation number="1">
  All inputs are defined under the `_inputs` key, regardless of where they are in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
</comp.Annotation>
<comp.Annotation number="2">
  This Array input is called `staff`.
</comp.Annotation>
<comp.Annotation number="3">
  The value of the `type` key determines the input type. This is an `array` input.
</comp.Annotation>
</comp.MultiCodeBlock>

<comp.Notice info_type="important">
  For Array inputs to function correctly, they must contain nested inputs. You can add nested inputs using the [Source Editor](/documentation/articles/the-source-editor/). Alternatively, you can use a structure to allow team members to populate the Array.
</comp.Notice>

Once configured, the Array input will appear in the Data Editor or sidebar of the Visual or Content Editor when you add it to a data file or the front matter of a markup file.

<comp.CodeBlock language="Markdown" source="page.md">
```
---
staff:
  - name: Karen Key
    position: Project Manager
    image: /images/team/karen_key.jpg
  - name: Holly James
    position: Marketing Manager
    image: /images/team/holly_james.jpg
---

Page content goes here.
```
</comp.CodeBlock>

## Array variations

There are two ways to tell CloudCannon how to populate your Array input, byt configuring the entry input type or a structure.

### Entry input type

By defining the entry input type, each new entry added to the Array will match the input type specified.

You can configure the entry input type for your Array input by specifying the input name and entry types under the `_inputs` key. To specify future entries within a particular Array, use the key name of the Array followed by the `[*]` characters. Define the type of input for those entries with the `type` key.

<comp.DocsImage path="https://cc-dam.imgix.net/documentation/images/editing/inputs/array/2024-08/CloudCannon-Documentation-Array-Input-of-Images.png" alt="" title="" type="ui-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

_inputs:
  gallery:
    type: array
  gallery[*]:
    type: image

```
</comp.MultiCodeBlock>

## Options

You can configure Array inputs using the `options` key under your input key, inside of `_inputs`.

<comp.MultiCodeBlock language="yaml" translate_into={["json", "JavaScript"]} source="cloudcannon.config.yml">
```

page_sections:
_inputs:
  page_sections:
    type: array
    options:
      structures: _structures.components
      preview:
        text:
          - key: name
        subtext:
          - key: description
        image:
          - key: my_image
        icon: article
      empty_type: string

```
</comp.MultiCodeBlock>

Array inputs have the following options available:

<comp.DataReference>
  <comp.DataReferenceRow label="structures" type_markdown="String or object">
    This key determines which predefined templates to use for populating the Object Input. When configured, team members can select a [structure](/documentation/articles/create-a-structure/) to populate the Object with input/input groups. This key has no default.

    If configured as an object, CloudCannon will use the values directly.

    If configured as a string, CloudCannon will use the matching structures value defined under `_structures` in the [configuration cascade](/documentation/articles/using-the-configuration-cascade/).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview" type_markdown="object">
    This key enables you to define the appearance of the input in the Data Editor or sidebar of the Visual or Content Editor. The following nested keys are available:

    * `text`
    * `subtext`
    * `icon`
    * `image`

    If the Object Input uses `structures`, CloudCannon will use the `preview` key from the structure instead.

    For more information about previews, please read our documentation on [configuring card previews](/documentation/articles/configure-your-card-previews/).
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.text" type_markdown="array, string, or boolean">
    This key determines the title text displayed on a default Object Input card. This key has no default, and falls back to the first nested value found (prioritizing text-based values).

    If set to `false`, no text is displayed.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.subtext" type_markdown="array, string, or boolean">
    This key determines the subtitle text displayed on a default Object Input card. This key has no default, and falls back to listing the `label` of each nested input. `subtext` is hidden if text and subtext are the same. If there is no text, subtext will appear in the place of text.

    If set to `false`, no text is displayed.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.icon" type_markdown="array, string, or boolean">
    This key determines the icon displayed on a default Object Input card. The value must match a name from the [Material Icons](https://fonts.google.com/icons?icon.set=Material+Icons) list. Defaults checking the `icon` key, then falls back to the `notes` icon.

    If set to `false`, no icon is displayed.

    If `image` is defined, the image replace `icon` when loaded successfully.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="preview.image" type_markdown="array, string, or boolean">
    This key determines the image displayed on a default Object Input card. This key has no default, and falls back to `thumbnail_image`, `thumbnail_image_path`, `image`, and `image_path` in that order. If no image is found, `icon` is displayed instead.

    If set to `false`, no image is displayed.
  </comp.DataReferenceRow>
  <comp.DataReferenceRow label="empty_type" type_markdown="String">
    This key determines how CloudCannon handles an empty value. This key does not apply to existing empty values.

    Value must be one of the following:

    * `string` - an empty value for this input will be stored as "".
    * `null` - an empty value for this input will be stored as a null value (default). This does not apply to TOML files.
  </comp.DataReferenceRow>
</comp.DataReference>

## Unconfigured Array inputs

In some cases, CloudCannon can still detect an Array input even if you have not configured it.

CloudCannon will interpret any unconfigured input with a list of nested values or keys as an Array input. You cannot create structured Array inputs without configuration.

<comp.CodeBlock language="YAML" source="data.yml">
```
animals:
  - bear
  - lizard
  - swan
  - name: Dog
    fun_fact: Man's best friend.
    image: /images/my-dog.png
```
</comp.CodeBlock>

This behavior is convenient if you have simple inputs, or do not want to configure inputs. It is also beneficial for new websites on CloudCannon where you have yet to create any CloudCannon-specific configuration.

<comp.Notice info_type="info">
  We recommend configuring your inputs for greater control over their functionality and appearance.
</comp.Notice>

## Array

Grouped list of inputs with controls to add, remove and reorder items.

Array inputs are shown for inputs configured with the type `array`, or for array input values.

### Inline values

Inline values in Array inputs are displayed and edited with inputs directly in the list. By default, text, number and boolean values are displayed inline as text, number and checkbox inputs.

<comp.DocsImage path="/documentation/static/inputs/array/array.png" alt="The array editor with draggable handles, individual menus, and a button at the bottom to add a new item." title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Value convention">
```

staff:
  - Karen
  - Billy
  - Holly

```
</comp.MultiCodeBlock>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

sections:
  _inputs:
    sections:
      type: array

```
</comp.MultiCodeBlock>

You can configure arrays to use different inputs for inline values with [input configuration](/documentation/articles/configure-your-inputs/). Each value is still stored as either a text, number or boolean value — but is now edited with a more suitable input.

Here's an example for an array of Image File inputs:

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

gallery:
  - /uploads/mountain.png
  - /uploads/beach.png
_inputs:
  gallery:
    type: array
  gallery[*]:
    type: image

```
</comp.MultiCodeBlock>

Here's another example for an array of Select inputs, populated from a `posts` collection:

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Input configuration">
```

other_posts:
  - /posts/top-10-things.md
  - /posts/yesterday-i-found.md
_inputs:
  other_posts:
    type: array
  other_posts[*]:
    type: select
    options:
      values: collections.posts

```
</comp.MultiCodeBlock>

### Structured or nested values

Array inputs containing object or array values are shown as a list of [Object](/documentation/articles/object-inputs/) inputs. Team members add, remove or reorder items directly on this input. Item values are edited after opening each item.

This allows your team members to intuitively manage complex data. When your data can be deeply nested, there is no end to what can be achieved. Rendering the array value from this input with a loop on your site creates powerful editing opportunities for your team members.

<comp.DocsImage path="/documentation/static/inputs/structures/structure.png" alt="The array editor with a list of staff members including their thumbnail image and name" title="" type="screenshot-snippet"/>

<comp.DocsImage path="/documentation/static/inputs/structures/structure-item-menu.png" alt="The array editor with a list of staff members including their thumbnail image and name" title="" type="screenshot-snippet"/>

<comp.MultiCodeBlock language="yaml" translate_into={["json", "toml"]} source="Value convention">
```

quotes:
  - name: 'Betty Jefferson'
    position: 'Jamstack Developer'
    image: /images/betty.svg
    quote_markdown: '“This quotation can be edited via the front matter textarea input.”'
  - name: 'Jamie Mills'
    position: 'Jamstack Developer'
    image: /images/jamie.svg
    quote_markdown: '“You can also reorder quotations via the front matter arrays input.”'

```
</comp.MultiCodeBlock>
