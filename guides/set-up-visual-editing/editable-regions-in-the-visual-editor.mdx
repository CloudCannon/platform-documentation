CloudCannon has four editing interfaces, each suited to editing different types of files. The Visual Editor is an intuitive interface for any HTML-like files and hybrid files (e.g., `.mdx`) that output to an HTML page on your website. By defining *Editable Regions* on your pages, you can visually edit content in HTML-like layouts, structured data in your front matter, and Markup content.

Let's take a look at some of the basics for *Editable Regions*. If you would rather start configuring your *Site* straight away, feel free to skip to the next step in this guide.

An *Editable Region* in the Visual Editor is signified by a yellow box around a piece of content. Clicking into the *Editable Region* will have a different result depending on the type of content it contains.

For text, you can type inline on your webpage. CloudCannon will also show a WYSIWYG toolbar for formatting your rich text content.

Clicking an image in *Editable Region* will open a *Data Panel*, allowing you to upload a new image, select an existing one from your repository or DAM, and update image details such as title or alt text.

You can turn the Visual Editor into a page builder. *Editable Regions* on arrays of content enable you to add new components to a page, remove existing ones, and reorganize your components live on a preview of your webpage at any time.

## Defining *Editable Regions*

In CloudCannon, defining *Editable Regions* in your files is as simple as adding an HTML attribute or web component to any DOM element on your website. There are five types you can add:

* **Text** —Adding the HTML attribute `data-editable="text"` or the web component `<editable-text>` allows you to edit inline text, paragraph-level rich text, or multi-paragraph rich text (i.e., block-level).
* **Image** — Adding the HTML attribute `data-editable="image"` or the web component `<editable-image>` allows you to upload or select existing images and manage image details.
* **Source** — Adding the HTML attribute `data-editable="source"` or the web component `<editable-source>` allows you to...
* **Array** — Adding the HTML attribute `data-editable="array"` or the web component `<editable-array>` allows you to...
* **Component** — `data-editable="text"` and `<editable-text>`

<comp.DataReference>
  <comp.DataReferenceRow label="data-editable" type_markdown="HTML Attribute">
    This HTML attribute defines the type of *Editable Region* that CloudCannon should use in the Visual Editor. You can add this HTML attribute to a DOM element in any file type as long as it outputs to an HTML file after a *Site* build.
    
    Values can be one of the following: `text`, `image`, `source`, `array`, `array-item`, or `component`.
    
    Depending on the value of this HTML element, the Visual Editor will also require other HTML attributes for *Editable Regions* to function.
    
    If you cannot use an appropriate DOM element, you can wrap elements in the equivalent web component, which will accept HTML attributes. The web component can be one of the following: `<editable-text>`, `<editable-image>`, `<editable-source>`, `<editable-array>`, `<editable-array-item>`, or `<editable-component>`.

    <comp.Example>

      In this example, we have added `data-editable="text"` HTML attribute to the `<h1>` tag to define a Text *Editable Region*.

      <comp.CodeBlock language="Astro"  source="file.astro">
      ``````````
      <html lang="en">
        <body>
          <main>
    		<article>                
              <h1 data-editable="text" data-prop="title" >{title}</h1>
            </article>
          </main>
        </body>
      </html>

      ``````````
      </comp.CodeBlock>
      
      In this example, we have wrapped the `<slot />` element in the `<editable-text>` web component to define a Text *Editable Region*, as no DOM element was available.

      <comp.CodeBlock language="Astro"  source="file.astro">
      ``````````
      <html lang="en">
        <body>
          <main>
    		<article>
              <text-editable data-prop="@content">
                <slot />
              </text-editable>
            </article>
          </main>
        </body>
      </html>

      ``````````
      </comp.CodeBlock>
    </comp.Example>
