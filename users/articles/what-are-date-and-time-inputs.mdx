Date and Time inputs are editing interfaces for date and time values. You can use these inputs in the Data Editor, Content Editor, or Visual Editor to edit structured data in data files or front matter.

There are three types of Date and Time inputs:

* Datetime
* Date
* Time

## Date and Time input types

### Datetime

The Datetime input provides an editing interface for date and time values.

You can edit the value of a Datetime input by clicking the date or time text field. Clicking on the date text field will open a calendar for selecting dates.

<comp.DocShot docshot_key="CloudCannon-Documentation-Datetime-Input-Closed" alt="A screenshot of the Datetime input in the Data Editor shows two text fields for date and time values." title="The Datetime Input" type="ui-snippet"/>

<comp.DocShot docshot_key="CloudCannon-Documentation-Datetime-Input-Open" alt="A screenshot of the Datetime input in the Data Editor shows an open calendar widget under the date text field." title="The Datetime Input " type="ui-snippet"/>

The Timezone for your Datetime input appears above the input text field.

### Date

The Date input provides an editing interface for date values only.

You can edit the value of a Date input by clicking the date text field. Clicking on the date text field will open a calendar for selecting dates.

<comp.DocShot docshot_key="CloudCannon-Documentation-Date-Input-Closed" alt="A screenshot of the Date input in the Data Editor shows a text field for a date value." title="The Date Input" type="ui-snippet"/>

<comp.DocShot docshot_key="CloudCannon-Documentation-Date-Input-Open" alt="A screenshot of the Date input in the Data Editor shows an open calendar widget under the date text field." title="The Date Input " type="ui-snippet"/>

The Timezone for your Date input appears above the input text field.

### Time

The Time input provides an editing interface for time values only.

You can edit the value of a Datetime input by typing into the time text field.

<comp.DocShot docshot_key="CloudCannon-Documentation-Time-Input" alt="A screenshot of the Time input in the Data Editor shows a text field for a time value." title="The Time Input" type="ui-snippet"/>

## Date and Time input appearance and behavior

You can customize the label, comment, documentation link, and context box for all inputs regardless of type.

<comp.DocShot docshot_key="CloudCannon-Documentation-Datetime-Input-Datetime-With-Context" alt="A screenshot of the Datetime Input in the Data Editor with a Context Box underneath the text field." title="The Datetime Input with Context Box" type="ui-snippet"/>

Depending on how your developer configured the input, your Date or Time input may have a different Timezone.

<comp.DocShot docshot_key="CloudCannon-Documentation-Datetime-Input-Datetime-Timezone" alt="A screenshot of the Datetime Input shows it is using the America/New_York timezone." title="The Datetime Input" type="ui-snippet"/>

### Input validation

Your Date or Time input may also have input validation, which requires the value to match predefined criteria before you can save your changes.

CloudCannon will show a red `*` next to the name of your Date or Time input if a value is required.

<comp.DocShot docshot_key="CloudCannon-Documentation-Datetime-Input-Datetime-Required" alt="A screenshot of the Datetime input in the Data Editor shows that no value is causing an input validation error." title="Datetime Input with required value" type="ui-snippet"/>

If you enter a value before the earliest allowed date or after the latest allowed date, CloudCannon will display an error message in red text under the input field.

<comp.DocShot docshot_key="CloudCannon-Documentation-Datetime-Input-Datetime-Start-From" alt="A screenshot of the Date input in the Data Editor shows that the current value does not meet validation criteria." title="The Date Input with value validation" type="ui-snippet"/>

For more information on configuration options, valid input values, and unconfigured input behavior, please read our developer documentation on [configuring a Date or Time input](/documentation/articles/configure-a-date-or-time-input/) .

## Misconfigured Date and Time inputs

You will see an orange warning box if your Date or Time input is misconfigured.

<comp.DocShot docshot_key="CloudCannon-Documentation-Datetime-Input-Date-Misconfigured" alt="A screenshot of a misconfigured Date input in the Data Editor shows an orange warning box." title="A misconfigured Date Input" type="ui-snippet"/>

Date and Time inputs are misconfigured if:

* The value of a Datetime or Date is not a valid <a href="https://www.iso.org/iso-8601-date-and-time-format.html" target="_blank" rel="noopener">ISO8601</a> format.
* The value of a Time input is not a string.

For more information on how to fix a misconfigured Date or Time input, please read our developer documentation on [configuring a Date or Time input](/documentation/articles/configure-a-date-or-time-input/).
