A <comp.GlossaryTerm term="/user/glossary/p/project.yml">Project</comp.GlossaryTerm> is a way to group multiple <comp.GlossaryTerm term="/user/glossary/s/site.yml">Sites</comp.GlossaryTerm> from the same <comp.GlossaryTerm term="/user/glossary/g/git-repository.yml">Git Repository</comp.GlossaryTerm>. The *Projects Browser* lists all your *Projects*, and each *Project* page contains tabs for browsing *Sites*, visualizing publishing, reviewing *Pull Requests*, exploring branches, and managing your *Project's* settings.

## The *Projects Browser*

<comp.DocShot docshot_key="CloudCannon-Documentation-Projects-Browser" alt="A screenshot of the Projects Browser showing a list of Project cards." title="The Projects Browser" type="screenshot"/>

The *Projects Browser* lists all the *Projects* in your *Organization*. You can access the *Projects Browser* by clicking the *Projects* link in the *App Sidebar*.

From the *Projects Browser*, you can:

* View all your *Projects* at a glance
* Sort and filter *Projects* by name, URL, or last synced date
* Click a *Project Card* to open the *Project Sites Browser*
* Create a new *Project* using the *Create a Project* button

## The *Project Sites Browser*

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Sites-Browser-With-Links" alt="A screenshot of the Project page showing the Sites tab with multiple Sites connected to different branches." title="The Project Sites Browser" type="screenshot"/>

The *Project Sites Browser* is the main view for an individual *Project*. It shows all the *Sites* connected to your *Project* and provides access to the *Publishing*, *Pull Requests*, *Branches*, and *Project Settings* tabs.

For each *Site*, the *Project Sites Browser* shows:

* The *Site* name and branch name
* The domain or testing URL
* The build and sync status
* Which branch the *Site* publishes to (e.g., "publishes to main")

You can switch between card and list views, and use the *Create a Site* button to [add a new Site to your Project](/documentation/developer-articles/add-a-site-to-your-project/).

## The *Project Settings* tab

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Settings-Details" alt="A screenshot of the Project Settings tab showing the Details section with fields for Project Name, Display URL, Description, and Related Links." title="The Project Settings tab" type="screenshot"/>

The *Project Settings* tab allows you to manage your *Project* across several sections:

* **Details** — Update your *Project Name*, *Display URL*, *Description*, and *Related Links*. You can also [turn on optional Project tabs](/documentation/developer-articles/turn-on-optional-project-tabs/) from this section.
* **Repository** — View or change the connected *Git Provider* and *Repository*, toggle *Allow all branching*, and [set the Main Branch](/documentation/developer-articles/set-the-main-branch-for-your-project/).
* **Branch Defaults** — Configure default settings for new branch *Sites*, including the publishing method, authentication, and permissions.
* **Deploy Previews** *(Team or Enterprise plan)* — Configure *Pull Request* comments and automatic *Site* creation for *Pull Requests*. For more information, please read our documentation on [Deploy Previews](/documentation/developer-articles/what-are-deploy-previews/).
* **Danger** — Delete your *Project*. This does not delete the *Sites* within it.

## Optional *Project* tabs

By default, new *Projects* show only the *Sites* and *Project Settings* tabs. Your developer can turn on three more tabs from the *Details* section under *Project Settings*:

* **Publishing** — a graph of how the *Sites* in your *Project* publish to each other.
* **Pull Requests** *(Team or Enterprise plan)* — the open and draft *Pull Requests* in the *Git Repository* connected to your *Project*, alongside any *Sites* connected to each.
* **Branches** — the branches in the *Git Repository* connected to your *Project*, grouped by status, alongside any *Sites* connected to each.

For more information on what each tab shows and how to read it, please read our documentation on [the optional Project tabs](/documentation/user-articles/what-are-the-optional-project-tabs/). To turn them on, please read our documentation on [turning on optional Project tabs](/documentation/developer-articles/turn-on-optional-project-tabs/).
