A <comp.GlossaryTerm term="/user/glossary/p/project.yml">Project</comp.GlossaryTerm> is a way to group multiple <comp.GlossaryTerm term="/user/glossary/s/site.yml">Sites</comp.GlossaryTerm> from the same <comp.GlossaryTerm term="/user/glossary/g/git-repository.yml">Git Repository</comp.GlossaryTerm>. The *Projects Browser* lists all your *Projects*, and each *Project* page contains tabs for browsing *Sites*, visualizing publishing, reviewing *Pull Requests*, exploring branches, and managing your *Project's* settings.

## The Projects Browser

<comp.DocShot docshot_key="CloudCannon-Documentation-Projects-Browser" alt="A screenshot of the Projects Browser showing a list of Project cards." title="The Projects Browser" type="screenshot" />

The *Projects Browser* lists all the *Projects* in your *Organization*. You can access the *Projects Browser* by clicking the *Projects* link in the *App Sidebar*.

From the *Projects Browser*, you can:

* View all your *Projects* at a glance
* Sort and filter *Projects* by name, URL, or last synced date
* Click a *Project Card* to open the *Project Sites Browser*
* Create a new *Project* using the *Create a Project* button

## The Project Sites Browser

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Sites-Browser-With-Links" alt="A screenshot of the Project page showing the Sites tab with multiple Sites connected to different branches." title="The Project Sites Browser" type="screenshot" />

The *Project Sites Browser* is the main page for an individual *Project*. It shows all the *Sites* connected to your *Project* and provides access to the *Publishing*, *Pull Requests*, *Branches*, and *Project Settings* tabs.

For each *Site*, the *Project Sites Browser* shows:

* The *Site* name and branch name
* The domain or testing URL
* The build and sync status
* Which branch the *Site* publishes to (e.g., "publishes to main")

You can switch between card and list views, and use the *Create a Site* button to [add a new Site to your Project](/documentation/developer-articles/add-a-site-to-your-project/).

Some tabs are optional. By default, new *Projects* show only the *Sites* and *Project Settings* tabs. To turn on the *Publishing*, *Pull Requests*, or *Branches* tabs, please read our documentation on [turning on optional Project tabs](/documentation/user-articles/turn-on-optional-project-tabs/).

## The Publishing tab

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Publishing-Tab" alt="A screenshot of the Publishing tab showing the relationship between branch Sites and the main branch." title="The Publishing tab" type="screenshot" />

The *Publishing* tab on your *Project* page gives you a visual overview of how your *Sites* relate to each other. It displays a graph showing each *Site*, which branch it is connected to, and which branch it publishes to. This makes it easy to see the full publishing workflow at a glance.

This is especially helpful for *Projects* with multiple levels of branching, where it can be difficult to remember the publishing relationships between *Sites*.

For more information on configuring publishing behavior for individual *Sites*, please read our documentation on [connecting a Publish Branch](/documentation/developer-articles/connect-a-publish-branch/).

## The Pull Requests tab

<comp.Notice info_type="pricing">
  **The *Pull Requests* tab is available on our [Team and Enterprise plans](/pricing/).**
</comp.Notice>

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Pull-Requests-Tab" alt="A screenshot of the Pull Requests tab on the Project page, showing a list of open Pull Requests grouped under Pull Requests and Drafts headings, with the Sites connected to each Pull Request." title="The Pull Requests tab" type="screenshot" />

The *Pull Requests* tab on your *Project* page shows the open *Pull Requests* in the *Git Repository* connected to your *Project*. *Pull Requests* are grouped into *Pull Requests* (open and ready for review) and *Drafts*.

Each entry shows the *Pull Request* title, branch, and any *Sites* connected to it, so you can jump from a *Pull Request* to its corresponding branch *Site* in CloudCannon.

For more information on configuring comments and automatic *Sites* for *Pull Requests*, please read our documentation on [Deploy Previews](/documentation/developer-articles/what-are-deploy-previews/).

## The Branches tab

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Branches-Tab" alt="A screenshot of the Branches tab on the Project page, showing branches grouped under Main Branch, Connected Branches, and Other Branches headings, with the Sites connected to each branch." title="The Branches tab" type="screenshot" />

The *Branches* tab on your *Project* page shows the branches in the *Git Repository* connected to your *Project*. Branches are grouped into the *Main Branch*, *Connected Branches* (branches with a connected *Site*), and *Other Branches* (branches without a connected *Site*).

Each entry shows the branch name and any *Sites* connected to it, so you can navigate from a branch to its corresponding *Site* in CloudCannon.

## The Project Settings tab

<comp.DocShot docshot_key="CloudCannon-Documentation-Project-Settings-Details" alt="A screenshot of the Project Settings tab showing the Details section with fields for Project Name, Display URL, Description, and Related Links." title="The Project Settings tab" type="screenshot" />

The *Project Settings* tab allows you to manage your *Project* across several sections:

* **Details** — Update your *Project Name*, *Display URL*, *Description*, and *Related Links*. You can also [turn on optional Project tabs](/documentation/user-articles/turn-on-optional-project-tabs/) from this section.
* **Repository** — View or change the connected *Git Provider* and *Repository*, toggle *Allow all branching*, and [set the Main Branch](/documentation/developer-articles/set-the-main-branch-for-your-project/).
* **Branch Defaults** — Configure default settings for new branch *Sites*, including the publishing method, authentication, and permissions.
* **Deploy Previews** *(Team and Enterprise plans)* — Configure *Pull Request* comments and automatic *Site* creation for *Pull Requests*. For more information, please read our documentation on [Deploy Previews](/documentation/developer-articles/what-are-deploy-previews/).
* **Danger** — Delete your *Project*. This does not delete the *Sites* within it.
