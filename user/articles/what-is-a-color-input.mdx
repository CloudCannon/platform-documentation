A Color input is an editing interface for color information. You can use this input in the Data Editor, Content Editor, or Visual Editor to edit structured data in data files or front matter.

There is only one type of Color input:

* Color

## Color input types

### Color

The Color input provides an editing interface for color values.

You can edit the value of a Color input by typing in the input text field.

<comp.DocShot docshot_key="CloudCannon-Documentation-Color-Input-HEX-Open" alt="A screenshot of the Color input in the Data Editor shows a HEX value in the text field." title="The Color Input" type="ui-snippet"/>

## Color input appearance and behavior

You can customize the label, comment, documentation link, and context box for all inputs regardless of type.

<comp.DocShot docshot_key="CloudCannon-Documentation-Color-Input-With-Context" alt="A screenshot of the Color Input in the Data Editor with a Context Box underneath the text field." title="Color Input with Context Box" type="ui-snippet"/>

Depending on how your developer configured the input, your Color input may have a different color format or color transparency controls.

<comp.DocShot docshot_key="CloudCannon-Documentation-Color-Input-RGBA-Open" alt="A screenshot of the Color input in the Data Editor shows a RGBA value in the text field." title="Color Input with RGBA value" type="ui-snippet"/>

### Input validation

Your Color input may also have input validation, which requires the value to match predefined criteria before you can save your changes.

CloudCannon will show a red `*` next to the name of your Color input if a value is required.

<comp.DocShot docshot_key="CloudCannon-Documentation-Color-Input-Required-Validation" alt="A screenshot of the Color input in the Data Editor shows that no value is causing an input validation error." title="Color Input with required value" type="ui-snippet"/>

If you enter a value under the minimum or over the maximum character count, CloudCannon will display an error message in red text under the input field.

<comp.DocShot docshot_key="CloudCannon-Documentation-Color-Input-Max-Length-Validation" alt="A screenshot of the Color input in the Data Editor shows that the value does not meet maximum length requirements." title="Color Input with max length validation" type="ui-snippet"/>

For more information on configuration options, valid input values, and unconfigured input behavior, please read our developer documentation on [configuring a Color input](/documentation/articles/configure-a-color-input/) .

## Misconfigured Color inputs

You will see an orange warning box if your Color input is misconfigured.

<comp.DocShot docshot_key="CloudCannon-Documentation-Color-Input-Misconfigured" alt="A screenshot of a misconfigured Color input in the Data Editor shows an orange warning box." title="A misconfigured Color Input" type="ui-snippet"/>

Color inputs are misconfigured if:

* The value is not a string.

For more information on how to fix a misconfigured Color input, please read our developer documentation on [configuring a Color input](/documentation/articles/configure-a-color-input/).
